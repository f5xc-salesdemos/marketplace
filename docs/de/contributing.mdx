Diese Anleitung erklärt, wie Sie ein neues Plugin erstellen und
zum f5xc-salesdemos Marktplatz hinzufügen.

## Plugin-Struktur

Jedes Plugin befindet sich in einem eigenen Verzeichnis unter `plugins/` und
folgt dieser Struktur:

<FileTree>
- plugins/
  - my-plugin/
    - .claude-plugin/
      - plugin.json
    - skills/
      - my-skill/
        - SKILL.md
        - references/
          - reference-data.md
    - commands/
      - my-command.md
    - agents/
      - my-agent.md
    - README.md
</FileTree>

## Ein Plugin erstellen

<Steps>

1. **Das Plugin-Verzeichnis erstellen**

   ```bash
   mkdir -p plugins/my-plugin/.claude-plugin
   mkdir -p plugins/my-plugin/skills
   mkdir -p plugins/my-plugin/commands
   ```

2. **`plugin.json` schreiben**

   Erstellen Sie `plugins/my-plugin/.claude-plugin/plugin.json`:

   ```json
   {
     "name": "my-plugin",
     "description": "What this plugin does",
     "version": "1.0.0",
     "author": {
       "name": "f5xc-salesdemos"
     },
     "homepage": "https://github.com/f5xc-salesdemos/marketplace/tree/main/plugins/my-plugin",
     "keywords": ["relevant", "keywords"],
     "license": "Apache-2.0",
     "repository": "https://github.com/f5xc-salesdemos/marketplace"
   }
   ```

3. **Skills hinzufügen**

   Erstellen Sie `plugins/my-plugin/skills/my-skill/SKILL.md` mit
   Frontmatter:

   ```markdown
   ---
   name: my-skill
   description: One-line description of what this skill does
   ---

   Detailed instructions for Claude when this skill activates.
   ```

   Platzieren Sie Referenzdaten in einem `references/`-Unterverzeichnis
   neben `SKILL.md`.

4. **Befehle hinzufügen**

   Erstellen Sie `plugins/my-plugin/commands/my-command.md`:

   ```markdown
   ---
   description: What this command does
   argument_hint: "[optional-args]"
   allowed_tools:
     - Read
     - Glob
     - Grep
   ---

   Instructions for Claude when the user invokes this command.
   ```

5. **Das Plugin zur marketplace.json hinzufügen**

   Fügen Sie einen Eintrag zum `plugins`-Array in
   `.claude-plugin/marketplace.json` hinzu:

   ```json
   {
     "name": "my-plugin",
     "description": "What this plugin does",
     "version": "1.0.0",
     "author": { "name": "f5xc-salesdemos" },
     "source": "./plugins/my-plugin",
     "category": "productivity",
     "homepage": "https://github.com/f5xc-salesdemos/marketplace/tree/main/plugins/my-plugin",
     "license": "Apache-2.0",
     "keywords": ["relevant", "keywords"],
     "tags": ["searchable", "tags"],
     "repository": "https://github.com/f5xc-salesdemos/marketplace"
   }
   ```

6. **Lokal testen**

   Testen Sie Ihr Plugin ohne Veröffentlichung, indem Sie Claude Code
   auf das lokale Verzeichnis verweisen:

   ```bash
   claude --plugin-dir ./plugins/my-plugin
   ```

   Überprüfen Sie, dass Skills im richtigen Kontext aktiviert werden und
   Befehle die erwartete Ausgabe liefern.

</Steps>

## Ihr Plugin einreichen

<Aside type="tip" title="Repository-Workflow">
  Dieses Repository setzt einen strikten Governance-Workflow durch.
  Siehe [CONTRIBUTING.md](https://github.com/f5xc-salesdemos/marketplace/blob/main/CONTRIBUTING.md)
  für den vollständigen Prozess.
</Aside>

<Steps>

1. Erstellen Sie ein GitHub-Issue, das das neue Plugin beschreibt
2. Erstellen Sie einen Feature-Branch von `main`
3. Fügen Sie Ihr Plugin-Verzeichnis hinzu und aktualisieren Sie `marketplace.json`
4. Fügen Sie eine Dokumentationsseite unter `docs/plugins/my-plugin.mdx` hinzu
5. Öffnen Sie einen PR mit Verweis auf das Issue mit `Closes #N`
6. Beheben Sie etwaige CI-Fehler und mergen Sie nach bestandenen Prüfungen

</Steps>

## Bewährte Vorgehensweisen

- **Skills fokussiert halten** — ein Skill sollte eine Sache gut erledigen
- **Referenzdateien verwenden** — platzieren Sie große Datensätze, Schemata und
  Nachschlagetabellen in `references/` anstatt sie inline in SKILL.md einzubetten
- **Intelligent eingrenzen** — prüfen Sie auf nicht committete Änderungen, um
  den Arbeitsbereich wenn möglich einzuschränken
- **Ausgabe nach Schweregrad gruppieren** — verwenden Sie die Stufen ERROR, WARNING und INFO,
  damit Benutzer Korrekturen priorisieren können
- **Eine README schreiben** — fügen Sie Verwendungsbeispiele und eine Beschreibung
  dessen hinzu, was jeder Skill und Befehl tut
