Das **f5xc-devcontainer**-Plugin bietet Container-Awareness für
die f5xc-salesdemos-Entwicklungsumgebung. Es pflegt einen
umfassenden Werkzeugkatalog, ermöglicht Live-Introspektion des
Containers und übernimmt die Werkzeuginstallation mit
Persistenz-Tracking über GitHub Issues.

<Badge text="v1.1.6" variant="note" /> <Badge text="Produktivität" variant="tip" />

## Installation

```
/plugin install f5xc-devcontainer@f5xc-salesdemos-marketplace
```

## Skills

### tool-catalog

Wissensbasis aller im Devcontainer installierten CLI-Werkzeuge,
organisiert nach Kategorie (Cloud, Netzwerk, Sicherheit,
Entwicklung usw.). Wird aktiviert, wenn Sie fragen, welches
Werkzeug Sie verwenden sollen, wie ein Werkzeug verwendet wird,
ob etwas installiert ist oder Hilfe bei einer bestimmten
Operation benötigen.

### self-awareness

Skill für Container-Identität und Selbstdiagnose. Bietet
Live-Introspektion des laufenden Containers über die GitHub-API
und lokale Build-Metadaten. Wird bei Fragen zu Identität,
Version, Build-Informationen, Gesundheitsprüfungen und
Mitwirkenden-Historie aktiviert.

<Aside type="tip">
  Fragen Sie "Wer bist du?" oder "Welche Version verwende ich?",
  um die Self-Awareness auszulösen.
</Aside>

## Agenten

### tool-advisor

Liest den Werkzeugkatalog und gibt Empfehlungen mit Zweck,
Schnellstart-Befehlen und Authentifizierungsanforderungen für
jede CLI-Werkzeug-Frage zurück.

### tool-auditor

Prüft das Devcontainer-Dockerfile gegen die
Werkzeugkatalog-Referenzdateien, um Abweichungen zu erkennen
&mdash; Werkzeuge, die zum Image hinzugefügt, aber nicht
katalogisiert wurden, oder Katalogeinträge für Werkzeuge, die
entfernt wurden.

### container-introspector

Führt Live-Introspektion des Containers über die GitHub-API und
lokale Metadaten durch – für Identität, Herkunft, Selbstdiagnose,
Mitwirkenden-Historie und Build-Abstammung. Ausschließlich
lesende Operationen.

### container-maintainer

Installiert, entfernt, sucht und aktualisiert CLI-Werkzeuge im
laufenden Container unter Verwendung korrekter
Paketmanager-Muster. Erstellt GitHub Issues im
Devcontainer-Repository für das Persistenz-Tracking.
