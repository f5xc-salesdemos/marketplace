Das **f5xc-docs-pipeline**-Plugin stellt Wissen über die
Dokumentations-Pipeline-Architektur für f5-sales-demo-Repositories
bereit. Es unterstützt die Inhalterstellung, klärt die
Konfigurationsverantwortung über Multi-Repo-Builds hinweg und
ermöglicht eine lokale Docker-basierte Vorschau.

<Badge text="v1.0.2" variant="note" /> <Badge text="Produktivität" variant="tip" />

## Installation

```
/plugin install f5xc-docs-pipeline@f5-sales-demo-marketplace
```

## Fähigkeiten

### content-author

Leitfaden zur Inhalterstellung, der Dateistruktur, MDX-Syntaxregeln,
Frontmatter-Anforderungen, Bildreferenzen und lokale Docker-Vorschau
abdeckt. Wird aktiviert beim Erstellen oder Bearbeiten von `docs/`-Inhalten,
beim Arbeiten mit MDX-Dateien oder beim Einrichten der lokalen Vorschau.

### pipeline-navigator

Erklärung der Dokumentations-Pipeline-Architektur, die aufzeigt, wo
Änderungen in den vier Upstream-Repositories vorgenommen werden müssen:
docs-theme, docs-builder, docs-control und docs-icons. Klärt die
Konfigurationsverantwortung (`astro.config.mjs`, `content.config.ts`),
die Release-Dispatch-Kette und das Tracking verwalteter Dateien.

<Aside type="tip">
  Verwenden Sie pipeline-navigator, wenn Sie verstehen möchten, welches
  Repository für eine bestimmte Konfigurationsdatei oder Komponente
  verantwortlich ist.
</Aside>

## Befehle

### /preview-docs

```
/f5xc-docs-pipeline:preview-docs
```

Startet einen lokalen Docker-Entwicklungsserver zur Vorschau der
Dokumentation. Führt das gemeinsam genutzte Docker-Build-Image aus,
wobei Ihr lokales `docs/`-Verzeichnis für die Live-Vorschau eingebunden wird.
