Das **osint-framework**-Plugin bietet einen OSINT-Werkzeugkatalog und
Ermittlungsfähigkeiten — 1.064 kostenlose Tools zur Informationsgewinnung
in 34 Kategorien, abgebildet von [osintframework.com](https://osintframework.com).
Es umfasst kategoriebasierte Skills, ausführbare Ermittlungs-Pipelines,
CLI-Tool-Ausführung, OPSEC-bewusste Workflows und einen persistenten
Entitätsgraphen für untersuchungsübergreifende Korrelation.

<Badge text="v1.0.1" variant="note" /> <Badge text="Security" variant="tip" />

## Installation

```
/plugin install osint-framework@f5xc-salesdemos-marketplace
```

## Befehle

### /osint-investigate

Führt eine vollständige OSINT-Ermittlung gegen ein beliebiges Ziel durch. Erkennt
automatisch den Zieltyp (E-Mail, Domain, IP, Benutzername, Unternehmen,
Personenname) und leitet an die entsprechenden Kategorie-Skills weiter.

```
/osint-investigate j.smith@example.com
/osint-investigate example.com
/osint-investigate 198.51.100.1
/osint-investigate example-user
```

### /osint-search

Durchsucht den Werkzeugkatalog nach Tools, die einer Fähigkeitsabfrage entsprechen.

```
/osint-search subdomain enumeration
/osint-search email breach check
/osint-search satellite imagery
```

### /osint-catalog

Durchsuchen Sie den vollständigen Werkzeugkatalog nach Kategorie. Ohne Argumente
werden alle 34 Kategorien mit Tool-Anzahlen angezeigt.

```
/osint-catalog
/osint-catalog domain
/osint-catalog threat-intel
```

## Skills

### osint-index

Top-Level-Intent-Router. Aktiviert sich automatisch, wenn Sie "Informationen finden
über", "ermitteln", "nachschlagen", "Hintergrundprüfung" sagen oder eine beliebige
Open-Source-Intelligence-Aufgabe beschreiben. Leitet basierend auf dem Zieltyp
an den richtigen Kategorie-Skill weiter.

| Ziel | Weiterleitung an |
|------|-----------------|
| Benutzername oder Handle | `username-recon` |
| E-Mail-Adresse | `email-recon` |
| Domainname | `domain-recon` |
| IP-Adresse | `ip-address-recon` |
| Personenname + Unternehmen | `people-search` + `business-records` |
| Unternehmen / Organisation | `business-records` + `domain-recon` |
| Malware-Hash | `malicious-file-analysis` + `threat-intelligence` |

### osint-catalog

Werkzeugkatalog-Browser. Bietet Zugriff auf die vollständige Tool-Registry,
den Kategorieindex, Ermittlungs-Pipelines, die Rate-Limit-Referenz
und die Dokumentation der Korrelations-Engine.

## Agenten

### osint-investigator

Autonomer Multi-Tool-Ermittlungs-Orchestrator. Führt vollständige
Ermittlungs-Workflows aus — Tool-Verfügbarkeitsprüfung, Zieltyp-Erkennung,
sequenzielle Tool-Ausführung (passiv zuerst), Ergebnis-Aggregation und
strukturierte Berichterstellung mit Konfidenzniveaus und OPSEC-Klassifizierungen.

Integriert sich mit dem Entitätsgraphen: Jedes Tool-Ergebnis erstellt Entitäten
und Beziehungen in `/tmp/osint-graph/` für untersuchungsübergreifende Korrelation.

<Aside type="caution">
  Aktive Recon-Tools (nmap, nuclei, sherlock) nehmen direkten Kontakt
  mit der Zielinfrastruktur auf. Der OPSEC-Warnungs-Hook warnt vor
  der Ausführung aktiver Tools. Stellen Sie immer sicher, dass Sie eine Autorisierung haben.
</Aside>

### osint-researcher

Schreibgeschützter Katalog-Lookup-Agent. Durchsucht Referenzdateien, um
die besten Tools für eine gegebene Aufgabe zu empfehlen. Führt niemals Tools aus —
empfiehlt und erklärt nur. Wird intern von Kategorie-Skills
für die Tool-Erkennung verwendet.

### osint-executor

Einzeltool-CLI-Ausführungsagent. Führt jeweils ein Tool mit
bestimmten Parametern aus, parst die Ausgabe und liefert strukturierte Ergebnisse.
Enthält Tool-spezifische Ausführungsrezepte für über 20 Tools mit exakten Flags,
Ausgabeformaten und jq-Parsing-Mustern.

## Ratenbegrenzung

Alle API-Aufrufe verwenden den `osint_curl`-Helper mit eingebautem exponentiellem
Backoff bei HTTP 429/503. Wichtige Limits:

| API | Kostenloses Limit | Umgebungsvariable |
|-----|-------------------|-------------------|
| ipinfo.io | 1.000/Tag | — |
| crt.sh | 60/Min | — |
| NVD | 5/30s (ohne Schlüssel) | `NVD_API_KEY` |
| OpenCorporates | Schlüssel erforderlich | `OPENCORPORATES_API_KEY` |
| Shodan | 1/Sek | `SHODAN_API_KEY` |
| GitHub API | 60/Std (ohne Auth) | `GH_TOKEN` |

APIs, die Schlüssel erfordern, werden automatisch mit einer klaren
Meldung übersprungen, wenn die Umgebungsvariable nicht gesetzt ist.

## Korrelations-Engine

Das Plugin enthält einen persistenten JSON-Entitätsgraphen unter
`/tmp/osint-graph/`, der Erkenntnisse über Ermittlungen hinweg akkumuliert.

```bash
source plugins/osint-framework/scripts/osint-graph.sh
osint_graph_init

# Entitäten hinzufügen
P=$(osint_entity_add "person" "J. Smith" --tool github-api)
C=$(osint_entity_add "company" "F5" --tool github-api)

# Verknüpfung erstellen
osint_rel_add "$P" "$C" "works_at" --tool github-api

# Abfrage: alle Entitäten innerhalb von 2 Hops von der Person
osint_graph_query "$P" 2

# Bericht generieren
osint_graph_report
```

**Konfidenzbewertung** verwendet Multi-Source-Fusion:
`confidence = 1 - product(1 - ci)` für jede bestätigende Quelle.
Zwei Quellen mit 0,80 → 0,96 kombinierte Konfidenz.
