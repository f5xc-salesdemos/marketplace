Das **f5xc-github-ops**-Plugin bietet exklusive GitHub-Operations-Automatisierung für f5-sales-demo-Repositories. Es verwaltet den gesamten Git- und GitHub-Lebenszyklus von der Issue-Erstellung bis zur Überwachung nach dem Merge und setzt Issue-getriebene Entwicklung sowie Pre-Commit-Lint-Gates durch.

<Badge text="v2.2.2" variant="note" /> <Badge text="Produktivität" variant="tip" />

## Installation

```
/plugin install f5xc-github-ops@f5-sales-demo-marketplace
```

## Skills

### workflow-lifecycle

Repository-Governance-Workflow zur Handhabung von Issue-Erstellung, Branch-Namenskonventionen, PR-Erstellung, CI-Polling mit Fehler-Feedback an Issues, Überwachung nach dem Merge, Verifizierung und Aufgabenabschlusskriterien. Wird aktiviert, wenn Sie "commit", "push", "erstelle einen PR", "merge" sagen oder eine Git-Operation durchführen.

### upstream-contribution

Upstream-Contribution-Workflow für Fork-basierte Entwicklung. Behandelt Fork-Erkennung, Upstream-Recherche (Issues, PRs, CONTRIBUTING.md, CI-Muster), repositoryübergreifende Issue- und PR-Erstellung, lokales Tracking-Issue-Management, Upstream-CI-Überwachung und geführte Problemlösung.

## Agenten

### github-ops

Exklusiver Agent für alle mechanischen Git- und GitHub-Operationen. Behandelt Pre-Commit-Lint-Gates, Issue-Erstellung, Branch-Erstellung, Staging, Commits, Pushes, PR-Erstellung, CI-Polling mit Fehler-Feedback, Wiederholung bei Infrastrukturfehlern, Überwachung nach dem Merge, Branch-Bereinigung und Repository-Einstellungsverwaltung über die GitHub-API.

Gibt strukturierte Statusberichte zurück: `COMPLETE`, `PRE_COMMIT_FAILED`, `CI_FAILED`, `BLOCKED` oder `FAILED`.

<Aside type="caution">
  Dieser Agent ist der **exklusive** Handler für Git-Operationen.
  Führen Sie niemals `git commit`, `git push` oder `gh pr create` direkt
  in Repositories aus, die dieses Plugin verwenden.
</Aside>
