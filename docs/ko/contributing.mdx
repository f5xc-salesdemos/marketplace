이 가이드는 새 플러그인을 만들고 f5xc-salesdemos 마켓플레이스에
추가하는 방법을 설명합니다.

## 플러그인 구조

모든 플러그인은 `plugins/` 아래의 자체 디렉토리에 위치하며
다음 구조를 따릅니다:

<FileTree>
- plugins/
  - my-plugin/
    - .claude-plugin/
      - plugin.json
    - skills/
      - my-skill/
        - SKILL.md
        - references/
          - reference-data.md
    - commands/
      - my-command.md
    - agents/
      - my-agent.md
    - README.md
</FileTree>

## 플러그인 만들기

<Steps>

1. **플러그인 디렉토리 생성**

   ```bash
   mkdir -p plugins/my-plugin/.claude-plugin
   mkdir -p plugins/my-plugin/skills
   mkdir -p plugins/my-plugin/commands
   ```

2. **`plugin.json` 작성**

   `plugins/my-plugin/.claude-plugin/plugin.json`을 생성합니다:

   ```json
   {
     "name": "my-plugin",
     "description": "What this plugin does",
     "version": "1.0.0",
     "author": {
       "name": "f5xc-salesdemos"
     },
     "homepage": "https://github.com/f5xc-salesdemos/marketplace/tree/main/plugins/my-plugin",
     "keywords": ["relevant", "keywords"],
     "license": "Apache-2.0",
     "repository": "https://github.com/f5xc-salesdemos/marketplace"
   }
   ```

3. **스킬 추가**

   프론트매터가 포함된 `plugins/my-plugin/skills/my-skill/SKILL.md`를
   생성합니다:

   ```markdown
   ---
   name: my-skill
   description: One-line description of what this skill does
   ---

   Detailed instructions for Claude when this skill activates.
   ```

   참조 데이터는 `SKILL.md` 옆에 있는 `references/` 하위 디렉토리에
   배치합니다.

4. **커맨드 추가**

   `plugins/my-plugin/commands/my-command.md`를 생성합니다:

   ```markdown
   ---
   description: What this command does
   argument_hint: "[optional-args]"
   allowed_tools:
     - Read
     - Glob
     - Grep
   ---

   Instructions for Claude when the user invokes this command.
   ```

5. **marketplace.json에 플러그인 추가**

   `.claude-plugin/marketplace.json`의 `plugins` 배열에
   항목을 추가합니다:

   ```json
   {
     "name": "my-plugin",
     "description": "What this plugin does",
     "version": "1.0.0",
     "author": { "name": "f5xc-salesdemos" },
     "source": "./plugins/my-plugin",
     "category": "productivity",
     "homepage": "https://github.com/f5xc-salesdemos/marketplace/tree/main/plugins/my-plugin",
     "license": "Apache-2.0",
     "keywords": ["relevant", "keywords"],
     "tags": ["searchable", "tags"],
     "repository": "https://github.com/f5xc-salesdemos/marketplace"
   }
   ```

6. **로컬에서 테스트**

   Claude Code를 로컬 디렉토리로 지정하여 게시 없이
   플러그인을 테스트합니다:

   ```bash
   claude --plugin-dir ./plugins/my-plugin
   ```

   스킬이 올바른 컨텍스트에서 활성화되는지, 커맨드가
   예상된 출력을 생성하는지 확인합니다.

</Steps>

## 플러그인 제출하기

<Aside type="tip" title="저장소 워크플로우">
  이 저장소는 엄격한 거버넌스 워크플로우를 적용합니다.
  전체 프로세스는 [CONTRIBUTING.md](https://github.com/f5xc-salesdemos/marketplace/blob/main/CONTRIBUTING.md)를
  참조하세요.
</Aside>

<Steps>

1. 새 플러그인을 설명하는 GitHub 이슈를 생성합니다
2. `main`에서 기능 브랜치를 생성합니다
3. 플러그인 디렉토리를 추가하고 `marketplace.json`을 업데이트합니다
4. `docs/plugins/my-plugin.mdx`에 문서 페이지를 추가합니다
5. `Closes #N`으로 이슈에 연결하는 PR을 엽니다
6. CI 실패를 수정하고 모든 검사가 통과한 후 병합합니다

</Steps>

## 모범 사례

- **스킬을 집중적으로 유지하세요** — 하나의 스킬은 하나의 작업을 잘 수행해야 합니다
- **참조 파일을 사용하세요** — 대규모 데이터셋, 스키마, 조회 테이블은 SKILL.md에 인라인으로 넣지 말고 `references/`에 배치하세요
- **지능적으로 범위를 설정하세요** — 가능한 경우 커밋되지 않은 변경 사항을 확인하여 작업 범위를 좁히세요
- **심각도별로 출력을 그룹화하세요** — 사용자가 수정 사항의 우선순위를 정할 수 있도록 ERROR, WARNING, INFO 수준을 사용하세요
- **README를 작성하세요** — 사용 예시와 각 스킬 및 커맨드가 수행하는 작업에 대한 설명을 포함하세요
