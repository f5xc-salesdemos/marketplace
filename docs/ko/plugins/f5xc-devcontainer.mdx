**f5xc-devcontainer** 플러그인은 f5xc-salesdemos 개발 환경을 위한
컨테이너 인식 기능을 제공합니다. 포괄적인 도구 카탈로그를 유지하고,
실시간 컨테이너 인트로스펙션을 지원하며, GitHub 이슈를 통한
지속성 추적으로 도구 설치를 처리합니다.

<Badge text="v1.1.6" variant="note" /> <Badge text="생산성" variant="tip" />

## 설치

```
/plugin install f5xc-devcontainer@f5xc-salesdemos-marketplace
```

## 스킬

### tool-catalog

데브컨테이너에 설치된 모든 CLI 도구의 지식 베이스로, 카테고리별로
정리되어 있습니다(클라우드, 네트워킹, 보안, 개발 등). 어떤 도구를
사용해야 하는지, 도구 사용 방법, 특정 항목의 설치 여부, 또는
특정 작업에 대한 도움이 필요할 때 활성화됩니다.

### self-awareness

컨테이너 신원 확인 및 자가 진단 스킬입니다. GitHub API와 로컬
빌드 메타데이터를 통해 실행 중인 컨테이너의 실시간 인트로스펙션을
제공합니다. 신원, 버전, 빌드 정보, 상태 점검, 기여자 히스토리
관련 질문 시 활성화됩니다.

<Aside type="tip">
  "너는 누구야?" 또는 "내가 실행 중인 버전이 뭐야?"라고 물어보면
  self-awareness가 트리거됩니다.
</Aside>

## 에이전트

### tool-advisor

도구 카탈로그를 읽고, CLI 도구 관련 질문에 대해 용도, 빠른 시작
명령어, 인증 요구 사항과 함께 추천 사항을 반환합니다.

### tool-auditor

데브컨테이너 Dockerfile을 도구 카탈로그 참조 파일과 비교하여
드리프트를 감지합니다 &mdash; 이미지에는 추가되었지만 카탈로그에
등록되지 않은 도구, 또는 제거된 도구에 대한 카탈로그 항목을
탐지합니다.

### container-introspector

GitHub API와 로컬 메타데이터를 통해 실시간 컨테이너 인트로스펙션을
수행하며, 신원, 계보, 자가 진단, 기여자 히스토리, 빌드 계통 정보를
제공합니다. 읽기 전용 작업만 수행합니다.

### container-maintainer

올바른 패키지 관리자 패턴을 사용하여 실행 중인 컨테이너에서
CLI 도구를 설치, 제거, 검색 및 업데이트합니다. 지속성 추적을 위해
데브컨테이너 리포지토리에 GitHub 이슈를 등록합니다.
