**f5xc-devcontainer** 플러그인은 f5-sales-demo 개발 환경에 컨테이너 인식 기능을 제공합니다. 포괄적인 도구 카탈로그를 유지 관리하고, 라이브 컨테이너 인트로스펙션을 가능하게 하며, GitHub 이슈를 통한 지속성 추적과 함께 도구 설치를 처리합니다.

<Badge text="v1.1.6" variant="note" /> <Badge text="Productivity" variant="tip" />

## 설치

```
/plugin install f5xc-devcontainer@f5-sales-demo-marketplace
```

## 스킬

### tool-catalog

클라우드, 네트워킹, 보안, 개발 등 카테고리별로 정리된 개발 컨테이너에 설치된 모든 CLI 도구의 지식 베이스입니다. 어떤 도구를 사용할지, 도구 사용 방법, 설치 여부 확인, 또는 특정 작업에 대한 도움이 필요할 때 활성화됩니다.

### self-awareness

컨테이너 식별 및 자가 진단 스킬입니다. GitHub API 및 로컬 빌드 메타데이터를 통해 실행 중인 컨테이너의 라이브 인트로스펙션을 제공합니다. 식별, 버전, 빌드 정보, 헬스 체크 및 기여자 히스토리 질문에 대해 활성화됩니다.

<Aside type="tip">
  "who are you?" 또는 "what version am I running?"을 물어보면 self-awareness가 트리거됩니다.
</Aside>

## 에이전트

### tool-advisor

도구 카탈로그를 읽고 모든 CLI 도구 질문에 대해 목적, 빠른 시작 명령어, 인증 요구 사항을 포함한 권장 사항을 반환합니다.

### tool-auditor

도구 카탈로그 참조 파일에 대해 개발 컨테이너 Dockerfile을 감사하여 드리프트를 감지합니다 &mdash; 이미지에 추가되었지만 카탈로그에 등록되지 않은 도구, 또는 제거된 도구에 대한 카탈로그 항목을 식별합니다.

### container-introspector

식별, 계보, 자가 진단, 기여자 히스토리 및 빌드 계통을 위해 GitHub API 및 로컬 메타데이터를 통해 라이브 컨테이너 인트로스펙션을 수행합니다. 읽기 전용 작업만 수행합니다.

### container-maintainer

올바른 패키지 매니저 패턴을 사용하여 실행 중인 컨테이너에서 CLI 도구를 설치, 제거, 검색 및 업데이트합니다. 지속성 추적을 위해 개발 컨테이너 저장소에 GitHub 이슈를 등록합니다.
