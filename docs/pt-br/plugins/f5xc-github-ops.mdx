O plugin **f5xc-github-ops** fornece automação exclusiva de operações
GitHub para repositórios f5xc-salesdemos. Ele gerencia todo o ciclo
de vida Git e GitHub, desde a criação de issues até o monitoramento
pós-merge, aplicando desenvolvimento orientado a issues e gates de
lint pré-commit.

<Badge text="v2.2.2" variant="note" /> <Badge text="Produtividade" variant="tip" />

## Instalação

```
/plugin install f5xc-github-ops@f5xc-salesdemos-marketplace
```

## Habilidades

### workflow-lifecycle

Fluxo de trabalho de governança de repositório que lida com criação
de issues, convenções de nomenclatura de branches, criação de PRs,
polling de CI com feedback de erros para issues, monitoramento
pós-merge, verificação e critérios de conclusão de tarefas. Ativa
quando você diz "commit", "push", "criar um PR", "merge" ou
realiza qualquer operação Git.

### upstream-contribution

Fluxo de trabalho de contribuição upstream para desenvolvimento
baseado em fork. Gerencia detecção de fork, pesquisa upstream
(issues, PRs, CONTRIBUTING.md, padrões de CI), criação de issues
e PRs cross-repo, gerenciamento de issues de rastreamento local,
monitoramento de CI upstream e resolução guiada.

## Agentes

### github-ops

Agente exclusivo para todas as operações mecânicas de Git e GitHub.
Gerencia gates de lint pré-commit, criação de issues, criação de
branches, staging, commits, push, criação de PRs, polling de CI
com feedback de erros, retry de falhas de infraestrutura,
monitoramento pós-merge, limpeza de branches e gerenciamento de
configurações de repositório via API do GitHub.

Retorna relatórios de status estruturados: `COMPLETE`,
`PRE_COMMIT_FAILED`, `CI_FAILED`, `BLOCKED` ou `FAILED`.

<Aside type="caution">
  Este agente é o handler **exclusivo** para operações Git.
  Nunca execute `git commit`, `git push` ou `gh pr create` diretamente
  em repositórios que utilizam este plugin.
</Aside>
