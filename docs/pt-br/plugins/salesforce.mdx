O plugin **salesforce** integra o Salesforce CLI com o Claude Code,
fornecendo autenticação de org e um agente CLI de propósito geral que pode
executar qualquer comando `sf`, incluindo consultas SOQL. Ele suporta
ambientes de contêiner headless com fluxos de autenticação JWT, access-token
e SFDX URL. O guia de uso abaixo demonstra como usar linguagem natural
para consultar contas, oportunidades, casos e contatos através do agente CLI.

<Badge text="v1.0.0" variant="note" /> <Badge text="Development" variant="tip" />

## Instalação

```
/plugin install salesforce@f5xc-salesdemos-marketplace
```

## Pré-requisitos

- **Salesforce CLI** (`@salesforce/cli`)
- **Org Salesforce** com acesso à API
- Uma conta de usuário com permissões para consultar objetos (Cases, Opportunities, Accounts, Contacts)

Instale o Salesforce CLI na sua estação de trabalho:

```bash
brew install sf
```

Verifique a instalação:

```bash
sf --version
```

Você deve ver uma saída como `@salesforce/cli/2.x.x`.

## Configuração de Autenticação

### Passo 1: Encontre seu domínio Salesforce

Observe a URL do Salesforce no navegador. Se você acessa o Salesforce em
`https://acme.lightning.force.com`, seu domínio de login é
`https://acme.my.salesforce.com`.

<Aside type="tip">
  Se sua empresa usa SSO, você deve especificar seu domínio personalizado com
  `--instance-url` para que o login redirecione através do seu provedor SSO
  em vez da página de login genérica do Salesforce.
</Aside>

### Passo 2: Autentique via navegador

```bash
sf org login web --alias my-org --set-default --instance-url https://YOUR-DOMAIN.my.salesforce.com
```

Substitua `YOUR-DOMAIN` pelo domínio da sua empresa do Passo 1. Seu navegador
será aberto — complete o fluxo de SSO ou login e autorize o aplicativo. Você
deve ver:

```
Successfully authorized your-email@company.com with org ID 00DXXXXXXXXXXXXXXX
```

### Passo 3: Verifique a conexão

```bash
sf org display --target-org my-org
```

Confirme que a saída mostra `Connected Status: Connected` com seu nome de
usuário e URL da instância.

### Passo 4: Autenticação portátil para contêineres (opcional)

Se você precisa autenticar em um ambiente headless (CI/CD, contêiner,
sessão remota), exporte a URL de autenticação SFDX da sua estação de trabalho:

```bash
sf org display --verbose --target-org my-org
```

Copie o valor da **Sfdx Auth Url** (começa com `force://`). No
contêiner, execute:

```bash
export SFDX_AUTH_URL="force://PlatformCLI::YOUR_AUTH_TOKEN@your-domain.my.salesforce.com"
echo "$SFDX_AUTH_URL" | sf org login sfdx-url --sfdx-url-stdin=- --alias=my-org --set-default
```

<Aside type="caution">
  A URL de autenticação SFDX é uma credencial — trate-a como uma senha. Não
  a salve em arquivos nem cole em chats. Use variáveis de ambiente
  e pipes stdin apenas.
</Aside>

### Referência de métodos de autenticação

| Método | Melhor Para | Requer |
|--------|-------------|--------|
| Web Login | Estações de trabalho com navegador | Navegador + SSO |
| SFDX URL | Contêineres, CI/CD, autenticação portátil | URL de autenticação de uma sessão autenticada |
| JWT Bearer | Pipelines automatizados | Connected App + chave privada + consumer key |
| Access Token | Autenticação via variável de ambiente | `SF_ACCESS_TOKEN` + `SF_ORG_INSTANCE_URL` |

**Nota:** O fluxo de dispositivo (`sf org login device`) está bloqueado desde agosto
de 2025 e não é suportado.

### Variáveis de ambiente

| Variável | Finalidade |
|----------|------------|
| `SF_ACCESS_TOKEN` | Token bearer para autenticação por access-token |
| `SFDX_AUTH_URL` | URL de autenticação force para autenticação SFDX URL |
| `SF_ORG_INSTANCE_URL` | URL da instância da org |
| `SF_JWT_KEY_FILE` | Caminho para a chave privada JWT |
| `SF_CLIENT_ID` | Consumer key do Connected App |
| `SF_USERNAME` | Nome de usuário Salesforce para JWT |

## Comandos

### /salesforce:sf-status

Verifique o status de conexão da sua org Salesforce, versão do CLI e
orgs autenticadas.

```
/salesforce:sf-status
```

**O que esperar:** Uma tabela mostrando a versão do seu sf CLI, alias da org
autenticada, nome de usuário, URL da instância, status de conexão e versão da API.

### /salesforce:sf-login

Autentique-se em uma org Salesforce. Opcionalmente forneça um alias.

```
/salesforce:sf-login
/salesforce:sf-login my-org
```

**O que esperar:** Se uma org com esse alias já estiver autenticada,
ele reporta os detalhes da conexão. Caso contrário, verifica as credenciais
disponíveis (variáveis de ambiente) e autentica usando o primeiro
método totalmente satisfeito.

Skill de autenticação adaptada para contêineres. Suporta quatro métodos de

Prioriza: access-token, depois JWT, depois SFDX URL, depois web login.

## Agentes

### cli-operator

Agente autônomo do Salesforce CLI que executa comandos `sf` com
proteções de segurança. Todas as operações Salesforce são delegadas a este
agente para manter o contexto da sessão principal enxuto.

**Regras de segurança:**

- Somente leitura por padrão — operações de escrita requerem aprovação explícita
- Nunca realiza deploy sem uma pré-visualização e etapa de confirmação
- Nunca exibe access tokens, URLs de autenticação ou credenciais na saída
- Sanitiza todos os valores fornecidos pelo usuário (rejeita metacaracteres de shell)
- Usa saída `--json` para resultados estruturados e parseáveis

**Formato de resposta:**

```
## Result: [SUCCESS | FAILURE | PARTIAL]
### Command Executed
### Output Summary
### Issues
```

## Guia de Uso: Gerenciamento de Contas

<Aside type="note">
  Os prompts abaixo não são funcionalidades codificadas do plugin. Eles funcionam
  porque o agente cli-operator pode executar qualquer comando `sf` do CLI, incluindo
  consultas SOQL arbitrárias. Os resultados dependem do modelo de dados da sua org e
  das suas permissões de usuário. Os prompts aqui foram testados e verificados
  para funcionar com objetos padrão do Salesforce (Account, Contact, Case,
  Opportunity, AccountTeamMember, OpportunityTeamMember).
</Aside>

Estes prompts funcionam com qualquer org Salesforce. Substitua os valores
placeholder pelas suas próprias informações. Cada prompt produz resultados
específicos para sua função, contas e território.

<Aside type="tip">
  Para melhores resultados, inclua seu endereço de e-mail nos prompts que precisam
  identificar seu registro de usuário. Orgs Salesforce podem ter múltiplos usuários
  com nomes semelhantes.
</Aside>

### Primeiros passos

Após autenticar, verifique se tudo funciona:

```
/salesforce:sf-status
```

Depois tente uma consulta simples em linguagem natural:

```
list my authenticated salesforce orgs
```

**O que esperar:** Uma tabela mostrando todas as orgs conectadas com alias,
nome de usuário, URL da instância e status de conexão.

### Encontre suas contas

Descubra a quais contas você está atribuído:

```
what salesforce accounts am I on the account team for? My email is your-email@company.com
```

**O que esperar:** Uma lista de contas agrupadas por proprietário da conta, mostrando
sua função de membro da equipe em cada uma. Se a maioria das funções aparecer vazia,
isso é uma lacuna comum de higiene de dados — as associações à equipe da conta existem,
mas as funções não foram preenchidas.

### Referência cruzada de cobertura com um colega

Compare sua cobertura de contas com um colega de equipe para encontrar lacunas:

```
In salesforce, find all accounts where Colleague Name is on the account team. Then for each of those accounts, check if your-email@company.com is also on the account team. Show me two lists: accounts where we are BOTH tagged, and accounts where only my colleague is tagged but I am missing.
```

**O que esperar:** Uma tabela resumo mostrando contagem de sobreposição e contagem de
lacunas, seguida de duas listas. Se você mudou de função recentemente, pode encontrar
sobreposição zero — isso revela quais contas você precisa ser adicionado.

<Aside type="note">
  Você também pode verificar um predecessor adicionando: "Also check if
  Previous Person Name appears on any of those accounts." Isso ajuda a
  identificar lacunas de handoff ao substituir alguém em um território.
</Aside>

### Visão geral do pipeline do território

Veja todas as oportunidades abertas nas suas contas:

```
show me all open salesforce opportunities on Colleague Name's account team accounts, sorted by close date, include the stage, amount, and probability
```

**O que esperar:** Um resumo com contagem total de oportunidades, pipeline bruto
e pipeline ponderado. Depois uma tabela de distribuição por estágio e uma lista das
principais oportunidades por valor. A saída sinaliza problemas de higiene de dados como
datas de fechamento vencidas ou valores nulos.

### Análise detalhada de oportunidade

Obtenha a visão completa de um negócio específico:

```
show me a detailed view of the OPPORTUNITY NAME opportunity in salesforce - include the opportunity team members, any activities or tasks, and the account contacts
```

**O que esperar:** A visão geral da oportunidade (estágio, valor, data de fechamento,
probabilidade, categoria de previsão, proprietário), membros da equipe da oportunidade
com funções, contatos vinculados, tarefas e eventos recentes, e histórico de alterações
de campo mostrando como o valor do negócio e a data de fechamento mudaram ao longo do tempo.

### Casos de suporte no território

Verifique os casos de suporte abertos nas suas contas:

```
show me all open salesforce cases across Colleague Name's account team accounts, grouped by account, sorted by most recent first
```

**O que esperar:** Uma contagem de casos abertos com detalhamento por prioridade e status,
depois detalhes dos casos agrupados por conta. Casos obsoletos (meses ou anos sem
atividade) são sinalizados como candidatos a higiene.

### Previsão de pipeline trimestral

Obtenha uma visão pronta para previsão do seu pipeline:

```
for Colleague Name's accounts, show me a quarterly pipeline summary - group the open opportunities by close date quarter with count, total amount, and weighted amount for each quarter
```

**O que esperar:** Uma tabela com uma linha por trimestre mostrando contagem de
oportunidades, pipeline total e pipeline ponderado. Inclui um detalhamento do mix de
estágios por trimestre e destaca os principais negócios ponderados. Oportunidades
vencidas são agrupadas separadamente.

### Visão geral completa da conta

Análise detalhada de um cliente específico:

```
give me a full account overview for ACCOUNT NAME in salesforce including contacts, open opportunities, and recent cases
```

**O que esperar:** Perfil da empresa (setor, receita, funcionários,
localização), contatos principais com cargos e e-mail, oportunidades abertas com
estágios e valores, e casos de suporte recentes com status.

### Consulta de caso

Consulte um caso de suporte específico por número:

```
look up salesforce case CASE-NUMBER and show me the case details, the customer account, and who owns it
```

**O que esperar:** Assunto do caso, status, prioridade, descrição, o
perfil da conta do cliente e o proprietário do caso com informações de contato.

## Guia de Uso: Consultas de Dados

Para usuários avançados, você pode solicitar consultas específicas no estilo SOQL em
linguagem natural. O plugin traduz sua solicitação na SOQL apropriada
e a executa.

### Campos específicos

```
query salesforce for all Contacts at ACCOUNT NAME - show Name, Title, Email, Phone, and Department
```

### Filtragem por data

```
show me all salesforce opportunities that closed won in the last 90 days on ACCOUNT NAME
```

### Agregação

```
count all open salesforce cases grouped by priority and status across my accounts
```

### Objetos personalizados

```
query the CUSTOM_OBJECT__c object in salesforce for records where Status__c = 'Active'
```

<Aside type="tip">
  Se uma consulta falhar porque um campo não existe, o plugin
  automaticamente tenta novamente sem aquele campo. Campos personalizados na sua org
  podem usar nomes diferentes — verifique o Setup do Salesforce se um campo
  não for encontrado.
</Aside>
