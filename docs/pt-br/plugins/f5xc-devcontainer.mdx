O plugin **f5xc-devcontainer** fornece consciência de contêiner para
o ambiente de desenvolvimento f5-sales-demo. Ele mantém um
catálogo abrangente de ferramentas, habilita introspecção ao vivo do contêiner
e gerencia a instalação de ferramentas com rastreamento de persistência via
issues do GitHub.

<Badge text="v1.1.6" variant="note" /> <Badge text="Productivity" variant="tip" />

## Instalação

```
/plugin install f5xc-devcontainer@f5-sales-demo-marketplace
```

## Skills

### tool-catalog

Base de conhecimento de todas as ferramentas CLI instaladas no devcontainer,
organizadas por categoria (cloud, networking, segurança, desenvolvimento,
etc.). Ativa quando você pergunta qual ferramenta usar, como usar
uma ferramenta, se algo está instalado, ou quando precisa de ajuda com uma
operação específica.

### self-awareness

Skill de identidade e autodiagnóstico do contêiner. Fornece introspecção
ao vivo do contêiner em execução via API do GitHub e metadados locais
de build. Ativa para perguntas sobre identidade, versão, informações de build,
verificações de integridade e histórico de colaboradores.

<Aside type="tip">
  Pergunte "quem é você?" ou "qual versão estou executando?" para acionar
  o self-awareness.
</Aside>

## Agents

### tool-advisor

Lê o catálogo de ferramentas e retorna recomendações com finalidade,
comandos de início rápido e requisitos de autenticação para qualquer
pergunta sobre ferramentas CLI.

### tool-auditor

Audita o Dockerfile do devcontainer em relação aos arquivos de referência
do catálogo de ferramentas para detectar desvios &mdash; ferramentas adicionadas à imagem
mas não catalogadas, ou entradas do catálogo para ferramentas que foram
removidas.

### container-introspector

Realiza introspecção ao vivo do contêiner via API do GitHub e metadados
locais para identidade, genealogia, autodiagnóstico, histórico de colaboradores
e linhagem de build. Somente operações de leitura.

### container-maintainer

Instala, remove, pesquisa e atualiza ferramentas CLI no contêiner
em execução usando padrões corretos de gerenciador de pacotes. Cria issues no GitHub
no repositório do devcontainer para rastreamento de persistência.
