يشرح هذا الدليل كيفية إنشاء إضافة جديدة وإضافتها إلى
سوق f5xc-salesdemos.

## هيكل الإضافة

تتواجد كل إضافة في مجلدها الخاص ضمن `plugins/` وتتبع
هذا الهيكل:

<FileTree>
- plugins/
  - my-plugin/
    - .claude-plugin/
      - plugin.json
    - skills/
      - my-skill/
        - SKILL.md
        - references/
          - reference-data.md
    - commands/
      - my-command.md
    - agents/
      - my-agent.md
    - README.md
</FileTree>

## إنشاء إضافة

<Steps>

1. **إنشاء مجلد الإضافة**

   ```bash
   mkdir -p plugins/my-plugin/.claude-plugin
   mkdir -p plugins/my-plugin/skills
   mkdir -p plugins/my-plugin/commands
   ```

2. **كتابة `plugin.json`**

   أنشئ الملف `plugins/my-plugin/.claude-plugin/plugin.json`:

   ```json
   {
     "name": "my-plugin",
     "description": "What this plugin does",
     "version": "1.0.0",
     "author": {
       "name": "f5xc-salesdemos"
     },
     "homepage": "https://github.com/f5xc-salesdemos/marketplace/tree/main/plugins/my-plugin",
     "keywords": ["relevant", "keywords"],
     "license": "Apache-2.0",
     "repository": "https://github.com/f5xc-salesdemos/marketplace"
   }
   ```

3. **إضافة المهارات**

   أنشئ الملف `plugins/my-plugin/skills/my-skill/SKILL.md` مع
   البيانات الوصفية الأمامية:

   ```markdown
   ---
   name: my-skill
   description: One-line description of what this skill does
   ---

   Detailed instructions for Claude when this skill activates.
   ```

   ضع أي بيانات مرجعية في مجلد فرعي `references/`
   بجانب `SKILL.md`.

4. **إضافة الأوامر**

   أنشئ الملف `plugins/my-plugin/commands/my-command.md`:

   ```markdown
   ---
   description: What this command does
   argument_hint: "[optional-args]"
   allowed_tools:
     - Read
     - Glob
     - Grep
   ---

   Instructions for Claude when the user invokes this command.
   ```

5. **إضافة الإضافة إلى marketplace.json**

   أضف إدخالاً إلى مصفوفة `plugins` في
   `.claude-plugin/marketplace.json`:

   ```json
   {
     "name": "my-plugin",
     "description": "What this plugin does",
     "version": "1.0.0",
     "author": { "name": "f5xc-salesdemos" },
     "source": "./plugins/my-plugin",
     "category": "productivity",
     "homepage": "https://github.com/f5xc-salesdemos/marketplace/tree/main/plugins/my-plugin",
     "license": "Apache-2.0",
     "keywords": ["relevant", "keywords"],
     "tags": ["searchable", "tags"],
     "repository": "https://github.com/f5xc-salesdemos/marketplace"
   }
   ```

6. **الاختبار محلياً**

   اختبر إضافتك دون نشرها عن طريق توجيه Claude Code
   إلى المجلد المحلي:

   ```bash
   claude --plugin-dir ./plugins/my-plugin
   ```

   تحقق من أن المهارات تُفعَّل في السياق الصحيح وأن
   الأوامر تُنتج المخرجات المتوقعة.

</Steps>

## تقديم إضافتك

<Aside type="tip" title="سير عمل المستودع">
  يفرض هذا المستودع سير عمل حوكمة صارم.
  راجع [CONTRIBUTING.md](https://github.com/f5xc-salesdemos/marketplace/blob/main/CONTRIBUTING.md)
  للاطلاع على العملية الكاملة.
</Aside>

<Steps>

1. أنشئ مشكلة (issue) على GitHub تصف الإضافة الجديدة
2. أنشئ فرعاً للميزة من `main`
3. أضف مجلد إضافتك وحدّث `marketplace.json`
4. أضف صفحة توثيق في `docs/plugins/my-plugin.mdx`
5. افتح طلب سحب (PR) مرتبطاً بالمشكلة مع `Closes #N`
6. أصلح أي إخفاقات في CI وادمج بعد نجاح جميع الفحوصات

</Steps>

## أفضل الممارسات

- **اجعل المهارات مركّزة** — يجب أن تؤدي كل مهارة شيئاً واحداً بشكل جيد
- **استخدم الملفات المرجعية** — ضع مجموعات البيانات الكبيرة والمخططات
  وجداول البحث في `references/` بدلاً من تضمينها في SKILL.md
- **حدد النطاق بذكاء** — تحقق من التغييرات غير المُودعة
  لتضييق نطاق العمل عند الإمكان
- **جمّع المخرجات حسب الخطورة** — استخدم مستويات ERROR وWARNING وINFO
  حتى يتمكن المستخدمون من ترتيب الإصلاحات حسب الأولوية
- **اكتب ملف README** — أضف أمثلة استخدام ووصفاً
  لما تفعله كل مهارة وكل أمر
