## مخطط marketplace.json

يوجد ملف بيان السوق في
`.claude-plugin/marketplace.json` في جذر المستودع.

### الحقول عالية المستوى

| الحقل | مطلوب | الوصف |
|-------|----------|-------------|
| `$schema` | لا | عنوان URL لمخطط JSON للتحقق |
| `name` | نعم | معرّف فريد للسوق |
| `version` | لا | إصدار مخطط السوق (semver) |
| `metadata.description` | نعم | وصف السوق القابل للقراءة البشرية |
| `owner.name` | نعم | المنظمة أو المستخدم المالك للسوق |
| `owner.url` | لا | عنوان URL لملف المالك الشخصي أو موقعه الإلكتروني |
| `owner.email` | لا | البريد الإلكتروني للتواصل |
| `plugins` | نعم | مصفوفة من إدخالات الإضافات |

### حقول إدخال الإضافة

| الحقل | مطلوب | الوصف |
|-------|----------|-------------|
| `name` | نعم | معرّف الإضافة (يجب أن يتطابق مع `plugin.json`) |
| `description` | نعم | وصف مختصر للإضافة |
| `version` | نعم | سلسلة الإصدار الدلالي |
| `author.name` | نعم | مؤلف الإضافة |
| `source` | نعم | المسار أو عنوان URL للإضافة (انظر أنواع المصادر) |
| `category` | نعم | فئة الإضافة (مثل `productivity`) |
| `homepage` | لا | عنوان URL لوثائق الإضافة أو صفحتها الرئيسية |
| `license` | لا | معرّف ترخيص SPDX |
| `keywords` | لا | مصفوفة من كلمات البحث المفتاحية |
| `tags` | لا | مصفوفة من العلامات للتصفية |
| `repository` | لا | عنوان URL للمستودع |

## أنواع مصادر الإضافات

يدعم حقل `source` في marketplace.json صيغاً متعددة:

| النوع | مثال | الوصف |
|------|---------|-------------|
| مسار نسبي | `./plugins/my-plugin` | إضافة في نفس المستودع |
| اختصار GitHub | `owner/repo` | إضافة في جذر المستودع |
| GitHub مع مسار | `owner/repo/path/to/plugin` | إضافة في دليل فرعي |
| عنوان URL لـ Git | `https://github.com/owner/repo.git` | مستودع Git |
| حزمة npm | `npm:@scope/package` | حزمة npm منشورة |

## مخطط plugin.json

كل إضافة لها ملف بيان في
`.claude-plugin/plugin.json` داخل دليلها.

| الحقل | مطلوب | الوصف |
|-------|----------|-------------|
| `name` | نعم | معرّف الإضافة |
| `description` | نعم | ما تفعله الإضافة |
| `version` | نعم | الإصدار الدلالي |
| `author.name` | نعم | مؤلف الإضافة |
| `author.url` | لا | عنوان URL لملف المؤلف الشخصي أو موقعه الإلكتروني |
| `homepage` | لا | عنوان URL للوثائق |
| `keywords` | لا | كلمات البحث المفتاحية |
| `license` | لا | معرّف ترخيص SPDX |
| `repository` | لا | عنوان URL للمستودع |

## هيكل دليل الإضافة

```
plugins/my-plugin/
  .claude-plugin/
    plugin.json          # Plugin manifest (required)
  skills/
    skill-name/
      SKILL.md           # Skill definition (required per skill)
      references/        # Reference data files (optional)
  commands/
    command-name.md      # Command definition (one per command)
  agents/
    agent-name.md        # Agent definition (one per agent)
  README.md              # Plugin documentation (recommended)
```

## متغيرات البيئة

يمكن للإضافات استخدام هذه المتغيرات في ملفات المهارات والأوامر الخاصة بها:

| المتغير | الوصف |
|----------|-------------|
| `$\{CLAUDE_PLUGIN_ROOT\}` | المسار المطلق لدليل جذر الإضافة |

## التحقق

تحقق من بيانات السوق والإضافات محلياً:

```bash
claude plugin validate .
```

يتحقق هذا من:

- امتثال `marketplace.json` للمخطط
- وجود جميع أدلة الإضافات المشار إليها
- أن كل إضافة تحتوي على `plugin.json` صالح
- أن المهارات تحتوي على بيانات أولية منسقة بشكل صحيح في `SKILL.md`
- أن الأوامر تحتوي على حقول البيانات الأولية المطلوبة

## تكوين الفريق

### تكوين الأسواق مسبقاً

أضف إلى `.claude/settings.json` في أي مستودع:

```json
{
  "extraKnownMarketplaces": [
    "f5xc-salesdemos/marketplace"
  ]
}
```

### تفعيل الإضافات مسبقاً

```json
{
  "enabledPlugins": [
    "f5xc-docs-tools@f5xc-salesdemos-marketplace"
  ]
}
```

<Aside type="tip">
  إيداع هذه الإعدادات في مستودعك يضمن حصول كل عضو في
  الفريق على نفس تكوين السوق والإضافات تلقائياً.
</Aside>

## استكشاف الأخطاء وإصلاحها

### السوق غير موجود

تحقق من أن السوق تمت إضافته:

```
/plugin marketplace list
```

إذا كان مفقوداً، أعد إضافته:

```
/plugin marketplace add f5xc-salesdemos/marketplace
```

### الإضافة لا تعمل

- تأكد من أن الإضافة مثبتة:
  `/plugin list`
- تحقق من أن إصدار Claude Code الخاص بك يدعم الإضافات
  (v1.0.33+)
- تحقق من أن شروط تفعيل المهارة تتطابق مع سياقك الحالي

### المهارات لا تعمل تلقائياً

تعمل المهارات بناءً على حقل `description` في البيانات الأولية لـ SKILL.md. إذا لم تعمل المهارة عند المتوقع:

- تحقق من أن الوصف يصف بدقة سياق التفعيل
- استخدم الأمر مباشرة (مثل `/f5xc-docs-tools:review-mdx`)
  كحل بديل

### أخطاء البناء بعد المراجعة

تُبلّغ الإضافة عن المشكلات لكنها لا تصلحها تلقائياً. طبّق
الإصلاحات المقترحة يدوياً، ثم أعد تشغيل المراجعة لتأكيد الحل.
