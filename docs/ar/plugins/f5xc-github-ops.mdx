يوفر ملحق **f5xc-github-ops** أتمتة حصرية لعمليات GitHub
لمستودعات f5-sales-demo. يتولى دورة حياة Git وGitHub الكاملة
بدءًا من إنشاء القضايا وحتى المراقبة بعد الدمج، مع فرض التطوير
المبني على القضايا وبوابات فحص الشفرة قبل الإيداع.

<Badge text="v2.2.2" variant="note" /> <Badge text="الإنتاجية" variant="tip" />

## التثبيت

```
/plugin install f5xc-github-ops@f5-sales-demo-marketplace
```

## المهارات

### workflow-lifecycle

سير عمل حوكمة المستودع الذي يتعامل مع إنشاء القضايا، واصطلاحات
تسمية الفروع، وإنشاء طلبات السحب، واستطلاع التكامل المستمر مع
تغذية راجعة للأخطاء على القضايا، والمراقبة بعد الدمج، والتحقق،
ومعايير إكمال المهام. يتم تفعيله عندما تقول "commit" أو "push"
أو "create a PR" أو "merge" أو تنفذ أي عملية Git.

### upstream-contribution

سير عمل المساهمة في المستودع الأصلي للتطوير القائم على النسخ
المتفرعة. يتعامل مع اكتشاف النسخ المتفرعة، والبحث في المستودع
الأصلي (القضايا، وطلبات السحب، وCONTRIBUTING.md، وأنماط التكامل
المستمر)، وإنشاء القضايا وطلبات السحب عبر المستودعات، وإدارة
قضايا التتبع المحلية، ومراقبة التكامل المستمر للمستودع الأصلي،
والحل الموجّه.

## الوكلاء

### github-ops

وكيل حصري لجميع عمليات Git وGitHub الميكانيكية. يتعامل مع
بوابات فحص الشفرة قبل الإيداع، وإنشاء القضايا، وإنشاء الفروع،
والتجهيز، والإيداع، والدفع، وإنشاء طلبات السحب، واستطلاع
التكامل المستمر مع تغذية راجعة للأخطاء، وإعادة محاولة فشل
البنية التحتية، والمراقبة بعد الدمج، وتنظيف الفروع، وإدارة
إعدادات المستودع عبر واجهة GitHub API.

يُرجع تقارير حالة منظمة: `COMPLETE`،
`PRE_COMMIT_FAILED`، `CI_FAILED`، `BLOCKED`، أو `FAILED`.

<Aside type="caution">
  هذا الوكيل هو المعالج **الحصري** لعمليات Git.
  لا تقم أبدًا بتشغيل `git commit` أو `git push` أو `gh pr create` مباشرةً
  في المستودعات التي تستخدم هذا الملحق.
</Aside>
