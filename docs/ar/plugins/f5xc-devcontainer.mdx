يوفر المكوّن الإضافي **f5xc-devcontainer** إدراكاً للحاوية في
بيئة تطوير f5-sales-demo. يحافظ على كتالوج شامل للأدوات، ويتيح
الفحص الفوري للحاوية، ويتولى تثبيت الأدوات مع تتبع الاستمرارية
عبر مشكلات GitHub.

<Badge text="v1.1.6" variant="note" /> <Badge text="Productivity" variant="tip" />

## التثبيت

```
/plugin install f5xc-devcontainer@f5-sales-demo-marketplace
```

## المهارات

### tool-catalog

قاعدة معرفة بكل أداة CLI مثبتة في حاوية التطوير،
منظمة حسب الفئة (السحابة، الشبكات، الأمان، التطوير،
إلخ). تُنشَّط عند سؤالك عن الأداة المناسبة للاستخدام، أو كيفية استخدام
أداة، أو ما إذا كان شيء ما مثبتاً، أو عند الحاجة إلى المساعدة في
عملية معينة.

### self-awareness

مهارة هوية الحاوية والتشخيص الذاتي. توفر فحصاً فورياً
للحاوية قيد التشغيل عبر GitHub API وبيانات التعريف للبناء المحلي.
تُنشَّط لأسئلة الهوية والإصدار ومعلومات البناء وفحوصات الصحة
وسجل المساهمين.

<Aside type="tip">
  اسأل "من أنت؟" أو "ما الإصدار الذي أشغّله؟" لتشغيل
  مهارة self-awareness.
</Aside>

## الوكلاء

### tool-advisor

يقرأ كتالوج الأدوات ويُعيد توصيات تتضمن الغرض،
وأوامر البدء السريع، ومتطلبات المصادقة لأي سؤال يتعلق بأدوات CLI.

### tool-auditor

يراجع ملف Dockerfile الخاص بحاوية التطوير مقابل ملفات مرجع كتالوج الأدوات
للكشف عن الانحراف &mdash; الأدوات المضافة إلى الصورة
دون إدراجها في الكتالوج، أو الإدخالات في الكتالوج لأدوات تمت إزالتها.

### container-introspector

يُجري فحصاً فورياً للحاوية عبر GitHub API والبيانات المحلية
للهوية والنسب والتشخيص الذاتي وسجل المساهمين وسلسلة البناء.
عمليات القراءة فقط.

### container-maintainer

يثبّت ويزيل ويبحث ويحدّث أدوات CLI في الحاوية قيد التشغيل
باستخدام أنماط مدير الحزم الصحيحة. يرفع مشكلات GitHub على مستودع حاوية التطوير
لتتبع الاستمرارية.
