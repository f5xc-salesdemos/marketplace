توفر إضافة **f5xc-devcontainer** التعرف على الحاوية لبيئة تطوير
f5xc-salesdemos. تحتفظ بكتالوج شامل للأدوات، وتمكّن من الاستبطان
المباشر للحاوية، وتتولى تثبيت الأدوات مع تتبع الاستمرارية عبر
مشكلات GitHub.

<Badge text="v1.1.6" variant="note" /> <Badge text="إنتاجية" variant="tip" />

## التثبيت

```
/plugin install f5xc-devcontainer@f5xc-salesdemos-marketplace
```

## المهارات

### tool-catalog

قاعدة معرفية لكل أداة سطر أوامر مثبتة في حاوية التطوير، منظمة حسب
الفئة (السحابة، الشبكات، الأمان، التطوير، إلخ). تنشط عندما تسأل عن
الأداة التي يجب استخدامها، أو كيفية استخدام أداة، أو ما إذا كان شيء
ما مثبتاً، أو تحتاج مساعدة في عملية معينة.

### self-awareness

مهارة هوية الحاوية والتشخيص الذاتي. توفر استبطاناً مباشراً للحاوية
قيد التشغيل عبر واجهة برمجة تطبيقات GitHub وبيانات البناء المحلية
الوصفية. تنشط لأسئلة الهوية، والإصدار، ومعلومات البناء، وفحوصات
السلامة، وسجل المساهمين.

<Aside type="tip">
  اسأل "من أنت؟" أو "ما الإصدار الذي أعمل عليه؟" لتفعيل
  التعرف الذاتي.
</Aside>

## الوكلاء

### tool-advisor

يقرأ كتالوج الأدوات ويعيد توصيات مع الغرض، وأوامر البدء السريع،
ومتطلبات المصادقة لأي سؤال يتعلق بأداة سطر الأوامر.

### tool-auditor

يدقق ملف Dockerfile لحاوية التطوير مقابل ملفات مرجع كتالوج الأدوات
للكشف عن الانحراف &mdash; الأدوات المضافة إلى الصورة ولكن غير المفهرسة،
أو إدخالات الكتالوج لأدوات تمت إزالتها.

### container-introspector

ينفذ استبطاناً مباشراً للحاوية عبر واجهة برمجة تطبيقات GitHub
والبيانات الوصفية المحلية للهوية، والنسب، والتشخيص الذاتي، وسجل
المساهمين، وسلالة البناء. عمليات القراءة فقط.

### container-maintainer

يثبت ويزيل ويبحث ويحدث أدوات سطر الأوامر في الحاوية قيد التشغيل
باستخدام أنماط مدير الحزم الصحيحة. يرفع مشكلات GitHub ضد مستودع
حاوية التطوير لتتبع الاستمرارية.
