يوفر ملحق **f5xc-firecrawl** استخراج ويب محلي مستضاف ذاتياً
عبر محرك [firecrawl](https://github.com/mendableai/firecrawl) مفتوح المصدر.
لا مفاتيح API، لا اشتراكات، لا اعتماد على السحابة. جميع
العمليات تعمل ضد مثيل firecrawl المحلي على `localhost:3002`
داخل حاوية التطوير.

<Badge text="v1.1.0" variant="note" /> <Badge text="إنتاجية" variant="tip" />

## التثبيت

```
/plugin install f5xc-firecrawl@f5xc-salesdemos-marketplace
```

## الأوامر

### /scrape

استخراج عنوان URL واحد واستخلاص المحتوى بصيغة markdown.

```
/scrape https://docs.example.com/getting-started
/scrape https://example.com --format markdown,links --wait 2000
```

### /batch-scrape

استخراج عناوين URL متعددة في وقت واحد.

```
/batch-scrape https://example.com https://example.org https://example.net
```

### /crawl

زحف صفحات متعددة بدءاً من عنوان URL.

```
/crawl https://docs.example.com --limit 20 --depth 2
/crawl https://docs.example.com --include /api/* --exclude /blog/*
```

### /map

اكتشاف جميع عناوين URL على موقع ويب.

```
/map https://docs.example.com
/map https://docs.example.com --search api --subdomains
```

### /search

البحث في الويب واستخراج النتائج اختيارياً.

```
/search "firecrawl web scraping" --limit 10
/search "AI tools 2026" --scrape --time month
```

### /extract

استخلاص بيانات منظمة من صفحات الويب بالاعتماد على نماذج اللغة الكبيرة (LLM).

```
/extract https://example.com "Extract the main heading and any links"
/extract https://example.com/pricing --schema '{"plans": [{"name": "string", "price": "string"}]}'
```

<Aside type="note">
  يتطلب أمر extract وكيل LLM متوافق مع OpenAI
  تم تكوينه عبر `OPENAI_BASE_URL` و `OPENAI_API_KEY`.
</Aside>

### /llmstxt

إنشاء ملف llms.txt لموقع ما.

```
/llmstxt https://docs.example.com
```

## المهارات

### web-scraper

يتم تفعيله تلقائياً عندما تطلب استخراج عنوان URL، أو زحف موقع ويب، أو تعيين
عناوين URL للموقع، أو البحث في الويب، أو استخلاص بيانات منظمة، أو إنشاء llms.txt،
أو استخراج عناوين URL متعددة دفعة واحدة، أو تحويل صفحة ويب إلى markdown.
يفوض فوراً إلى وكيل firecrawl-operator.

## الوكلاء

### firecrawl-operator

وكيل استخراج ويب مستقل ينفذ تسلسلات `curl` + `jq`
ضد واجهة برمجة تطبيقات firecrawl المحلية. يدعم 11 بروتوكولاً تغطي جميع
نقاط نهاية v1. وكيل للقراءة فقط (بدون أدوات الكتابة أو التحرير أو الوكيل).

| البروتوكول | نقطة النهاية | النوع |
|----------|----------|------|
| HEALTH | `GET /` | متزامن |
| SCRAPE | `POST /v1/scrape` | متزامن |
| BATCH_SCRAPE | `POST /v1/batch/scrape` | غير متزامن |
| CRAWL | `POST /v1/crawl` | غير متزامن |
| CRAWL_CANCEL | `DELETE /v1/crawl/:id` | متزامن |
| CRAWL_ACTIVE | `GET /v1/crawl/active` | متزامن |
| CRAWL_ERRORS | `GET /v1/crawl/:id/errors` | متزامن |
| MAP | `POST /v1/map` | متزامن |
| SEARCH | `POST /v1/search` | متزامن |
| EXTRACT | `POST /v1/extract` | غير متزامن |
| LLMSTXT | `POST /v1/llmstxt` | غير متزامن |

## البنية التحتية

يتطلب الملحق تشغيل حزمة firecrawl في حاوية التطوير:

| المكون | المنفذ | الغرض |
|-----------|------|---------|
| Firecrawl API | 3002 | جميع نقاط نهاية الاستخراج/الزحف/التعيين/البحث/الاستخلاص |
| Playwright | 3000 | محرك عرض JavaScript |
| Redis | 6379 | الواجهة الخلفية لقائمة المهام |
| PostgreSQL | socket | استمرارية مهام الزحف/الدُفعات |
| LiteLLM proxy | OPENAI_BASE_URL | الواجهة الخلفية لـ LLM للاستخلاص (اختياري) |

تبدأ الحزمة تلقائياً عندما يكون `ENABLE_FIRECRAWL=true` (الإعداد
الافتراضي). يتحقق خطاف SessionStart من إمكانية الوصول إلى واجهة برمجة التطبيقات
ويحذر إذا كانت الخدمة معطلة.

## الاختلافات عن Firecrawl السحابي

يستخدم هذا الملحق الإصدار المستضاف ذاتياً مفتوح المصدر:

- لا حاجة لمصادقة أو مفاتيح API للاستخراج
- لا حدود رصيد أو تحديد معدل
- يستخدم نقاط نهاية واجهة برمجة التطبيقات v1 (وليس v2)
- جلسات المتصفح والبحث المعمق غير متاحة
- يستخدم Extract وكيل LLM الخاص بك بدلاً من النماذج المستضافة
- يعمل بالكامل ضمن شبكة الحاويات المحلية
