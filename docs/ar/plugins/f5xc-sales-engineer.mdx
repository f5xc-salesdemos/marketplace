يوفر إضافة **f5xc-sales-engineer** إطار عمل قابل لإعادة الاستخدام لشخصية مهندس المبيعات لمستودعات العروض التوضيحية لـ F5 XC. يتعامل مع دورة حياة اجتماع العرض التوضيحي بالكامل — من التحقق قبل الاجتماع وحتى التنفيذ المباشر والأسئلة والأجوبة والتفكيك.

<Badge text="v1.0.6" variant="note" /> <Badge text="الإنتاجية" variant="tip" />

## التثبيت

```
/plugin install f5xc-sales-engineer@f5-sales-demo-marketplace
```

## المهارات

### demo-executor

تنفيذ العرض التوضيحي عبر واجهة برمجة التطبيقات مع دورة حياة اجتماع من أربع مراحل:

1. **التحضير** — التحقق من البيئة قبل الاجتماع عبر وكيل `demo-housekeeping`
2. **التنفيذ** — تنفيذ حتمي للمراحل مع سرد موجه للعميل
3. **الأسئلة والأجوبة** — وضع ارتجالي مع البيئة المباشرة
4. **التفكيك** — التنظيف بعد الاجتماع عبر وكيل `demo-housekeeping`

يقرأ المحتوى الخاص بالمنتج من `DEMO_PRODUCT_EXPERTISE.md` وأوامر العرض التوضيحي من `docs/demo/`.

### presenter

عرض تقديمي تفصيلي للتكوين الفعلي باستخدام بيئة عرض توضيحي مُعدة مسبقاً. يتبع نمط عرض → سرد → ربط → توقف عند كل خطوة. يقرأ ترتيب الجولة التفصيلية من `DEMO_WALKTHROUGH_CONFIG.md`.

### subject-matter-expert

أسئلة وأجوبة مدعومة بالمراجع مع متطلبات استشهاد صارمة. لا يخمن أبداً — كل إجابة تتضمن مرجعاً أو دليلاً. يفوض البحث إلى وكيل `demo-researcher` عند الحاجة.

### sales-engineer

فهرس الأدوار الذي يوجه نية المستخدم إلى مهارة الشخصية الصحيحة. يُفعَّل عندما يكون الطلب متعلقاً بالعرض التوضيحي ولكنه لا يتطابق مع عبارة تفعيل محددة.

## الوكلاء

### demo-housekeeping

وكيل مستقل للتحقق قبل الاجتماع والتفكيك بعد الاجتماع. يُشغّل مصفوفة التحقق من الجاهزية (T0–T5) المحددة في `DEMO_READINESS_MATRIX.md` وينفذ أوامر التفكيك من `docs/demo/phase-4-teardown.mdx`.

### demo-researcher

أمين مكتبة بحثي للقراءة فقط يجد إجابات موثقة مع استشهادات. يبحث في `docs/` المحلي أولاً، ثم المصادر الخارجية المفهرسة من `DEMO_SOURCE_INDEX.md`، مع WebSearch كخيار احتياطي.

## ملفات الاتفاقيات

<Aside type="tip" title="المحتوى الخاص بالمنتج">
  هذه الإضافة مستقلة عن المنتج. يوفر كل مستودع محتوى خاصاً بالمنتج من خلال ملفات اتفاقيات موحدة.
</Aside>

أنشئ هذه الملفات في جذر مستودعك لتكوين الإضافة:

| الملف | الغرض | مطلوب لـ |
| ---- | ------- | ------------ |
| `DEMO_PRODUCT_EXPERTISE.md` | قدرات المنتج، إشارات الكشف، تغطية التهديدات، التوافق التنظيمي، مرجع واجهة برمجة التطبيقات | جميع المهارات |
| `DEMO_WALKTHROUGH_CONFIG.md` | عنوان URL لتطبيق العرض التوضيحي، ترتيب الجولة التفصيلية، توقيت الكشف | presenter |
| `DEMO_SOURCE_INDEX.md` | كتالوج مصادر البحث مع المستندات المحلية وعناوين URL الخارجية | demo-researcher |
| `DEMO_READINESS_MATRIX.md` | المتغيرات المطلوبة/الاختيارية، فحوصات الجاهزية (T0–T5)، نقاط نهاية واجهة برمجة التطبيقات | demo-housekeeping |
| `docs/demo/` | ملفات المراحل مع أوامر cURL وبوابات الأدلة | demo-executor |

## أسلوب السرد

تشترك جميع الشخصيات في أسلوب سرد متسق:

- **زمن المضارع، ضمير المتكلم الجمع** — "ما ننظر إليه هنا…"
- **اهتمام واحد لكل فقرة** — كل سرد يجيب على: ما هذا؟ لماذا هو مهم؟ ماذا يجب أن أفعل حيال ذلك؟
- **تسمية الإشارة** — الإشارة إلى القدرة المعنية العاملة
- **ربط تنظيمي عند الاقتضاء** — ذكر التوافق التنظيمي بشكل طبيعي
- **علامة توقف مؤقت** — `> ⏸ *[توقف للجمهور — مستعدون للمتابعة؟]*`
