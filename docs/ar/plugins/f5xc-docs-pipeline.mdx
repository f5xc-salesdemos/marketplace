يوفر ملحق **f5xc-docs-pipeline** معرفة ببنية خط أنابيب التوثيق
لمستودعات f5xc-salesdemos. يرشد في تأليف المحتوى، ويوضح ملكية
الإعدادات عبر البناء متعدد المستودعات، ويوفر معاينة محلية قائمة
على Docker.

<Badge text="v1.0.2" variant="note" /> <Badge text="إنتاجية" variant="tip" />

## التثبيت

```
/plugin install f5xc-docs-pipeline@f5xc-salesdemos-marketplace
```

## المهارات

### content-author

دليل تأليف المحتوى الذي يغطي بنية الملفات وقواعد صياغة MDX
ومتطلبات frontmatter ومراجع الصور والمعاينة المحلية عبر Docker.
يُفعَّل عند إنشاء أو تحرير محتوى `docs/`، أو العمل مع ملفات
MDX، أو إعداد المعاينة المحلية.

### pipeline-navigator

بنية خط أنابيب التوثيق التي تشرح أين يجب إجراء التغييرات عبر
المستودعات الأربعة الأساسية: docs-theme وdocs-builder وdocs-control
وdocs-icons. يوضح ملكية الإعدادات (`astro.config.mjs`،
`content.config.ts`)، وسلسلة إرسال الإصدارات، وتتبع الملفات
المُدارة.

<Aside type="tip">
  استخدم pipeline-navigator عندما تحتاج إلى فهم أي مستودع يملك
  ملف إعدادات أو مكوّن معين.
</Aside>

## الأوامر

### /preview-docs

```
/f5xc-docs-pipeline:preview-docs
```

يبدأ خادم تطوير Docker محلي لمعاينة التوثيق. يشغّل صورة بناء
Docker المشتركة مع تركيب مجلد `docs/` المحلي الخاص بك للمعاينة
المباشرة.
