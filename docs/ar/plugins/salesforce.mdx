يدمج إضافة **salesforce** واجهة سطر أوامر Salesforce مع Claude Code،
مما يوفر مصادقة المؤسسات ووكيل CLI متعدد الأغراض يمكنه
تشغيل أي أمر `sf` بما في ذلك استعلامات SOQL. يدعم بيئات
الحاويات بدون واجهة رسومية مع تدفقات مصادقة JWT ورمز الوصول وعنوان URL الخاص بـ SFDX.
يوضح دليل الاستخدام أدناه كيفية استخدام
اللغة الطبيعية للاستعلام عن الحسابات والفرص والحالات وجهات الاتصال
من خلال وكيل CLI.

<Badge text="v1.0.0" variant="note" /> <Badge text="Development" variant="tip" />

## التثبيت

```
/plugin install salesforce@f5xc-salesdemos-marketplace
```

## المتطلبات الأساسية

- **واجهة سطر أوامر Salesforce** (`@salesforce/cli`)
- **مؤسسة Salesforce** مع وصول API
- حساب مستخدم بصلاحيات الاستعلام عن الكائنات (الحالات، الفرص، الحسابات، جهات الاتصال)

ثبّت واجهة سطر أوامر Salesforce على محطة العمل الخاصة بك:

```bash
brew install sf
```

تحقق من التثبيت:

```bash
sf --version
```

يجب أن ترى مخرجات مثل `@salesforce/cli/2.x.x`.

## إعداد المصادقة

### الخطوة 1: ابحث عن نطاق Salesforce الخاص بك

انظر إلى عنوان URL الخاص بـ Salesforce في المتصفح. إذا كنت تصل إلى Salesforce عبر
`https://acme.lightning.force.com`، فإن نطاق تسجيل الدخول الخاص بك هو
`https://acme.my.salesforce.com`.

<Aside type="tip">
  إذا كانت شركتك تستخدم SSO، يجب عليك تحديد النطاق المخصص باستخدام
  `--instance-url` حتى يتم إعادة توجيه تسجيل الدخول عبر مزود SSO الخاص بك
  بدلاً من صفحة تسجيل الدخول العامة لـ Salesforce.
</Aside>

### الخطوة 2: المصادقة عبر المتصفح

```bash
sf org login web --alias my-org --set-default --instance-url https://YOUR-DOMAIN.my.salesforce.com
```

استبدل `YOUR-DOMAIN` بنطاق شركتك من الخطوة 1. سيفتح المتصفح —
أكمل تدفق SSO أو تسجيل الدخول وقم بتفويض التطبيق. يجب أن ترى:

```
Successfully authorized your-email@company.com with org ID 00DXXXXXXXXXXXXXXX
```

### الخطوة 3: تحقق من الاتصال

```bash
sf org display --target-org my-org
```

تأكد من أن المخرجات تعرض `Connected Status: Connected` مع اسم المستخدم
وعنوان URL للمثيل.

### الخطوة 4: مصادقة محمولة للحاويات (اختياري)

إذا كنت بحاجة إلى المصادقة في بيئة بدون واجهة رسومية (CI/CD، حاوية،
جلسة عن بُعد)، قم بتصدير عنوان URL لمصادقة SFDX من محطة العمل الخاصة بك:

```bash
sf org display --verbose --target-org my-org
```

انسخ قيمة **Sfdx Auth Url** (تبدأ بـ `force://`). في
الحاوية، شغّل:

```bash
export SFDX_AUTH_URL="force://PlatformCLI::YOUR_AUTH_TOKEN@your-domain.my.salesforce.com"
echo "$SFDX_AUTH_URL" | sf org login sfdx-url --sfdx-url-stdin=- --alias=my-org --set-default
```

<Aside type="caution">
  عنوان URL لمصادقة SFDX هو بيانات اعتماد — تعامل معه ككلمة مرور. لا
  تحفظه في ملفات أو تلصقه في المحادثة. استخدم متغيرات البيئة
  وأنابيب stdin فقط.
</Aside>

### مرجع طرق المصادقة

| الطريقة | الأنسب لـ | يتطلب |
|--------|----------|----------|
| تسجيل الدخول عبر الويب | محطات العمل التي بها متصفح | متصفح + SSO |
| عنوان URL لـ SFDX | الحاويات، CI/CD، المصادقة المحمولة | عنوان URL للمصادقة من جلسة مصادق عليها |
| JWT Bearer | خطوط الأنابيب الآلية | تطبيق متصل + مفتاح خاص + مفتاح المستهلك |
| رمز الوصول | مصادقة متغيرات البيئة | `SF_ACCESS_TOKEN` + `SF_ORG_INSTANCE_URL` |

**ملاحظة:** تدفق الجهاز (`sf org login device`) محظور منذ أغسطس
2025 وغير مدعوم.

### متغيرات البيئة

| المتغير | الغرض |
|----------|---------|
| `SF_ACCESS_TOKEN` | رمز Bearer لمصادقة رمز الوصول |
| `SFDX_AUTH_URL` | عنوان URL لمصادقة Force لمصادقة SFDX URL |
| `SF_ORG_INSTANCE_URL` | عنوان URL لمثيل المؤسسة |
| `SF_JWT_KEY_FILE` | مسار المفتاح الخاص لـ JWT |
| `SF_CLIENT_ID` | مفتاح المستهلك للتطبيق المتصل |
| `SF_USERNAME` | اسم مستخدم Salesforce لـ JWT |

## الأوامر

### /salesforce:sf-status

تحقق من حالة اتصال مؤسسة Salesforce وإصدار CLI والمؤسسات
المصادق عليها.

```
/salesforce:sf-status
```

**ما يمكن توقعه:** جدول يعرض إصدار sf CLI والاسم المستعار للمؤسسة المصادق عليها
واسم المستخدم وعنوان URL للمثيل وحالة الاتصال وإصدار API.

### /salesforce:sf-login

المصادقة على مؤسسة Salesforce. يمكنك تقديم اسم مستعار اختيارياً.

```
/salesforce:sf-login
/salesforce:sf-login my-org
```

**ما يمكن توقعه:** إذا كانت المؤسسة بهذا الاسم المستعار مصادقاً عليها بالفعل،
فإنها تبلغ عن تفاصيل الاتصال. إذا لم تكن كذلك، فإنها تتحقق من بيانات الاعتماد
المتاحة (متغيرات البيئة) وتصادق باستخدام أول طريقة مستوفاة بالكامل.

## المهارات

### salesforce-index

موجّه النوايا عالي المستوى. يُفعَّل تلقائياً عند ذكر
Salesforce أو إدارة المؤسسات أو استعلامات البيانات بالغة الطبيعية.

| ما تقوله | يوجَّه إلى |
|---------|-----------|
| "تسجيل الدخول إلى salesforce"، "مصادقة المؤسسة" | مهارة `salesforce-auth` |
| "تحقق من حالة المؤسسة"، "اعرض المؤسسات" | وكيل `cli-operator` |
| "أرني الفرص"، "ابحث عن حالة" | وكيل `cli-operator` (استعلام SOQL) |
| "أنشئ فئة Apex"، "أنشئ Flow" | مهارات تطوير afv-library |

### salesforce-auth

مهارة مصادقة مكيّفة للحاويات. تدعم أربع طرق مصادقة
وتكتشف تلقائياً بيانات الاعتماد المتاحة. تعطي الأولوية لـ:
رمز الوصول، ثم JWT، ثم عنوان URL لـ SFDX، ثم تسجيل الدخول عبر الويب.

## الوكلاء

### cli-operator

وكيل CLI مستقل لـ Salesforce ينفذ أوامر `sf` مع حواجز
أمان. يتم تفويض جميع عمليات Salesforce إلى هذا الوكيل للحفاظ
على سياق الجلسة الرئيسية خفيفاً.

**قواعد الأمان:**

- للقراءة فقط بشكل افتراضي — عمليات الكتابة تتطلب موافقة صريحة
- لا ينشر أبداً دون معاينة وخطوة تأكيد
- لا يعرض أبداً رموز الوصول أو عناوين URL للمصادقة أو بيانات الاعتماد في المخرجات
- يطهّر جميع القيم المقدمة من المستخدم (يرفض محارف الصدفة الخاصة)
- يستخدم مخرجات `--json` لنتائج منظمة وقابلة للتحليل

**تنسيق الاستجابة:**

```
## Result: [SUCCESS | FAILURE | PARTIAL]
### Command Executed
### Output Summary
### Issues
```

## دليل الاستخدام: إدارة الحسابات

<Aside type="note">
  الأوامر أدناه ليست ميزات مبرمجة مسبقاً في الإضافة. إنها تعمل
  لأن وكيل cli-operator يمكنه تشغيل أي أمر CLI لـ `sf`، بما في ذلك
  استعلامات SOQL عشوائية. تعتمد النتائج على نموذج بيانات مؤسستك
  وصلاحيات المستخدم الخاصة بك. تم اختبار الأوامر هنا والتحقق من عملها
  مع كائنات Salesforce القياسية (Account، Contact، Case،
  Opportunity، AccountTeamMember، OpportunityTeamMember).
</Aside>

تعمل هذه الأوامر مع أي مؤسسة Salesforce. استبدل القيم
المؤقتة بمعلوماتك الخاصة. ينتج كل أمر نتائج خاصة
بدورك وحساباتك وإقليمك.

<Aside type="tip">
  للحصول على أفضل النتائج، أضف عنوان بريدك الإلكتروني في الأوامر التي تحتاج
  إلى تحديد سجل المستخدم الخاص بك. يمكن أن تحتوي مؤسسات Salesforce على عدة
  مستخدمين بأسماء متشابهة.
</Aside>

### البدء

بعد المصادقة، تحقق من أن كل شيء يعمل:

```
/salesforce:sf-status
```

ثم جرّب استعلاماً بسيطاً بالغة الطبيعية:

```
list my authenticated salesforce orgs
```

**ما يمكن توقعه:** جدول يعرض جميع المؤسسات المتصلة مع الاسم المستعار
واسم المستخدم وعنوان URL للمثيل وحالة الاتصال.

### ابحث عن حساباتك

اكتشف الحسابات المعيّنة لك:

```
what salesforce accounts am I on the account team for? My email is your-email@company.com
```

**ما يمكن توقعه:** قائمة بالحسابات مجمّعة حسب مالك الحساب، تعرض
دورك كعضو فريق في كل حساب. إذا ظهرت معظم الأدوار فارغة، فهذه
فجوة شائعة في نظافة البيانات — عضويات فريق الحساب موجودة لكن الأدوار
لم تُملأ.

### مقارنة التغطية مع زميل

قارن تغطية حسابك مع زميل في الفريق لاكتشاف الفجوات:

```
In salesforce, find all accounts where Colleague Name is on the account team. Then for each of those accounts, check if your-email@company.com is also on the account team. Show me two lists: accounts where we are BOTH tagged, and accounts where only my colleague is tagged but I am missing.
```

**ما يمكن توقعه:** جدول ملخص يعرض عدد التداخل وعدد الفجوات،
متبوعاً بقائمتين. إذا قمت بتغيير الأدوار مؤخراً، فقد تجد صفر
تداخل — وهذا يكشف عن الحسابات التي تحتاج إلى إضافتك إليها.

<Aside type="note">
  يمكنك أيضاً التحقق من سلف عن طريق إضافة: "Also check if
  Previous Person Name appears on any of those accounts." يساعد هذا
  في تحديد فجوات التسليم عند استبدال شخص ما في إقليم.
</Aside>

### نظرة عامة على خط أنابيب الإقليم

اعرض جميع الفرص المفتوحة عبر حساباتك:

```
show me all open salesforce opportunities on Colleague Name's account team accounts, sorted by close date, include the stage, amount, and probability
```

**ما يمكن توقعه:** ملخص بإجمالي عدد الفرص وخط الأنابيب الخام
وخط الأنابيب المرجّح. ثم جدول توزيع المراحل وقائمة بأهم
الفرص حسب المبلغ. تُشير المخرجات إلى مشكلات نظافة البيانات مثل
تواريخ الإغلاق المتأخرة أو المبالغ الفارغة.

### تحليل معمّق للفرصة

احصل على الصورة الكاملة لصفقة محددة:

```
show me a detailed view of the OPPORTUNITY NAME opportunity in salesforce - include the opportunity team members, any activities or tasks, and the account contacts
```

**ما يمكن توقعه:** نظرة عامة على الفرصة (المرحلة، المبلغ، تاريخ الإغلاق،
الاحتمالية، فئة التوقعات، المالك)، أعضاء فريق الفرصة مع الأدوار،
جهات الاتصال المرتبطة، المهام والأحداث الأخيرة، وسجل تغييرات الحقول
الذي يُظهر كيف تحرك مبلغ الصفقة وتاريخ الإغلاق بمرور الوقت.

### حالات الدعم عبر الإقليم

تحقق من حالات الدعم المفتوحة على حساباتك:

```
show me all open salesforce cases across Colleague Name's account team accounts, grouped by account, sorted by most recent first
```

**ما يمكن توقعه:** عدد الحالات المفتوحة مع تفصيل الأولوية والحالة،
ثم تفاصيل الحالات مجمّعة حسب الحساب. تُوسم الحالات القديمة (أشهر أو
سنوات بدون نشاط) كمرشحات للتنظيف.

### توقعات خط أنابيب ربع سنوية

احصل على عرض جاهز للتوقعات لخط أنابيبك:

```
for Colleague Name's accounts, show me a quarterly pipeline summary - group the open opportunities by close date quarter with count, total amount, and weighted amount for each quarter
```

**ما يمكن توقعه:** جدول بصف واحد لكل ربع يعرض عدد الفرص
وإجمالي خط الأنابيب وخط الأنابيب المرجّح. يتضمن تفصيل مزيج المراحل
لكل ربع ويسلّط الضوء على أهم الصفقات المرجّحة. تُجمّع الفرص المتأخرة
بشكل منفصل.

### نظرة شاملة على الحساب

تعمّق في عميل محدد:

```
give me a full account overview for ACCOUNT NAME in salesforce including contacts, open opportunities, and recent cases
```

**ما يمكن توقعه:** ملف الشركة (القطاع، الإيرادات، الموظفين،
الموقع)، جهات الاتصال الرئيسية مع المناصب والبريد الإلكتروني، الفرص المفتوحة مع
المراحل والمبالغ، وحالات الدعم الأخيرة مع الحالة.

### البحث عن حالة

ابحث عن حالة دعم محددة حسب الرقم:

```
look up salesforce case CASE-NUMBER and show me the case details, the customer account, and who owns it
```

**ما يمكن توقعه:** موضوع الحالة والحالة والأولوية والوصف وملف
حساب العميل ومالك الحالة مع معلومات الاتصال.

## دليل الاستخدام: استعلامات البيانات

للمستخدمين المتقدمين، يمكنك طلب استعلامات بأسلوب SOQL محدد بالغة
الطبيعية. تترجم الإضافة طلبك إلى SOQL المناسب
وتشغّله.

### حقول محددة

```
query salesforce for all Contacts at ACCOUNT NAME - show Name, Title, Email, Phone, and Department
```

### تصفية بالتاريخ

```
show me all salesforce opportunities that closed won in the last 90 days on ACCOUNT NAME
```

### التجميع

```
count all open salesforce cases grouped by priority and status across my accounts
```

### كائنات مخصصة

```
query the CUSTOM_OBJECT__c object in salesforce for records where Status__c = 'Active'
```

<Aside type="tip">
  إذا فشل استعلام لأن حقلاً غير موجود، تعيد الإضافة
  المحاولة تلقائياً بدون ذلك الحقل. قد تستخدم الحقول المخصصة في مؤسستك
  أسماء مختلفة — تحقق من إعداد Salesforce الخاص بك إذا لم يُعثر على
  حقل ما.
</Aside>
