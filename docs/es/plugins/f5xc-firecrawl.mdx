El plugin **f5xc-firecrawl** proporciona raspado web local autoalojado
mediante el motor de código abierto [firecrawl](https://github.com/mendableai/firecrawl).
Sin claves API, sin suscripciones, sin dependencia de la nube. Todas
las operaciones se ejecutan contra la instancia local de firecrawl en `localhost:3002`
dentro del devcontainer.

<Badge text="v1.1.0" variant="note" /> <Badge text="Productivity" variant="tip" />

## Instalación

```
/plugin install f5xc-firecrawl@f5xc-salesdemos-marketplace
```

## Comandos

### /scrape

Raspa una única URL y extrae el contenido como markdown.

```
/scrape https://docs.example.com/getting-started
/scrape https://example.com --format markdown,links --wait 2000
```

### /batch-scrape

Raspa múltiples URLs a la vez.

```
/batch-scrape https://example.com https://example.org https://example.net
```

### /crawl

Rastrea múltiples páginas a partir de una URL inicial.

```
/crawl https://docs.example.com --limit 20 --depth 2
/crawl https://docs.example.com --include /api/* --exclude /blog/*
```

### /map

Descubre todas las URLs de un sitio web.

```
/map https://docs.example.com
/map https://docs.example.com --search api --subdomains
```

### /search

Busca en la web y opcionalmente raspa los resultados.

```
/search "firecrawl web scraping" --limit 10
/search "AI tools 2026" --scrape --time month
```

### /extract

Extracción de datos estructurados potenciada por LLM desde páginas web.

```
/extract https://example.com "Extract the main heading and any links"
/extract https://example.com/pricing --schema '{"plans": [{"name": "string", "price": "string"}]}'
```

<Aside type="note">
  El comando extract requiere un proxy LLM compatible con OpenAI
  configurado mediante `OPENAI_BASE_URL` y `OPENAI_API_KEY`.
</Aside>

### /llmstxt

Genera un archivo llms.txt para un sitio.

```
/llmstxt https://docs.example.com
```

## Habilidades

### web-scraper

Se activa automáticamente cuando solicita raspar una URL, rastrear un sitio web, mapear
URLs del sitio, buscar en la web, extraer datos estructurados, generar llms.txt,
raspar múltiples URLs por lotes o convertir una página web a markdown.
Delega inmediatamente al agente firecrawl-operator.

## Agentes

### firecrawl-operator

Agente autónomo de raspado web que ejecuta secuencias de `curl` + `jq`
contra la API local de firecrawl. Soporta 11 protocolos que cubren todos
los endpoints v1. Agente de solo lectura (sin herramientas de escritura, edición o agente).

| Protocolo | Endpoint | Tipo |
|----------|----------|------|
| HEALTH | `GET /` | Síncrono |
| SCRAPE | `POST /v1/scrape` | Síncrono |
| BATCH_SCRAPE | `POST /v1/batch/scrape` | Asíncrono |
| CRAWL | `POST /v1/crawl` | Asíncrono |
| CRAWL_CANCEL | `DELETE /v1/crawl/:id` | Síncrono |
| CRAWL_ACTIVE | `GET /v1/crawl/active` | Síncrono |
| CRAWL_ERRORS | `GET /v1/crawl/:id/errors` | Síncrono |
| MAP | `POST /v1/map` | Síncrono |
| SEARCH | `POST /v1/search` | Síncrono |
| EXTRACT | `POST /v1/extract` | Asíncrono |
| LLMSTXT | `POST /v1/llmstxt` | Asíncrono |

## Infraestructura

El plugin requiere la pila de firecrawl ejecutándose en el devcontainer:

| Componente | Puerto | Propósito |
|-----------|------|---------|
| Firecrawl API | 3002 | Todos los endpoints de scrape/crawl/map/search/extract |
| Playwright | 3000 | Motor de renderizado JavaScript |
| Redis | 6379 | Backend de cola de trabajos |
| PostgreSQL | socket | Persistencia de trabajos de rastreo/lotes |
| Proxy LiteLLM | OPENAI_BASE_URL | Backend LLM para extract (opcional) |

La pila se inicia automáticamente cuando `ENABLE_FIRECRAWL=true` (el
valor predeterminado). Un hook de SessionStart verifica que la API sea accesible y
advierte si el servicio no está disponible.

## Diferencias con Firecrawl en la nube

Este plugin utiliza la versión autoalojada de código abierto:

- No se requiere autenticación ni claves API para el raspado
- Sin límites de crédito ni limitación de velocidad
- Utiliza endpoints de la API v1 (no v2)
- Las sesiones de navegador e investigación profunda no están disponibles
- Extract utiliza su propio proxy LLM en lugar de modelos alojados
- Se ejecuta completamente dentro de la red local del contenedor
