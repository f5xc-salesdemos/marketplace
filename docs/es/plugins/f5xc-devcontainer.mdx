El plugin **f5xc-devcontainer** proporciona reconocimiento de contenedores para
el entorno de desarrollo f5xc-salesdemos. Mantiene un
catálogo completo de herramientas, permite la introspección en vivo del contenedor
y gestiona la instalación de herramientas con seguimiento de persistencia mediante
issues de GitHub.

<Badge text="v1.1.6" variant="note" /> <Badge text="Productividad" variant="tip" />

## Instalación

```
/plugin install f5xc-devcontainer@f5xc-salesdemos-marketplace
```

## Habilidades

### tool-catalog

Base de conocimientos de cada herramienta CLI instalada en el devcontainer,
organizada por categoría (nube, redes, seguridad, desarrollo,
etc.). Se activa cuando preguntas qué herramienta usar, cómo usar una
herramienta, si algo está instalado o necesitas ayuda con una
operación específica.

### self-awareness

Habilidad de identidad del contenedor y autodiagnóstico. Proporciona
introspección en vivo del contenedor en ejecución a través de la API de GitHub y
metadatos locales de compilación. Se activa para preguntas sobre identidad, versión, información de compilación,
verificaciones de estado e historial de contribuidores.

<Aside type="tip">
  Pregunta "¿quién eres?" o "¿qué versión estoy ejecutando?" para activar
  self-awareness.
</Aside>

## Agentes

### tool-advisor

Lee el catálogo de herramientas y devuelve recomendaciones con propósito,
comandos de inicio rápido y requisitos de autenticación para cualquier pregunta
sobre herramientas CLI.

### tool-auditor

Audita el Dockerfile del devcontainer contra los archivos de referencia del catálogo
de herramientas para detectar desviaciones &mdash; herramientas añadidas a la imagen
pero no catalogadas, o entradas del catálogo para herramientas que han sido
eliminadas.

### container-introspector

Realiza introspección en vivo del contenedor a través de la API de GitHub y
metadatos locales para identidad, genealogía, autodiagnóstico, historial
de contribuidores y linaje de compilación. Solo operaciones de lectura.

### container-maintainer

Instala, elimina, busca y actualiza herramientas CLI en el contenedor
en ejecución utilizando los patrones correctos del gestor de paquetes. Crea issues en GitHub
contra el repositorio del devcontainer para el seguimiento de persistencia.
