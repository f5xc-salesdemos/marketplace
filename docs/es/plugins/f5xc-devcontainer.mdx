El plugin **f5xc-devcontainer** proporciona conciencia de contenedor para
el entorno de desarrollo f5-sales-demo. Mantiene un
catálogo completo de herramientas, habilita la introspección en vivo del contenedor
y gestiona la instalación de herramientas con seguimiento de persistencia mediante
issues de GitHub.

<Badge text="v1.1.6" variant="note" /> <Badge text="Productivity" variant="tip" />

## Instalación

```
/plugin install f5xc-devcontainer@f5-sales-demo-marketplace
```

## Habilidades

### tool-catalog

Base de conocimiento de todas las herramientas CLI instaladas en el contenedor de desarrollo,
organizadas por categoría (nube, redes, seguridad, desarrollo,
etc.). Se activa cuando pregunta qué herramienta utilizar, cómo usar
una herramienta, si algo está instalado, o necesita ayuda con una
operación específica.

### self-awareness

Habilidad de identidad del contenedor y autodiagnóstico. Proporciona introspección
en vivo del contenedor en ejecución mediante la API de GitHub y metadatos
de compilación locales. Se activa para preguntas de identidad, versión, información de compilación,
verificaciones de salud e historial de colaboradores.

<Aside type="tip">
  Pregunte "¿quién eres?" o "¿qué versión estoy ejecutando?" para activar
  la autoconciencia.
</Aside>

## Agentes

### tool-advisor

Lee el catálogo de herramientas y devuelve recomendaciones con propósito,
comandos de inicio rápido y requisitos de autenticación para cualquier pregunta
sobre herramientas CLI.

### tool-auditor

Audita el Dockerfile del contenedor de desarrollo contra los archivos de referencia del catálogo
de herramientas para detectar desviaciones &mdash; herramientas añadidas a la imagen
pero no catalogadas, o entradas del catálogo para herramientas que han sido
eliminadas.

### container-introspector

Realiza introspección en vivo del contenedor mediante la API de GitHub y metadatos
locales para identidad, genealogía, autodiagnóstico, historial de colaboradores
y linaje de compilación. Solo operaciones de solo lectura.

### container-maintainer

Instala, elimina, busca y actualiza herramientas CLI en el contenedor
en ejecución utilizando patrones correctos del gestor de paquetes. Crea issues de GitHub
contra el repositorio del contenedor de desarrollo para el
seguimiento de persistencia.
