本指南说明如何创建新插件并将其添加到
f5xc-salesdemos 市场。

## 插件结构

每个插件都位于 `plugins/` 下的独立目录中，
遵循以下结构：

<FileTree>
- plugins/
  - my-plugin/
    - .claude-plugin/
      - plugin.json
    - skills/
      - my-skill/
        - SKILL.md
        - references/
          - reference-data.md
    - commands/
      - my-command.md
    - agents/
      - my-agent.md
    - README.md
</FileTree>

## 创建插件

<Steps>

1. **创建插件目录**

   ```bash
   mkdir -p plugins/my-plugin/.claude-plugin
   mkdir -p plugins/my-plugin/skills
   mkdir -p plugins/my-plugin/commands
   ```

2. **编写 `plugin.json`**

   创建 `plugins/my-plugin/.claude-plugin/plugin.json`：

   ```json
   {
     "name": "my-plugin",
     "description": "What this plugin does",
     "version": "1.0.0",
     "author": {
       "name": "f5xc-salesdemos"
     },
     "homepage": "https://github.com/f5xc-salesdemos/marketplace/tree/main/plugins/my-plugin",
     "keywords": ["relevant", "keywords"],
     "license": "Apache-2.0",
     "repository": "https://github.com/f5xc-salesdemos/marketplace"
   }
   ```

3. **添加技能**

   创建 `plugins/my-plugin/skills/my-skill/SKILL.md`，
   包含 frontmatter：

   ```markdown
   ---
   name: my-skill
   description: One-line description of what this skill does
   ---

   Detailed instructions for Claude when this skill activates.
   ```

   将任何参考数据放在 `SKILL.md` 旁边的 `references/`
   子目录中。

4. **添加命令**

   创建 `plugins/my-plugin/commands/my-command.md`：

   ```markdown
   ---
   description: What this command does
   argument_hint: "[optional-args]"
   allowed_tools:
     - Read
     - Glob
     - Grep
   ---

   Instructions for Claude when the user invokes this command.
   ```

5. **将插件添加到 marketplace.json**

   在 `.claude-plugin/marketplace.json` 的 `plugins`
   数组中添加一个条目：

   ```json
   {
     "name": "my-plugin",
     "description": "What this plugin does",
     "version": "1.0.0",
     "author": { "name": "f5xc-salesdemos" },
     "source": "./plugins/my-plugin",
     "category": "productivity",
     "homepage": "https://github.com/f5xc-salesdemos/marketplace/tree/main/plugins/my-plugin",
     "license": "Apache-2.0",
     "keywords": ["relevant", "keywords"],
     "tags": ["searchable", "tags"],
     "repository": "https://github.com/f5xc-salesdemos/marketplace"
   }
   ```

6. **本地测试**

   无需发布即可测试您的插件，将 Claude Code
   指向本地目录：

   ```bash
   claude --plugin-dir ./plugins/my-plugin
   ```

   验证技能在正确的上下文中激活，且命令产生
   预期的输出。

</Steps>

## 提交您的插件

<Aside type="tip" title="仓库工作流">
  本仓库强制执行严格的治理工作流。
  请参阅 [CONTRIBUTING.md](https://github.com/f5xc-salesdemos/marketplace/blob/main/CONTRIBUTING.md)
  了解完整流程。
</Aside>

<Steps>

1. 创建一个 GitHub issue 描述新插件
2. 从 `main` 创建一个功能分支
3. 添加您的插件目录并更新 `marketplace.json`
4. 在 `docs/plugins/my-plugin.mdx` 添加文档页面
5. 提交 PR 并通过 `Closes #N` 链接到对应 issue
6. 修复所有 CI 失败问题，检查通过后合并

</Steps>

## 最佳实践

- **保持技能专注** — 一个技能应该只做好一件事
- **使用参考文件** — 将大型数据集、模式和查找表放在
  `references/` 中，而不是内联在 SKILL.md 中
- **智能限定范围** — 尽可能检查未提交的更改以
  缩小工作集
- **按严重程度分组输出** — 使用 ERROR、WARNING 和 INFO
  级别，以便用户优先处理修复
- **编写 README** — 包含使用示例以及每个技能和
  命令的功能描述
