**f5xc-devcontainer** 插件为 f5xc-salesdemos 开发环境提供容器感知能力。它维护一个全面的工具目录，支持实时容器内省，并通过 GitHub issues 处理带有持久化跟踪的工具安装。

<Badge text="v1.1.6" variant="note" /> <Badge text="生产力" variant="tip" />

## 安装

```
/plugin install f5xc-devcontainer@f5xc-salesdemos-marketplace
```

## 技能

### tool-catalog

开发容器中安装的所有 CLI 工具的知识库，按类别（云、网络、安全、开发等）组织。当您询问应使用哪个工具、如何使用某个工具、某个工具是否已安装，或需要特定操作帮助时会被激活。

### self-awareness

容器身份识别和自我诊断技能。通过 GitHub API 和本地构建元数据对运行中的容器进行实时内省。当涉及身份、版本、构建信息、健康检查和贡献者历史等问题时会被激活。

<Aside type="tip">
  尝试询问"你是谁？"或"我运行的是什么版本？"来触发自我感知。
</Aside>

## 代理

### tool-advisor

读取工具目录，并针对任何 CLI 工具问题返回包含用途、快速入门命令和身份验证要求的建议。

### tool-auditor

将开发容器 Dockerfile 与工具目录参考文件进行审计比对，以检测偏差——即已添加到镜像但未编入目录的工具，或目录中记录了但已被移除的工具条目。

### container-introspector

通过 GitHub API 和本地元数据执行实时容器内省，用于身份识别、谱系追溯、自我诊断、贡献者历史和构建血统。仅执行只读操作。

### container-maintainer

使用正确的包管理器模式在运行中的容器中安装、移除、搜索和更新 CLI 工具。向开发容器仓库提交 GitHub issues 以进行持久化跟踪。
