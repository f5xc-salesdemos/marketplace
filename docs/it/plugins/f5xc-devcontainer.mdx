Il plugin **f5xc-devcontainer** fornisce la consapevolezza del contenitore per
l'ambiente di sviluppo f5-sales-demo. Mantiene un
catalogo completo degli strumenti, abilita l'introspezione live del contenitore
e gestisce l'installazione degli strumenti con tracciamento della persistenza tramite
issue GitHub.

<Badge text="v1.1.6" variant="note" /> <Badge text="Produttività" variant="tip" />

## Installazione

```
/plugin install f5xc-devcontainer@f5-sales-demo-marketplace
```

## Competenze

### tool-catalog

Base di conoscenza di ogni strumento CLI installato nel contenitore di sviluppo,
organizzata per categoria (cloud, rete, sicurezza, sviluppo,
ecc.). Si attiva quando si chiede quale strumento utilizzare, come utilizzare
uno strumento, se qualcosa è installato, o si ha bisogno di aiuto con
un'operazione specifica.

### self-awareness

Competenza di identità del contenitore e autodiagnosi. Fornisce
l'introspezione live del contenitore in esecuzione tramite GitHub API e metadati
locali della build. Si attiva per domande su identità, versione, informazioni sulla build,
controlli di integrità e cronologia dei contributi.

<Aside type="tip">
  Digitare "who are you?" o "what version am I running?" per attivare
  self-awareness.
</Aside>

## Agenti

### tool-advisor

Legge il catalogo degli strumenti e restituisce raccomandazioni con scopo,
comandi di avvio rapido e requisiti di autenticazione per qualsiasi domanda
sugli strumenti CLI.

### tool-auditor

Verifica il Dockerfile del contenitore di sviluppo rispetto ai file di riferimento
del catalogo degli strumenti per rilevare derive &mdash; strumenti aggiunti all'immagine
ma non catalogati, o voci del catalogo per strumenti che sono stati
rimossi.

### container-introspector

Esegue l'introspezione live del contenitore tramite GitHub API e metadati
locali per identità, genealogia, autodiagnosi, cronologia dei contributi
e lineage della build. Solo operazioni di sola lettura.

### container-maintainer

Installa, rimuove, cerca e aggiorna gli strumenti CLI nel contenitore
in esecuzione utilizzando i corretti pattern del gestore di pacchetti. Apre issue GitHub
nel repository del contenitore di sviluppo per il
tracciamento della persistenza.
