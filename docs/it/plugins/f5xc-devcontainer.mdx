Il plugin **f5xc-devcontainer** fornisce consapevolezza del container per
l'ambiente di sviluppo f5xc-salesdemos. Mantiene un catalogo
completo degli strumenti, abilita l'introspezione in tempo reale del container
e gestisce l'installazione degli strumenti con tracciamento della persistenza tramite
issue di GitHub.

<Badge text="v1.1.6" variant="note" /> <Badge text="Produttività" variant="tip" />

## Installazione

```
/plugin install f5xc-devcontainer@f5xc-salesdemos-marketplace
```

## Competenze

### tool-catalog

Base di conoscenza di ogni strumento CLI installato nel devcontainer,
organizzato per categoria (cloud, networking, sicurezza, sviluppo,
ecc.). Si attiva quando chiedi quale strumento utilizzare, come usare uno
strumento, se qualcosa è installato, o hai bisogno di aiuto con
un'operazione specifica.

### self-awareness

Competenza di identità e autodiagnosi del container. Fornisce
introspezione in tempo reale del container in esecuzione tramite API GitHub e
metadati di build locali. Si attiva per domande su identità, versione, informazioni
di build, controlli di stato e cronologia dei contributori.

<Aside type="tip">
  Chiedi "chi sei?" o "quale versione sto eseguendo?" per attivare
  self-awareness.
</Aside>

## Agenti

### tool-advisor

Legge il catalogo degli strumenti e restituisce raccomandazioni con scopo,
comandi di avvio rapido e requisiti di autenticazione per qualsiasi domanda
sugli strumenti CLI.

### tool-auditor

Verifica il Dockerfile del devcontainer rispetto ai file di riferimento
del catalogo degli strumenti per rilevare disallineamenti &mdash; strumenti aggiunti
all'immagine ma non catalogati, o voci di catalogo per strumenti che sono stati
rimossi.

### container-introspector

Esegue l'introspezione in tempo reale del container tramite API GitHub e metadati
locali per identità, genealogia, autodiagnosi, cronologia dei contributori
e lineage di build. Solo operazioni in sola lettura.

### container-maintainer

Installa, rimuove, cerca e aggiorna strumenti CLI nel container
in esecuzione utilizzando i pattern corretti del gestore di pacchetti. Crea issue
su GitHub nel repository del devcontainer per il tracciamento
della persistenza.
