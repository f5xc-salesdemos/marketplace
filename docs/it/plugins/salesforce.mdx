Il plugin **salesforce** integra la CLI di Salesforce con Claude Code,
fornendo l'autenticazione org e un agente CLI generico che può
eseguire qualsiasi comando `sf`, incluse le query SOQL. Supporta
ambienti container headless con flussi di autenticazione JWT, access-token
e SFDX URL. La guida all'uso seguente mostra come utilizzare
il linguaggio naturale per interrogare account, opportunità, casi e contatti
tramite l'agente CLI.

<Badge text="v1.0.0" variant="note" /> <Badge text="Development" variant="tip" />

## Installazione

```
/plugin install salesforce@f5xc-salesdemos-marketplace
```

## Prerequisiti

- **Salesforce CLI** (`@salesforce/cli`)
- **Org Salesforce** con accesso API
- Un account utente con permessi per interrogare gli oggetti (Cases, Opportunities, Accounts, Contacts)

Installa la CLI di Salesforce sulla tua workstation:

```bash
brew install sf
```

Verifica l'installazione:

```bash
sf --version
```

Dovresti vedere un output come `@salesforce/cli/2.x.x`.

## Configurazione dell'autenticazione

### Passo 1: Trova il tuo dominio Salesforce

Osserva l'URL di Salesforce nel browser. Se accedi a Salesforce tramite
`https://acme.lightning.force.com`, il tuo dominio di login è
`https://acme.my.salesforce.com`.

<Aside type="tip">
  Se la tua azienda utilizza SSO, devi specificare il tuo dominio personalizzato con
  `--instance-url` in modo che il login venga reindirizzato attraverso il tuo provider SSO
  invece della pagina di login generica di Salesforce.
</Aside>

### Passo 2: Autenticati tramite browser

```bash
sf org login web --alias my-org --set-default --instance-url https://YOUR-DOMAIN.my.salesforce.com
```

Sostituisci `YOUR-DOMAIN` con il dominio della tua azienda dal Passo 1. Il browser
si apre — completa il flusso SSO o di login e autorizza l'app. Dovresti
vedere:

```
Successfully authorized your-email@company.com with org ID 00DXXXXXXXXXXXXXXX
```

### Passo 3: Verifica la connessione

```bash
sf org display --target-org my-org
```

Conferma che l'output mostri `Connected Status: Connected` con il tuo username
e l'URL dell'istanza.

### Passo 4: Autenticazione portabile per container (opzionale)

Se hai bisogno di autenticarti in un ambiente headless (CI/CD, container,
sessione remota), esporta l'URL di autenticazione SFDX dalla tua workstation:

```bash
sf org display --verbose --target-org my-org
```

Copia il valore **Sfdx Auth Url** (inizia con `force://`). Nel
container, esegui:

```bash
export SFDX_AUTH_URL="force://PlatformCLI::YOUR_AUTH_TOKEN@your-domain.my.salesforce.com"
echo "$SFDX_AUTH_URL" | sf org login sfdx-url --sfdx-url-stdin=- --alias=my-org --set-default
```

<Aside type="caution">
  L'URL di autenticazione SFDX è una credenziale — trattalo come una password. Non
  salvarlo in file e non incollarlo nella chat. Usa solo variabili d'ambiente
  e pipe stdin.
</Aside>

### Riferimento metodi di autenticazione

| Metodo | Ideale per | Requisiti |
|--------|-----------|-----------|
| Web Login | Workstation con browser | Browser + SSO |
| SFDX URL | Container, CI/CD, autenticazione portabile | URL di autenticazione da una sessione autenticata |
| JWT Bearer | Pipeline automatizzate | Connected App + chiave privata + consumer key |
| Access Token | Autenticazione tramite variabile d'ambiente | `SF_ACCESS_TOKEN` + `SF_ORG_INSTANCE_URL` |

**Nota:** Il flusso device (`sf org login device`) è bloccato da agosto
2025 e non è supportato.

### Variabili d'ambiente

| Variabile | Scopo |
|----------|-------|
| `SF_ACCESS_TOKEN` | Token bearer per l'autenticazione access-token |
| `SFDX_AUTH_URL` | URL di autenticazione Force per l'autenticazione SFDX URL |
| `SF_ORG_INSTANCE_URL` | URL dell'istanza org |
| `SF_JWT_KEY_FILE` | Percorso della chiave privata JWT |
| `SF_CLIENT_ID` | Consumer key della Connected App |
| `SF_USERNAME` | Username Salesforce per JWT |

## Comandi

### /salesforce:sf-status

Controlla lo stato della connessione alla tua org Salesforce, la versione della CLI e
le org autenticate.

```
/salesforce:sf-status
```

**Cosa aspettarsi:** Una tabella che mostra la versione della CLI sf, l'alias dell'org
autenticata, username, URL dell'istanza, stato della connessione e versione API.

### /salesforce:sf-login

Autenticati a una org Salesforce. Opzionalmente fornisci un alias.

```
/salesforce:sf-login
/salesforce:sf-login my-org
```

**Cosa aspettarsi:** Se un'org con quell'alias è già autenticata,
vengono riportati i dettagli della connessione. In caso contrario, vengono verificate le credenziali
disponibili (variabili d'ambiente) e l'autenticazione avviene utilizzando il primo
metodo completamente soddisfatto.

## Competenze

### salesforce-index

Router di intent di primo livello. Si attiva automaticamente quando menzioni
Salesforce, gestione org o query sui dati in linguaggio naturale.

| Tu dici | Instrada verso |
|---------|---------------|
| "login to salesforce", "authenticate org" | competenza `salesforce-auth` |
| "check org status", "list orgs" | agente `cli-operator` |
| "show me opportunities", "look up case" | agente `cli-operator` (query SOQL) |
| "generate Apex class", "create Flow" | competenze di sviluppo afv-library |

### salesforce-auth

Competenza di autenticazione adattata ai container. Supporta quattro metodi di autenticazione e
rileva automaticamente quali credenziali sono disponibili. Priorità:
access-token, poi JWT, poi SFDX URL, poi web login.

## Agenti

### cli-operator

Agente CLI Salesforce autonomo che esegue comandi `sf` con
protezioni di sicurezza. Tutte le operazioni Salesforce vengono delegate a questo agente per
mantenere snello il contesto della sessione principale.

**Regole di sicurezza:**

- Sola lettura per impostazione predefinita — le operazioni di scrittura richiedono approvazione esplicita
- Non esegue mai deploy senza un'anteprima e un passaggio di conferma
- Non mostra mai access token, URL di autenticazione o credenziali nell'output
- Sanitizza tutti i valori forniti dall'utente (rifiuta metacaratteri della shell)
- Utilizza l'output `--json` per risultati strutturati e analizzabili

**Formato della risposta:**

```
## Result: [SUCCESS | FAILURE | PARTIAL]
### Command Executed
### Output Summary
### Issues
```

## Guida all'uso: Gestione account

<Aside type="note">
  I prompt seguenti non sono funzionalità hard-coded del plugin. Funzionano
  perché l'agente cli-operator può eseguire qualsiasi comando `sf` della CLI, incluse
  query SOQL arbitrarie. I risultati dipendono dal modello dati della tua org e
  dai tuoi permessi utente. I prompt qui riportati sono stati testati e verificati
  per funzionare con gli oggetti standard di Salesforce (Account, Contact, Case,
  Opportunity, AccountTeamMember, OpportunityTeamMember).
</Aside>

Questi prompt funzionano con qualsiasi org Salesforce. Sostituisci i valori
segnaposto con le tue informazioni. Ogni prompt produce risultati specifici
per il tuo ruolo, i tuoi account e il tuo territorio.

<Aside type="tip">
  Per risultati migliori, includi il tuo indirizzo email nei prompt che necessitano di
  identificare il tuo record utente. Le org Salesforce possono avere più utenti
  con nomi simili.
</Aside>

### Per iniziare

Dopo l'autenticazione, verifica che tutto funzioni:

```
/salesforce:sf-status
```

Poi prova una semplice query in linguaggio naturale:

```
list my authenticated salesforce orgs
```

**Cosa aspettarsi:** Una tabella che mostra tutte le org connesse con alias,
username, URL dell'istanza e stato della connessione.

### Trova i tuoi account

Scopri a quali account sei assegnato:

```
what salesforce accounts am I on the account team for? My email is your-email@company.com
```

**Cosa aspettarsi:** Un elenco di account raggruppati per proprietario dell'account, che mostra
il tuo ruolo come membro del team per ciascuno. Se la maggior parte dei ruoli risulta vuota, si tratta
di una comune lacuna nell'igiene dei dati — le appartenenze al team dell'account esistono ma i ruoli
non sono stati popolati.

### Verifica incrociata della copertura con un collega

Confronta la copertura dei tuoi account con quella di un collega per individuare lacune:

```
In salesforce, find all accounts where Colleague Name is on the account team. Then for each of those accounts, check if your-email@company.com is also on the account team. Show me two lists: accounts where we are BOTH tagged, and accounts where only my colleague is tagged but I am missing.
```

**Cosa aspettarsi:** Una tabella riepilogativa che mostra il conteggio delle sovrapposizioni e delle lacune,
seguita da due elenchi. Se hai cambiato ruolo di recente, potresti trovare zero
sovrapposizioni — questo rivela a quali account devi essere aggiunto.

<Aside type="note">
  Puoi anche verificare un predecessore aggiungendo: "Also check if
  Previous Person Name appears on any of those accounts." Questo aiuta
  a identificare le lacune nei passaggi di consegna quando si sostituisce qualcuno in un territorio.
</Aside>

### Panoramica della pipeline del territorio

Visualizza tutte le opportunità aperte nei tuoi account:

```
show me all open salesforce opportunities on Colleague Name's account team accounts, sorted by close date, include the stage, amount, and probability
```

**Cosa aspettarsi:** Un riepilogo con il conteggio totale delle opportunità, la pipeline grezza
e la pipeline ponderata. Poi una tabella di distribuzione per stage e un elenco delle
principali opportunità per importo. L'output segnala problemi di igiene dei dati come
date di chiusura scadute o importi nulli.

### Approfondimento opportunità

Ottieni il quadro completo di un deal specifico:

```
show me a detailed view of the OPPORTUNITY NAME opportunity in salesforce - include the opportunity team members, any activities or tasks, and the account contacts
```

**Cosa aspettarsi:** La panoramica dell'opportunità (stage, importo, data di chiusura,
probabilità, categoria di forecast, proprietario), i membri del team dell'opportunità con
ruoli, i contatti associati, le attività e gli eventi recenti, e la
cronologia delle modifiche ai campi che mostra come l'importo del deal e la data di chiusura sono cambiati nel tempo.

### Casi di supporto nel territorio

Verifica i casi di supporto aperti sui tuoi account:

```
show me all open salesforce cases across Colleague Name's account team accounts, grouped by account, sorted by most recent first
```

**Cosa aspettarsi:** Un conteggio dei casi aperti con ripartizione per priorità e stato,
poi i dettagli dei casi raggruppati per account. I casi stagnanti (mesi o
anni senza attività) vengono segnalati come candidati per la pulizia dei dati.

### Previsione pipeline trimestrale

Ottieni una vista della pipeline pronta per il forecast:

```
for Colleague Name's accounts, show me a quarterly pipeline summary - group the open opportunities by close date quarter with count, total amount, and weighted amount for each quarter
```

**Cosa aspettarsi:** Una tabella con una riga per trimestre che mostra il conteggio delle
opportunità, la pipeline totale e la pipeline ponderata. Include una ripartizione del mix
di stage per trimestre ed evidenzia i principali deal ponderati. Le opportunità
scadute vengono raggruppate separatamente.

### Panoramica completa dell'account

Approfondimento su un cliente specifico:

```
give me a full account overview for ACCOUNT NAME in salesforce including contacts, open opportunities, and recent cases
```

**Cosa aspettarsi:** Profilo aziendale (settore, fatturato, dipendenti,
sede), contatti chiave con titoli ed email, opportunità aperte con
stage e importi, e casi di supporto recenti con stato.

### Ricerca caso

Cerca un caso di supporto specifico per numero:

```
look up salesforce case CASE-NUMBER and show me the case details, the customer account, and who owns it
```

**Cosa aspettarsi:** Oggetto del caso, stato, priorità, descrizione, il
profilo dell'account cliente e il proprietario del caso con le informazioni di contatto.

## Guida all'uso: Query sui dati

Per gli utenti avanzati, è possibile richiedere query specifiche in stile SOQL in
linguaggio naturale. Il plugin traduce la tua richiesta nella query SOQL appropriata
e la esegue.

### Campi specifici

```
query salesforce for all Contacts at ACCOUNT NAME - show Name, Title, Email, Phone, and Department
```

### Filtro per data

```
show me all salesforce opportunities that closed won in the last 90 days on ACCOUNT NAME
```

### Aggregazione

```
count all open salesforce cases grouped by priority and status across my accounts
```

### Oggetti personalizzati

```
query the CUSTOM_OBJECT__c object in salesforce for records where Status__c = 'Active'
```

<Aside type="tip">
  Se una query fallisce perché un campo non esiste, il plugin
  ritenta automaticamente senza quel campo. I campi personalizzati nella tua org
  potrebbero avere nomi diversi — controlla il Setup di Salesforce se un campo
  non viene trovato.
</Aside>
