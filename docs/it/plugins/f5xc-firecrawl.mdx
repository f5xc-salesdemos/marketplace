Il plugin **f5xc-firecrawl** fornisce web scraping locale self-hosted
tramite il motore open-source [firecrawl](https://github.com/mendableai/firecrawl).
Nessuna chiave API, nessun abbonamento, nessuna dipendenza dal cloud. Tutte
le operazioni vengono eseguite contro l'istanza locale di firecrawl su `localhost:3002`
all'interno del devcontainer.

<Badge text="v1.1.0" variant="note" /> <Badge text="Produttività" variant="tip" />

## Installazione

```
/plugin install f5xc-firecrawl@f5xc-salesdemos-marketplace
```

## Comandi

### /scrape

Esegue lo scraping di un singolo URL ed estrae il contenuto come markdown.

```
/scrape https://docs.example.com/getting-started
/scrape https://example.com --format markdown,links --wait 2000
```

### /batch-scrape

Esegue lo scraping di più URL contemporaneamente.

```
/batch-scrape https://example.com https://example.org https://example.net
```

### /crawl

Esegue il crawling di più pagine a partire da un URL iniziale.

```
/crawl https://docs.example.com --limit 20 --depth 2
/crawl https://docs.example.com --include /api/* --exclude /blog/*
```

### /map

Scopre tutti gli URL di un sito web.

```
/map https://docs.example.com
/map https://docs.example.com --search api --subdomains
```

### /search

Cerca nel web e opzionalmente esegue lo scraping dei risultati.

```
/search "firecrawl web scraping" --limit 10
/search "AI tools 2026" --scrape --time month
```

### /extract

Estrazione di dati strutturati da pagine web tramite LLM.

```
/extract https://example.com "Extract the main heading and any links"
/extract https://example.com/pricing --schema '{"plans": [{"name": "string", "price": "string"}]}'
```

<Aside type="note">
  Il comando extract richiede un proxy LLM compatibile con OpenAI
  configurato tramite `OPENAI_BASE_URL` e `OPENAI_API_KEY`.
</Aside>

### /llmstxt

Genera un file llms.txt per un sito.

```
/llmstxt https://docs.example.com
```

## Competenze

### web-scraper

Si attiva automaticamente quando si richiede di eseguire lo scraping di un URL, il crawling di un sito web, la mappatura
degli URL del sito, la ricerca nel web, l'estrazione di dati strutturati, la generazione di llms.txt,
lo scraping batch di più URL o la conversione di una pagina web in markdown.
Delega immediatamente all'agente firecrawl-operator.

## Agenti

### firecrawl-operator

Agente autonomo di web scraping che esegue sequenze `curl` + `jq`
contro l'API locale di firecrawl. Supporta 11 protocolli che coprono tutti
gli endpoint v1. Agente in sola lettura (nessuno strumento Write, Edit o Agent).

| Protocollo | Endpoint | Tipo |
|----------|----------|------|
| HEALTH | `GET /` | Sincrono |
| SCRAPE | `POST /v1/scrape` | Sincrono |
| BATCH_SCRAPE | `POST /v1/batch/scrape` | Asincrono |
| CRAWL | `POST /v1/crawl` | Asincrono |
| CRAWL_CANCEL | `DELETE /v1/crawl/:id` | Sincrono |
| CRAWL_ACTIVE | `GET /v1/crawl/active` | Sincrono |
| CRAWL_ERRORS | `GET /v1/crawl/:id/errors` | Sincrono |
| MAP | `POST /v1/map` | Sincrono |
| SEARCH | `POST /v1/search` | Sincrono |
| EXTRACT | `POST /v1/extract` | Asincrono |
| LLMSTXT | `POST /v1/llmstxt` | Asincrono |

## Infrastruttura

Il plugin richiede lo stack firecrawl in esecuzione nel devcontainer:

| Componente | Porta | Scopo |
|-----------|------|---------|
| Firecrawl API | 3002 | Tutti gli endpoint scrape/crawl/map/search/extract |
| Playwright | 3000 | Motore di rendering JavaScript |
| Redis | 6379 | Backend per la coda dei job |
| PostgreSQL | socket | Persistenza dei job crawl/batch |
| Proxy LiteLLM | OPENAI_BASE_URL | Backend LLM per extract (opzionale) |

Lo stack si avvia automaticamente quando `ENABLE_FIRECRAWL=true` (il
valore predefinito). Un hook SessionStart verifica che l'API sia raggiungibile e
avvisa se il servizio non è disponibile.

## Differenze rispetto a Firecrawl Cloud

Questo plugin utilizza la versione open-source self-hosted:

- Nessuna autenticazione o chiave API richiesta per lo scraping
- Nessun limite di crediti o rate limiting
- Utilizza gli endpoint API v1 (non v2)
- Sessioni browser e deep research non disponibili
- Extract utilizza il proprio proxy LLM invece dei modelli hosted
- Funziona interamente all'interno della rete locale del container
