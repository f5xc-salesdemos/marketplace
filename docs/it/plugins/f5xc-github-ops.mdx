Il plugin **f5xc-github-ops** fornisce un'automazione esclusiva delle
operazioni GitHub per i repository f5xc-salesdemos. Gestisce l'intero
ciclo di vita Git e GitHub dalla creazione degli issue fino al
monitoraggio post-merge, applicando lo sviluppo basato su issue e i
gate di lint pre-commit.

<Badge text="v2.2.2" variant="note" /> <Badge text="Produttività" variant="tip" />

## Installazione

```
/plugin install f5xc-github-ops@f5xc-salesdemos-marketplace
```

## Competenze

### workflow-lifecycle

Gestione del workflow di governance del repository che comprende
creazione di issue, convenzioni di denominazione dei branch, creazione
di PR, polling CI con feedback degli errori agli issue, monitoraggio
post-merge, verifica e criteri di completamento delle attività. Si
attiva quando si dice "commit", "push", "crea una PR", "merge" o si
esegue qualsiasi operazione Git.

### upstream-contribution

Workflow di contribuzione upstream per lo sviluppo basato su fork.
Gestisce il rilevamento dei fork, la ricerca upstream (issue, PR,
CONTRIBUTING.md, pattern CI), la creazione di issue e PR cross-repo,
la gestione degli issue di tracciamento locali, il monitoraggio CI
upstream e la risoluzione guidata.

## Agenti

### github-ops

Agente esclusivo per tutte le operazioni meccaniche Git e GitHub.
Gestisce i gate di lint pre-commit, la creazione di issue, la
creazione di branch, lo staging, il commit, il push, la creazione
di PR, il polling CI con feedback degli errori, il retry in caso di
errori infrastrutturali, il monitoraggio post-merge, la pulizia dei
branch e la gestione delle impostazioni del repository tramite le
API GitHub.

Restituisce report di stato strutturati: `COMPLETE`,
`PRE_COMMIT_FAILED`, `CI_FAILED`, `BLOCKED` o `FAILED`.

<Aside type="caution">
  Questo agente è il gestore **esclusivo** delle operazioni Git.
  Non eseguire mai `git commit`, `git push` o `gh pr create`
  direttamente nei repository che utilizzano questo plugin.
</Aside>
