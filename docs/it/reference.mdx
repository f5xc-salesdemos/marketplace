## Schema di marketplace.json

Il manifest del marketplace si trova in
`.claude-plugin/marketplace.json` nella radice del repository.

### Campi di primo livello

| Campo | Obbligatorio | Descrizione |
|-------|----------|-------------|
| `$schema` | No | URL dello JSON Schema per la validazione |
| `name` | Sì | Identificativo univoco del marketplace |
| `version` | No | Versione dello schema del marketplace (semver) |
| `metadata.description` | Sì | Descrizione del marketplace leggibile dall'utente |
| `owner.name` | Sì | Organizzazione o utente proprietario del marketplace |
| `owner.url` | No | URL al profilo o sito web del proprietario |
| `owner.email` | No | Email di contatto |
| `plugins` | Sì | Array di voci dei plugin |

### Campi delle voci dei plugin

| Campo | Obbligatorio | Descrizione |
|-------|----------|-------------|
| `name` | Sì | Identificativo del plugin (deve corrispondere a `plugin.json`) |
| `description` | Sì | Breve descrizione del plugin |
| `version` | Sì | Stringa di versione semantica |
| `author.name` | Sì | Autore del plugin |
| `source` | Sì | Percorso o URL del plugin (vedi Tipi di sorgente) |
| `category` | Sì | Categoria del plugin (es. `productivity`) |
| `homepage` | No | URL alla documentazione o homepage del plugin |
| `license` | No | Identificativo di licenza SPDX |
| `keywords` | No | Array di parole chiave per la ricerca |
| `tags` | No | Array di tag per il filtraggio |
| `repository` | No | URL del repository |

## Tipi di sorgente dei plugin

Il campo `source` in marketplace.json supporta molteplici
formati:

| Tipo | Esempio | Descrizione |
|------|---------|-------------|
| Percorso relativo | `./plugins/my-plugin` | Plugin nello stesso repository |
| Abbreviazione GitHub | `owner/repo` | Plugin nella radice del repository |
| GitHub con percorso | `owner/repo/path/to/plugin` | Plugin in una sottodirectory |
| URL Git | `https://github.com/owner/repo.git` | Repository Git |
| Pacchetto npm | `npm:@scope/package` | Pacchetto npm pubblicato |

## Schema di plugin.json

Ogni plugin ha un manifest in
`.claude-plugin/plugin.json` all'interno della propria directory.

| Campo | Obbligatorio | Descrizione |
|-------|----------|-------------|
| `name` | Sì | Identificativo del plugin |
| `description` | Sì | Cosa fa il plugin |
| `version` | Sì | Versione semantica |
| `author.name` | Sì | Autore del plugin |
| `author.url` | No | URL al profilo o sito web dell'autore |
| `homepage` | No | URL della documentazione |
| `keywords` | No | Parole chiave per la ricerca |
| `license` | No | Identificativo di licenza SPDX |
| `repository` | No | URL del repository |

## Struttura della directory del plugin

```
plugins/my-plugin/
  .claude-plugin/
    plugin.json          # Plugin manifest (required)
  skills/
    skill-name/
      SKILL.md           # Skill definition (required per skill)
      references/        # Reference data files (optional)
  commands/
    command-name.md      # Command definition (one per command)
  agents/
    agent-name.md        # Agent definition (one per agent)
  README.md              # Plugin documentation (recommended)
```

## Variabili d'ambiente

I plugin possono utilizzare queste variabili nei propri file
di skill e comandi:

| Variabile | Descrizione |
|----------|-------------|
| `$\{CLAUDE_PLUGIN_ROOT\}` | Percorso assoluto alla directory radice del plugin |

## Validazione

Validate i manifest del marketplace e dei plugin localmente:

```bash
claude plugin validate .
```

Questo verifica:

- Conformità allo schema di `marketplace.json`
- Esistenza di tutte le directory dei plugin referenziati
- Ogni plugin ha un `plugin.json` valido
- Le skill hanno un frontmatter di `SKILL.md` formattato correttamente
- I comandi hanno i campi frontmatter obbligatori

## Configurazione del team

### Pre-configurazione dei marketplace

Aggiungete a `.claude/settings.json` in qualsiasi repository:

```json
{
  "extraKnownMarketplaces": [
    "f5xc-salesdemos/marketplace"
  ]
}
```

### Pre-abilitazione dei plugin

```json
{
  "enabledPlugins": [
    "f5xc-docs-tools@f5xc-salesdemos-marketplace"
  ]
}
```

<Aside type="tip">
  Committare queste impostazioni nel vostro repository assicura
  che ogni membro del team riceva automaticamente la stessa
  configurazione del marketplace e dei plugin.
</Aside>

## Risoluzione dei problemi

### Marketplace non trovato

Verificate che il marketplace sia stato aggiunto:

```
/plugin marketplace list
```

Se mancante, aggiungetelo nuovamente:

```
/plugin marketplace add f5xc-salesdemos/marketplace
```

### Il plugin non si attiva

- Confermate che il plugin sia installato:
  `/plugin list`
- Verificate che la vostra versione di Claude Code supporti i plugin
  (v1.0.33+)
- Verificate che le condizioni di attivazione della skill corrispondano
  al vostro contesto attuale

### Le skill non si attivano automaticamente

Le skill si attivano in base al campo `description` nel
frontmatter di SKILL.md. Se una skill non si attiva quando
previsto:

- Verificate che la descrizione descriva accuratamente il
  contesto di attivazione
- Utilizzate il comando direttamente (es. `/f5xc-docs-tools:review-mdx`)
  come soluzione alternativa

### Errori di build dopo la revisione

Il plugin segnala i problemi ma non li corregge automaticamente.
Applicate le correzioni suggerite manualmente, poi rieseguite
la revisione per confermare la risoluzione.
