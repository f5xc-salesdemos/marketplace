Le plugin **osint-framework** fournit un catalogue d'outils OSINT et
des compétences d'investigation — 1 064 outils gratuits de collecte
de renseignements répartis dans 34 catégories, cartographiés depuis [osintframework.com](https://osintframework.com).
Il inclut des compétences basées sur les catégories, des pipelines d'investigation
exécutables, l'exécution d'outils CLI, des workflows respectant l'OPSEC, et un graphe
d'entités persistant pour la corrélation inter-investigations.

<Badge text="v1.0.1" variant="note" /> <Badge text="Sécurité" variant="tip" />

## Installation

```
/plugin install osint-framework@f5xc-salesdemos-marketplace
```

## Commandes

### /osint-investigate

Exécute une investigation OSINT complète sur n'importe quelle cible. Détecte
automatiquement le type de cible (e-mail, domaine, IP, nom d'utilisateur, entreprise,
nom de personne) et oriente vers les compétences de catégorie appropriées.

```
/osint-investigate j.smith@example.com
/osint-investigate example.com
/osint-investigate 198.51.100.1
/osint-investigate example-user
```

### /osint-search

Recherche dans le catalogue d'outils ceux correspondant à une requête de capacité.

```
/osint-search subdomain enumeration
/osint-search email breach check
/osint-search satellite imagery
```

### /osint-catalog

Parcourt le catalogue complet d'outils par catégorie. Sans argument, affiche
les 34 catégories avec le nombre d'outils.

```
/osint-catalog
/osint-catalog domain
/osint-catalog threat-intel
```

## Compétences

### osint-index

Routeur d'intention de niveau supérieur. S'active automatiquement lorsque vous dites
« trouver des informations sur », « investiguer », « rechercher », « vérification d'antécédents »,
ou décrivez toute tâche de renseignement en sources ouvertes. Oriente vers la compétence
de catégorie appropriée en fonction du type de cible.

| Cible | Oriente vers |
|-------|-------------|
| Nom d'utilisateur ou pseudo | `username-recon` |
| Adresse e-mail | `email-recon` |
| Nom de domaine | `domain-recon` |
| Adresse IP | `ip-address-recon` |
| Nom de personne + entreprise | `people-search` + `business-records` |
| Entreprise / organisation | `business-records` + `domain-recon` |
| Hash de malware | `malicious-file-analysis` + `threat-intelligence` |

### osint-catalog

Navigateur du catalogue d'outils. Fournit l'accès au registre complet des outils,
à l'index des catégories, aux pipelines d'investigation, à la référence des limites de débit,
et à la documentation du moteur de corrélation.

## Agents

### osint-investigator

Orchestrateur d'investigation autonome multi-outils. Exécute des workflows
d'investigation complets — vérification de la disponibilité des outils, détection
du type de cible, exécution séquentielle des outils (passif en premier), agrégation
des résultats et génération de rapports structurés avec niveaux de confiance
et classifications OPSEC.

S'intègre au graphe d'entités : chaque résultat d'outil crée des entités
et des relations dans `/tmp/osint-graph/` pour la corrélation inter-investigations.

<Aside type="caution">
  Les outils de reconnaissance active (nmap, nuclei, sherlock) établissent un contact direct
  avec l'infrastructure cible. Le hook d'avertissement OPSEC alerte avant
  d'exécuter les outils actifs. Assurez-vous toujours d'avoir l'autorisation nécessaire.
</Aside>

### osint-researcher

Agent de consultation du catalogue en lecture seule. Recherche dans les fichiers de référence
pour recommander les meilleurs outils pour une tâche donnée. N'exécute jamais d'outils —
recommande et explique uniquement. Utilisé en interne par les compétences de catégorie
pour la découverte d'outils.

### osint-executor

Agent d'exécution CLI d'outil unique. Exécute un outil à la fois avec
des paramètres spécifiques, analyse la sortie et retourne des résultats structurés.
Inclut des recettes d'exécution par outil pour plus de 20 outils avec les drapeaux exacts,
les formats de sortie et les modèles d'analyse jq.

## Limitation de débit

Tous les appels API utilisent l'utilitaire `osint_curl` avec un backoff exponentiel
intégré sur HTTP 429/503. Limites principales :

| API | Limite gratuite | Variable d'env. |
|-----|----------------|-----------------|
| ipinfo.io | 1 000/jour | — |
| crt.sh | 60/min | — |
| NVD | 5/30s (sans clé) | `NVD_API_KEY` |
| OpenCorporates | Clé requise | `OPENCORPORATES_API_KEY` |
| Shodan | 1/sec | `SHODAN_API_KEY` |
| GitHub API | 60/h (sans auth) | `GH_TOKEN` |

Les API nécessitant des clés sont automatiquement ignorées avec un message
clair si la variable d'environnement n'est pas définie.

## Moteur de corrélation

Le plugin inclut un graphe d'entités JSON persistant dans
`/tmp/osint-graph/` qui accumule les découvertes à travers les investigations.

```bash
source plugins/osint-framework/scripts/osint-graph.sh
osint_graph_init

# Ajouter des entités
P=$(osint_entity_add "person" "J. Smith" --tool github-api)
C=$(osint_entity_add "company" "F5" --tool github-api)

# Les relier
osint_rel_add "$P" "$C" "works_at" --tool github-api

# Requête : toutes les entités à 2 sauts de la personne
osint_graph_query "$P" 2

# Générer le rapport
osint_graph_report
```

**Le scoring de confiance** utilise la fusion multi-sources :
`confidence = 1 - product(1 - ci)` pour chaque source confirmante.
Deux sources à 0,80 → 0,96 de confiance combinée.
