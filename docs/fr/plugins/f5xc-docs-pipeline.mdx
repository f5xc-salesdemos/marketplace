Le plugin **f5xc-docs-pipeline** fournit des connaissances sur
l'architecture du pipeline de documentation pour les dépôts
f5xc-salesdemos. Il guide la rédaction de contenu, clarifie la
propriété des configurations à travers les builds multi-dépôts,
et offre une prévisualisation locale basée sur Docker.

<Badge text="v1.0.2" variant="note" /> <Badge text="Productivité" variant="tip" />

## Installation

```
/plugin install f5xc-docs-pipeline@f5xc-salesdemos-marketplace
```

## Compétences

### content-author

Guide de rédaction de contenu couvrant la structure des fichiers,
les règles de syntaxe MDX, les exigences du frontmatter, les
références d'images et la prévisualisation locale avec Docker.
S'active lors de la création ou de l'édition de contenu dans `docs/`,
lors du travail avec des fichiers MDX, ou lors de la configuration
de la prévisualisation locale.

### pipeline-navigator

Architecture du pipeline de documentation expliquant où effectuer
des modifications à travers les quatre dépôts en amont : docs-theme,
docs-builder, docs-control et docs-icons. Clarifie la propriété
des configurations (`astro.config.mjs`, `content.config.ts`), la
chaîne de dispatch des releases et le suivi des fichiers gérés.

<Aside type="tip">
  Utilisez pipeline-navigator lorsque vous avez besoin de comprendre
  quel dépôt est propriétaire d'un fichier de configuration ou d'un
  composant spécifique.
</Aside>

## Commandes

### /preview-docs

```
/f5xc-docs-pipeline:preview-docs
```

Démarre un serveur de développement Docker local pour prévisualiser
la documentation. Exécute l'image Docker de build partagée avec
votre répertoire local `docs/` monté pour une prévisualisation en
temps réel.
