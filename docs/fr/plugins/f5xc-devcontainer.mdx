Le plugin **f5xc-devcontainer** fournit une conscience des conteneurs pour
l'environnement de développement f5-sales-demo. Il maintient un
catalogue d'outils complet, permet l'introspection en direct des conteneurs
et gère l'installation des outils avec suivi de persistance via
des issues GitHub.

<Badge text="v1.1.6" variant="note" /> <Badge text="Productivity" variant="tip" />

## Installation

```
/plugin install f5xc-devcontainer@f5-sales-demo-marketplace
```

## Compétences

### tool-catalog

Base de connaissances de chaque outil CLI installé dans le conteneur de développement,
organisée par catégorie (cloud, réseau, sécurité, développement,
etc.). S'active lorsque vous demandez quel outil utiliser, comment utiliser
un outil, si quelque chose est installé, ou si vous avez besoin d'aide pour
une opération spécifique.

### self-awareness

Compétence d'identité et d'auto-diagnostic du conteneur. Fournit une
introspection en direct du conteneur en cours d'exécution via l'API GitHub et les
métadonnées de construction locales. S'active pour les questions d'identité, de version, d'informations
de construction, de contrôles de santé et d'historique des contributeurs.

<Aside type="tip">
  Posez la question « qui êtes-vous ? » ou « quelle version est-ce que j'exécute ? » pour déclencher
  la conscience de soi.
</Aside>

## Agents

### tool-advisor

Lit le catalogue d'outils et retourne des recommandations avec l'objectif,
les commandes de démarrage rapide et les exigences d'authentification pour toute question
relative à un outil CLI.

### tool-auditor

Audite le Dockerfile du conteneur de développement par rapport aux fichiers de référence du catalogue
d'outils pour détecter les dérives &mdash; les outils ajoutés à l'image
mais non répertoriés, ou les entrées du catalogue pour les outils qui ont été
supprimés.

### container-introspector

Effectue une introspection en direct du conteneur via l'API GitHub et les
métadonnées locales pour l'identité, la généalogie, l'auto-diagnostic, l'historique
des contributeurs et la lignée de construction. Opérations en lecture seule uniquement.

### container-maintainer

Installe, supprime, recherche et met à jour les outils CLI dans le conteneur
en cours d'exécution en utilisant les modèles de gestionnaire de paquets appropriés. Crée des issues GitHub
dans le dépôt du conteneur de développement pour le suivi
de persistance.
