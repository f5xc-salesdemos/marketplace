Ce guide explique comment créer un nouveau plugin et l'ajouter
au marketplace f5xc-salesdemos.

## Structure d'un plugin

Chaque plugin réside dans son propre répertoire sous `plugins/` et
suit cette structure :

<FileTree>
- plugins/
  - my-plugin/
    - .claude-plugin/
      - plugin.json
    - skills/
      - my-skill/
        - SKILL.md
        - references/
          - reference-data.md
    - commands/
      - my-command.md
    - agents/
      - my-agent.md
    - README.md
</FileTree>

## Créer un plugin

<Steps>

1. **Créer le répertoire du plugin**

   ```bash
   mkdir -p plugins/my-plugin/.claude-plugin
   mkdir -p plugins/my-plugin/skills
   mkdir -p plugins/my-plugin/commands
   ```

2. **Rédiger `plugin.json`**

   Créez `plugins/my-plugin/.claude-plugin/plugin.json` :

   ```json
   {
     "name": "my-plugin",
     "description": "What this plugin does",
     "version": "1.0.0",
     "author": {
       "name": "f5xc-salesdemos"
     },
     "homepage": "https://github.com/f5xc-salesdemos/marketplace/tree/main/plugins/my-plugin",
     "keywords": ["relevant", "keywords"],
     "license": "Apache-2.0",
     "repository": "https://github.com/f5xc-salesdemos/marketplace"
   }
   ```

3. **Ajouter des compétences (skills)**

   Créez `plugins/my-plugin/skills/my-skill/SKILL.md` avec
   le frontmatter :

   ```markdown
   ---
   name: my-skill
   description: One-line description of what this skill does
   ---

   Detailed instructions for Claude when this skill activates.
   ```

   Placez toutes les données de référence dans un sous-répertoire
   `references/` aux côtés de `SKILL.md`.

4. **Ajouter des commandes**

   Créez `plugins/my-plugin/commands/my-command.md` :

   ```markdown
   ---
   description: What this command does
   argument_hint: "[optional-args]"
   allowed_tools:
     - Read
     - Glob
     - Grep
   ---

   Instructions for Claude when the user invokes this command.
   ```

5. **Ajouter le plugin à marketplace.json**

   Ajoutez une entrée dans le tableau `plugins` de
   `.claude-plugin/marketplace.json` :

   ```json
   {
     "name": "my-plugin",
     "description": "What this plugin does",
     "version": "1.0.0",
     "author": { "name": "f5xc-salesdemos" },
     "source": "./plugins/my-plugin",
     "category": "productivity",
     "homepage": "https://github.com/f5xc-salesdemos/marketplace/tree/main/plugins/my-plugin",
     "license": "Apache-2.0",
     "keywords": ["relevant", "keywords"],
     "tags": ["searchable", "tags"],
     "repository": "https://github.com/f5xc-salesdemos/marketplace"
   }
   ```

6. **Tester localement**

   Testez votre plugin sans le publier en dirigeant Claude Code
   vers le répertoire local :

   ```bash
   claude --plugin-dir ./plugins/my-plugin
   ```

   Vérifiez que les compétences s'activent dans le bon contexte et
   que les commandes produisent le résultat attendu.

</Steps>

## Soumettre votre plugin

<Aside type="tip" title="Flux de travail du dépôt">
  Ce dépôt applique un flux de travail de gouvernance strict.
  Consultez [CONTRIBUTING.md](https://github.com/f5xc-salesdemos/marketplace/blob/main/CONTRIBUTING.md)
  pour le processus complet.
</Aside>

<Steps>

1. Créez une issue GitHub décrivant le nouveau plugin
2. Créez une branche de fonctionnalité à partir de `main`
3. Ajoutez le répertoire de votre plugin et mettez à jour `marketplace.json`
4. Ajoutez une page de documentation à `docs/plugins/my-plugin.mdx`
5. Ouvrez une PR liée à l'issue avec `Closes #N`
6. Corrigez tout échec CI et fusionnez après la réussite des vérifications

</Steps>

## Bonnes pratiques

- **Gardez les compétences ciblées** — une compétence doit faire une seule chose et la faire bien
- **Utilisez les fichiers de référence** — placez les grands ensembles de données, schémas et
  tables de correspondance dans `references/` plutôt qu'en ligne dans SKILL.md
- **Définissez le périmètre intelligemment** — vérifiez les modifications non commitées pour
  réduire l'ensemble de travail lorsque c'est possible
- **Regroupez les résultats par sévérité** — utilisez les niveaux ERROR, WARNING et INFO
  afin que les utilisateurs puissent prioriser les corrections
- **Rédigez un README** — incluez des exemples d'utilisation et une description
  de ce que chaque compétence et commande fait
