**osint-framework** प्लगइन OSINT टूल कैटलॉग और जांच कौशल प्रदान
करता है — [osintframework.com](https://osintframework.com) से मैप
की गई 34 श्रेणियों में 1,064 मुफ्त इंटेलिजेंस-गैदरिंग टूल्स।
इसमें श्रेणी-आधारित कौशल, निष्पादन योग्य जांच पाइपलाइन,
CLI टूल निष्पादन, OPSEC-सचेत वर्कफ़्लो, और क्रॉस-इन्वेस्टिगेशन
सहसंबंध के लिए एक स्थायी एंटिटी ग्राफ शामिल है।

<Badge text="v1.0.1" variant="note" /> <Badge text="Security" variant="tip" />

## स्थापना

```
/plugin install osint-framework@f5xc-salesdemos-marketplace
```

## कमांड्स

### /osint-investigate

किसी भी लक्ष्य के विरुद्ध पूर्ण OSINT जांच चलाएं। यह स्वचालित
रूप से लक्ष्य प्रकार (ईमेल, डोमेन, IP, उपयोगकर्ता नाम, कंपनी,
व्यक्ति का नाम) का पता लगाता है और सही श्रेणी कौशल पर रूट करता है।

```
/osint-investigate j.smith@example.com
/osint-investigate example.com
/osint-investigate 198.51.100.1
/osint-investigate example-user
```

### /osint-search

किसी क्षमता क्वेरी से मेल खाने वाले टूल्स के लिए टूल कैटलॉग खोजें।

```
/osint-search subdomain enumeration
/osint-search email breach check
/osint-search satellite imagery
```

### /osint-catalog

श्रेणी के अनुसार पूर्ण टूल कैटलॉग ब्राउज़ करें। बिना आर्ग्युमेंट्स
के यह टूल काउंट्स के साथ सभी 34 श्रेणियां दिखाता है।

```
/osint-catalog
/osint-catalog domain
/osint-catalog threat-intel
```

## कौशल

### osint-index

शीर्ष-स्तरीय इंटेंट राउटर। जब आप "find information about",
"investigate", "look up", "background check" कहते हैं, या किसी
ओपन-सोर्स इंटेलिजेंस कार्य का वर्णन करते हैं, तो यह स्वचालित
रूप से सक्रिय हो जाता है। लक्ष्य प्रकार के आधार पर सही श्रेणी
कौशल पर रूट करता है।

| लक्ष्य | रूट किया जाता है |
|--------|----------|
| उपयोगकर्ता नाम या हैंडल | `username-recon` |
| ईमेल पता | `email-recon` |
| डोमेन नाम | `domain-recon` |
| IP पता | `ip-address-recon` |
| व्यक्ति का नाम + कंपनी | `people-search` + `business-records` |
| कंपनी / संगठन | `business-records` + `domain-recon` |
| मालवेयर हैश | `malicious-file-analysis` + `threat-intelligence` |

### श्रेणी कौशल (कुल 34)

प्रत्येक श्रेणी में CLI टूल्स, वेब संसाधन, जांच वर्कफ़्लो,
क्रॉस-कैटेगरी पिवट्स, और OPSEC नोट्स के साथ एक समर्पित कौशल है।

| श्रेणी | टूल्स | प्रमुख CLI टूल्स |
|----------|-------|--------------|
| `username-recon` | 18 | sherlock, maigret, sylva |
| `email-recon` | 28 | holehe, h8mail, theHarvester |
| `domain-recon` | 131 | subfinder, amass, whois, dig |
| `cloud-recon` | 23 | cloud_enum, S3Scanner, ScoutSuite |
| `ip-address-recon` | 55 | nmap, masscan, checkip |
| `images-videos` | 89 | exiftool, exiv2, yt-dlp |
| `social-networks` | 63 | maigret, yt-dlp, Osintgram |
| `threat-intelligence` | 38 | iocextract, ioc_parser, pymisp |
| `malicious-file-analysis` | 33 | oletools, pdfid, Ghidra |
| `search-engines` | 73 | gitleaks, pagodo, gobuster |
| `geolocation` | 47 | Hyperlapse |
| `opsec` | 47 | 47 वेब-आधारित गोपनीयता टूल्स |
| `people-search` | 18 | GitHub API, GitLab API |
| `business-records` | 28 | SEC EDGAR, OpenCorporates |
| *(20 और श्रेणियां)* | 500+ | — |

<Aside type="tip">
  सभी 34 श्रेणियां ब्राउज़ करने के लिए `/osint-catalog` का उपयोग करें या
  किसी विशिष्ट कार्य के लिए टूल्स खोजने के लिए `/osint-search <capability>` का उपयोग करें।
</Aside>

### osint-catalog

टूल कैटलॉग ब्राउज़र। पूर्ण टूल रजिस्ट्री, श्रेणी सूचकांक,
जांच पाइपलाइन, रेट-लिमिट संदर्भ, और सहसंबंध इंजन
दस्तावेज़ीकरण तक पहुंच प्रदान करता है।

## एजेंट्स

### osint-investigator

स्वायत्त मल्टी-टूल जांच ऑर्केस्ट्रेटर। पूर्ण जांच वर्कफ़्लो
चलाता है — टूल उपलब्धता जांच, लक्ष्य प्रकार पहचान, अनुक्रमिक
टूल निष्पादन (पहले पैसिव), परिणाम एकत्रीकरण, और विश्वास
स्तरों तथा OPSEC वर्गीकरणों के साथ संरचित रिपोर्ट निर्माण।

एंटिटी ग्राफ के साथ एकीकृत: प्रत्येक टूल परिणाम
क्रॉस-इन्वेस्टिगेशन सहसंबंध के लिए `/tmp/osint-graph/` में
एंटिटीज़ और संबंध बनाता है।

<Aside type="caution">
  सक्रिय रिकॉन टूल्स (nmap, nuclei, sherlock) लक्ष्य
  इन्फ्रास्ट्रक्चर से सीधे संपर्क करते हैं। OPSEC चेतावनी हुक
  सक्रिय टूल्स चलाने से पहले अलर्ट करता है। हमेशा सुनिश्चित करें
  कि आपके पास प्राधिकरण है।
</Aside>

### osint-researcher

केवल-पठन कैटलॉग लुकअप एजेंट। किसी दिए गए कार्य के लिए
सर्वोत्तम टूल्स की सिफारिश करने हेतु संदर्भ फ़ाइलें खोजता है।
कभी टूल्स निष्पादित नहीं करता — केवल सिफारिश और व्याख्या करता
है। टूल खोज के लिए श्रेणी कौशलों द्वारा आंतरिक रूप से उपयोग
किया जाता है।

### osint-executor

एकल-टूल CLI निष्पादन एजेंट। विशिष्ट पैरामीटर्स के साथ एक
बार में एक टूल चलाता है, आउटपुट को पार्स करता है, और संरचित
परिणाम लौटाता है। 20+ टूल्स के लिए सटीक फ़्लैग्स, आउटपुट
फ़ॉर्मेट्स, और jq पार्सिंग पैटर्न के साथ प्रति-टूल निष्पादन
रेसिपी शामिल हैं।

## रेट लिमिटिंग

सभी API कॉल HTTP 429/503 पर बिल्ट-इन एक्सपोनेंशियल बैकऑफ़ के
साथ `osint_curl` हेल्पर का उपयोग करते हैं। प्रमुख सीमाएं:

| API | मुफ्त सीमा | एन्व वेरिएबल |
|-----|-----------|---------|
| ipinfo.io | 1,000/दिन | — |
| crt.sh | 60/मिनट | — |
| NVD | 5/30 सेकंड (बिना की) | `NVD_API_KEY` |
| OpenCorporates | की आवश्यक | `OPENCORPORATES_API_KEY` |
| Shodan | 1/सेकंड | `SHODAN_API_KEY` |
| GitHub API | 60/घंटा (बिना auth) | `GH_TOKEN` |

जिन API को कीज़ की आवश्यकता होती है, यदि एन्वायरनमेंट वेरिएबल
सेट नहीं है तो उन्हें स्पष्ट संदेश के साथ स्वचालित रूप से
छोड़ दिया जाता है।

## सहसंबंध इंजन

प्लगइन में `/tmp/osint-graph/` पर एक स्थायी JSON एंटिटी ग्राफ
शामिल है जो जांचों में निष्कर्ष संचित करता है।

```bash
source plugins/osint-framework/scripts/osint-graph.sh
osint_graph_init

# एंटिटीज़ जोड़ें
P=$(osint_entity_add "person" "J. Smith" --tool github-api)
C=$(osint_entity_add "company" "F5" --tool github-api)

# उन्हें लिंक करें
osint_rel_add "$P" "$C" "works_at" --tool github-api

# क्वेरी: व्यक्ति से 2 हॉप्स के भीतर सभी एंटिटीज़
osint_graph_query "$P" 2

# रिपोर्ट जनरेट करें
osint_graph_report
```

**विश्वास स्कोरिंग** मल्टी-सोर्स फ़्यूज़न का उपयोग करती है:
प्रत्येक पुष्टि करने वाले स्रोत के लिए
`confidence = 1 - product(1 - ci)`।
0.80 पर दो स्रोत → 0.96 संयुक्त विश्वास।

## कानूनी सूचना

सभी टूल्स केवल सार्वजनिक रूप से उपलब्ध जानकारी का उपयोग करते
हैं। उपयोगकर्ता लागू कानूनों, विनियमों, और प्लेटफ़ॉर्म सेवा
शर्तों के अनुपालन को सुनिश्चित करने के लिए स्वयं जिम्मेदार
हैं। कभी भी OSINT टूल्स का उपयोग अनधिकृत पहुंच, उत्पीड़न,
या किसी अवैध गतिविधि के लिए न करें।
