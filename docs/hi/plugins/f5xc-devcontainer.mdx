**f5xc-devcontainer** प्लगइन f5-sales-demo डेवलपमेंट एनवायरनमेंट के लिए कंटेनर अवेयरनेस प्रदान करता है। यह एक व्यापक टूल कैटलॉग बनाए रखता है, लाइव कंटेनर इंट्रोस्पेक्शन सक्षम करता है, और GitHub इशू के माध्यम से परसिस्टेंस ट्रैकिंग के साथ टूल इंस्टालेशन संभालता है।

<Badge text="v1.1.6" variant="note" /> <Badge text="Productivity" variant="tip" />

## इंस्टालेशन

```
/plugin install f5xc-devcontainer@f5-sales-demo-marketplace
```

## स्किल्स

### tool-catalog

डेवकंटेनर में इंस्टॉल किए गए प्रत्येक CLI टूल का नॉलेज बेस, श्रेणी के अनुसार व्यवस्थित (क्लाउड, नेटवर्किंग, सुरक्षा, डेवलपमेंट, आदि)। यह तब सक्रिय होता है जब आप पूछते हैं कि कौन सा टूल उपयोग करना है, किसी टूल का उपयोग कैसे करना है, कुछ इंस्टॉल है या नहीं, या किसी विशिष्ट ऑपरेशन में सहायता की आवश्यकता है।

### self-awareness

कंटेनर आइडेंटिटी और सेल्फ-डायग्नोसिस स्किल। GitHub API और लोकल बिल्ड मेटाडेटा के माध्यम से रनिंग कंटेनर का लाइव इंट्रोस्पेक्शन प्रदान करता है। आइडेंटिटी, वर्शन, बिल्ड इन्फो, हेल्थ चेक, और कंट्रीब्यूटर हिस्ट्री संबंधी प्रश्नों के लिए सक्रिय होता है।

<Aside type="tip">
  self-awareness को ट्रिगर करने के लिए "who are you?" या "what version am I running?" पूछें।
</Aside>

## एजेंट्स

### tool-advisor

टूल कैटलॉग पढ़ता है और किसी भी CLI टूल प्रश्न के लिए उद्देश्य, क्विक-स्टार्ट कमांड, और ऑथेंटिकेशन आवश्यकताओं के साथ अनुशंसाएँ लौटाता है।

### tool-auditor

ड्रिफ्ट का पता लगाने के लिए टूल कैटलॉग रेफरेंस फाइलों के विरुद्ध डेवकंटेनर Dockerfile का ऑडिट करता है &mdash; इमेज में जोड़े गए लेकिन कैटलॉग न किए गए टूल्स, या उन टूल्स के लिए कैटलॉग एंट्री जो हटा दिए गए हैं।

### container-introspector

आइडेंटिटी, जेनेलॉजी, सेल्फ-डायग्नोसिस, कंट्रीब्यूटर हिस्ट्री, और बिल्ड लिनिएज के लिए GitHub API और लोकल मेटाडेटा के माध्यम से लाइव कंटेनर इंट्रोस्पेक्शन करता है। केवल रीड-ओनली ऑपरेशन।

### container-maintainer

सही पैकेज मैनेजर पैटर्न का उपयोग करके रनिंग कंटेनर में CLI टूल्स को इंस्टॉल, हटाता, खोजता, और अपडेट करता है। परसिस्टेंस ट्रैकिंग के लिए डेवकंटेनर रिपॉजिटरी के विरुद्ध GitHub इशू फाइल करता है।
