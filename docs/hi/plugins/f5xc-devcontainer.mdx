**f5xc-devcontainer** प्लगइन f5xc-salesdemos डेवलपमेंट एनवायरनमेंट के लिए
कंटेनर अवेयरनेस प्रदान करता है। यह एक व्यापक टूल कैटलॉग बनाए रखता है,
लाइव कंटेनर इंट्रोस्पेक्शन सक्षम करता है, और GitHub इश्यूज़ के माध्यम से
पर्सिस्टेंस ट्रैकिंग के साथ टूल इंस्टॉलेशन को संभालता है।

<Badge text="v1.1.6" variant="note" /> <Badge text="उत्पादकता" variant="tip" />

## इंस्टॉलेशन

```
/plugin install f5xc-devcontainer@f5xc-salesdemos-marketplace
```

## स्किल्स

### tool-catalog

Devcontainer में इंस्टॉल किए गए प्रत्येक CLI टूल का नॉलेज बेस,
कैटेगरी के अनुसार व्यवस्थित (क्लाउड, नेटवर्किंग, सुरक्षा, डेवलपमेंट,
आदि)। जब आप पूछते हैं कि कौन सा टूल उपयोग करना है, किसी टूल का
उपयोग कैसे करना है, कुछ इंस्टॉल है या नहीं, या किसी विशिष्ट ऑपरेशन
में सहायता चाहिए, तब यह सक्रिय होता है।

### self-awareness

कंटेनर आइडेंटिटी और सेल्फ-डायग्नोसिस स्किल। GitHub API और लोकल
बिल्ड मेटाडेटा के माध्यम से चल रहे कंटेनर का लाइव इंट्रोस्पेक्शन
प्रदान करता है। आइडेंटिटी, वर्शन, बिल्ड जानकारी, हेल्थ चेक, और
कंट्रीब्यूटर हिस्ट्री संबंधी प्रश्नों के लिए सक्रिय होता है।

<Aside type="tip">
  Self-awareness को ट्रिगर करने के लिए "तुम कौन हो?" या "मैं कौन सा
  वर्शन चला रहा हूँ?" पूछें।
</Aside>

## एजेंट्स

### tool-advisor

टूल कैटलॉग पढ़ता है और किसी भी CLI टूल प्रश्न के लिए उद्देश्य,
क्विक-स्टार्ट कमांड्स, और ऑथेंटिकेशन आवश्यकताओं सहित अनुशंसाएँ
लौटाता है।

### tool-auditor

ड्रिफ्ट का पता लगाने के लिए टूल कैटलॉग रेफरेंस फाइलों के विरुद्ध
devcontainer Dockerfile का ऑडिट करता है &mdash; ऐसे टूल जो इमेज में
जोड़े गए हैं लेकिन कैटलॉग में नहीं हैं, या ऐसी कैटलॉग एंट्रीज़
जिनके टूल हटा दिए गए हैं।

### container-introspector

आइडेंटिटी, वंशावली, सेल्फ-डायग्नोसिस, कंट्रीब्यूटर हिस्ट्री, और
बिल्ड लिनिएज के लिए GitHub API और लोकल मेटाडेटा के माध्यम से लाइव
कंटेनर इंट्रोस्पेक्शन करता है। केवल रीड-ओनली ऑपरेशन।

### container-maintainer

सही पैकेज मैनेजर पैटर्न का उपयोग करके चल रहे कंटेनर में CLI टूल्स
को इंस्टॉल, हटाता, खोजता, और अपडेट करता है। पर्सिस्टेंस ट्रैकिंग
के लिए devcontainer रिपॉजिटरी के विरुद्ध GitHub इश्यूज़ फाइल करता है।
