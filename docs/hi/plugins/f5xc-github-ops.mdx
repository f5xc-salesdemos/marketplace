**f5xc-github-ops** प्लगइन f5xc-salesdemos रिपॉज़िटरीज़ के लिए विशेष GitHub संचालन
स्वचालन प्रदान करता है। यह इश्यू निर्माण से लेकर पोस्ट-मर्ज
निगरानी तक संपूर्ण Git और GitHub जीवनचक्र को संभालता है, जो
इश्यू-संचालित विकास और प्री-कमिट लिंट गेट्स को लागू करता है।

<Badge text="v2.2.2" variant="note" /> <Badge text="Productivity" variant="tip" />

## स्थापना

```
/plugin install f5xc-github-ops@f5xc-salesdemos-marketplace
```

## कौशल

### workflow-lifecycle

रिपॉज़िटरी गवर्नेंस वर्कफ़्लो जो इश्यू निर्माण, ब्रांच
नामकरण परंपराएँ, PR निर्माण, त्रुटि प्रतिक्रिया के साथ CI पोलिंग
इश्यूज़ पर, पोस्ट-मर्ज निगरानी, सत्यापन, और कार्य
पूर्णता मानदंडों को संभालता है। जब आप "commit", "push",
"create a PR", "merge" कहते हैं, या कोई भी Git ऑपरेशन करते हैं तो यह सक्रिय होता है।

### upstream-contribution

फ़ोर्क-आधारित विकास के लिए अपस्ट्रीम योगदान वर्कफ़्लो। फ़ोर्क
पहचान, अपस्ट्रीम अनुसंधान (इश्यूज़, PRs, CONTRIBUTING.md,
CI पैटर्न), क्रॉस-रिपो इश्यू और PR निर्माण, स्थानीय ट्रैकिंग
इश्यू प्रबंधन, अपस्ट्रीम CI निगरानी, और मार्गदर्शित समाधान
को संभालता है।

## एजेंट

### github-ops

सभी यांत्रिक Git और GitHub संचालन के लिए विशेष एजेंट।
प्री-कमिट लिंट गेट्स, इश्यू निर्माण, ब्रांच निर्माण,
स्टेजिंग, कमिटिंग, पुशिंग, PR निर्माण, त्रुटि
प्रतिक्रिया के साथ CI पोलिंग, इन्फ्रास्ट्रक्चर विफलता पुनः प्रयास, पोस्ट-मर्ज निगरानी,
ब्रांच क्लीनअप, और GitHub API के माध्यम से रिपॉज़िटरी सेटिंग्स प्रबंधन
को संभालता है।

संरचित स्थिति रिपोर्ट लौटाता है: `COMPLETE`,
`PRE_COMMIT_FAILED`, `CI_FAILED`, `BLOCKED`, या `FAILED`।

<Aside type="caution">
  यह एजेंट Git संचालन का **विशेष** हैंडलर है।
  इस प्लगइन का उपयोग करने वाली रिपॉज़िटरीज़ में कभी भी सीधे
  `git commit`, `git push`, या `gh pr create` न चलाएँ।
</Aside>
