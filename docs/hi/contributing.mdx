यह मार्गदर्शिका बताती है कि एक नया प्लगइन कैसे बनाएं और इसे
f5xc-salesdemos मार्केटप्लेस में कैसे जोड़ें।

## प्लगइन संरचना

हर प्लगइन `plugins/` के अंतर्गत अपनी स्वयं की निर्देशिका में रहता है और
इस संरचना का पालन करता है:

<FileTree>
- plugins/
  - my-plugin/
    - .claude-plugin/
      - plugin.json
    - skills/
      - my-skill/
        - SKILL.md
        - references/
          - reference-data.md
    - commands/
      - my-command.md
    - agents/
      - my-agent.md
    - README.md
</FileTree>

## प्लगइन बनाना

<Steps>

1. **प्लगइन निर्देशिका बनाएं**

   ```bash
   mkdir -p plugins/my-plugin/.claude-plugin
   mkdir -p plugins/my-plugin/skills
   mkdir -p plugins/my-plugin/commands
   ```

2. **`plugin.json` लिखें**

   `plugins/my-plugin/.claude-plugin/plugin.json` बनाएं:

   ```json
   {
     "name": "my-plugin",
     "description": "What this plugin does",
     "version": "1.0.0",
     "author": {
       "name": "f5xc-salesdemos"
     },
     "homepage": "https://github.com/f5xc-salesdemos/marketplace/tree/main/plugins/my-plugin",
     "keywords": ["relevant", "keywords"],
     "license": "Apache-2.0",
     "repository": "https://github.com/f5xc-salesdemos/marketplace"
   }
   ```

3. **स्किल्स जोड़ें**

   `plugins/my-plugin/skills/my-skill/SKILL.md` को
   फ्रंटमैटर के साथ बनाएं:

   ```markdown
   ---
   name: my-skill
   description: One-line description of what this skill does
   ---

   Detailed instructions for Claude when this skill activates.
   ```

   किसी भी संदर्भ डेटा को `SKILL.md` के साथ एक `references/`
   उपनिर्देशिका में रखें।

4. **कमांड जोड़ें**

   `plugins/my-plugin/commands/my-command.md` बनाएं:

   ```markdown
   ---
   description: What this command does
   argument_hint: "[optional-args]"
   allowed_tools:
     - Read
     - Glob
     - Grep
   ---

   Instructions for Claude when the user invokes this command.
   ```

5. **प्लगइन को marketplace.json में जोड़ें**

   `.claude-plugin/marketplace.json` में `plugins` ऐरे में
   एक एंट्री जोड़ें:

   ```json
   {
     "name": "my-plugin",
     "description": "What this plugin does",
     "version": "1.0.0",
     "author": { "name": "f5xc-salesdemos" },
     "source": "./plugins/my-plugin",
     "category": "productivity",
     "homepage": "https://github.com/f5xc-salesdemos/marketplace/tree/main/plugins/my-plugin",
     "license": "Apache-2.0",
     "keywords": ["relevant", "keywords"],
     "tags": ["searchable", "tags"],
     "repository": "https://github.com/f5xc-salesdemos/marketplace"
   }
   ```

6. **स्थानीय रूप से परीक्षण करें**

   प्रकाशित किए बिना अपने प्लगइन का परीक्षण करने के लिए Claude Code
   को स्थानीय निर्देशिका की ओर इंगित करें:

   ```bash
   claude --plugin-dir ./plugins/my-plugin
   ```

   सत्यापित करें कि स्किल्स सही संदर्भ में सक्रिय होती हैं और
   कमांड अपेक्षित आउटपुट देते हैं।

</Steps>

## अपना प्लगइन सबमिट करना

<Aside type="tip" title="रिपॉजिटरी वर्कफ़्लो">
  यह रिपॉजिटरी एक सख्त गवर्नेंस वर्कफ़्लो लागू करती है।
  पूरी प्रक्रिया के लिए [CONTRIBUTING.md](https://github.com/f5xc-salesdemos/marketplace/blob/main/CONTRIBUTING.md)
  देखें।
</Aside>

<Steps>

1. नए प्लगइन का वर्णन करते हुए एक GitHub इश्यू बनाएं
2. `main` से एक फीचर ब्रांच बनाएं
3. अपनी प्लगइन निर्देशिका जोड़ें और `marketplace.json` अपडेट करें
4. `docs/plugins/my-plugin.mdx` पर एक डॉक्यूमेंटेशन पेज जोड़ें
5. `Closes #N` के साथ इश्यू से लिंक करते हुए एक PR खोलें
6. किसी भी CI विफलता को ठीक करें और जांच पास होने के बाद मर्ज करें

</Steps>

## सर्वोत्तम प्रथाएं

- **स्किल्स को केंद्रित रखें** — एक स्किल को एक काम अच्छी तरह करना चाहिए
- **संदर्भ फाइलों का उपयोग करें** — बड़े डेटासेट, स्कीमा, और
  लुकअप टेबल को SKILL.md में इनलाइन के बजाय `references/` में रखें
- **बुद्धिमानी से स्कोप करें** — जब संभव हो तो वर्किंग सेट को
  सीमित करने के लिए अनकमिटेड परिवर्तनों की जांच करें
- **आउटपुट को गंभीरता के अनुसार समूहित करें** — ERROR, WARNING, और INFO
  स्तरों का उपयोग करें ताकि उपयोगकर्ता सुधारों को प्राथमिकता दे सकें
- **एक README लिखें** — उपयोग के उदाहरण और प्रत्येक स्किल
  और कमांड क्या करता है उसका विवरण शामिल करें
