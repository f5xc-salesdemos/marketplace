यह मार्गदर्शिका बताती है कि एक नया प्लगइन कैसे बनाएं और उसे
f5-sales-demo मार्केटप्लेस में कैसे जोड़ें।

## प्लगइन संरचना

प्रत्येक प्लगइन `plugins/` के अंतर्गत अपनी निर्देशिका में रहता है और
इस संरचना का पालन करता है:

<FileTree>
- plugins/
  - example-plugin/
    - .claude-plugin/
      - plugin.json
    - skills/
      - example-skill/
        - SKILL.md
        - references/
          - reference-data.md
    - commands/
      - example-command.md
    - agents/
      - example-agent.md
    - README.md
</FileTree>

## प्लगइन बनाना

<Steps>

1. **प्लगइन निर्देशिका बनाएं**

   ```bash
   mkdir -p plugins/example-plugin/.claude-plugin
   mkdir -p plugins/example-plugin/skills
   mkdir -p plugins/example-plugin/commands
   ```

2. **`plugin.json` लिखें**

   `plugins/example-plugin/.claude-plugin/plugin.json` बनाएं:

   ```json
   {
     "name": "example-plugin",
     "description": "What this plugin does",
     "version": "1.0.0",
     "author": {
       "name": "f5-sales-demo"
     },
     "homepage": "https://github.com/f5-sales-demo/marketplace/tree/main/plugins/example-plugin",
     "keywords": ["relevant", "keywords"],
     "license": "Apache-2.0",
     "repository": "https://github.com/f5-sales-demo/marketplace"
   }
   ```

3. **स्किल्स जोड़ें**

   `plugins/example-plugin/skills/example-skill/SKILL.md` को
   frontmatter के साथ बनाएं:

   ```markdown
   ---
   name: example-skill
   description: One-line description of what this skill does
   ---

   Detailed instructions for Claude when this skill activates.
   ```

   किसी भी संदर्भ डेटा को `SKILL.md` के साथ `references/` उपनिर्देशिका में रखें।

4. **कमांड जोड़ें**

   `plugins/example-plugin/commands/example-command.md` बनाएं:

   ```markdown
   ---
   description: What this command does
   argument_hint: "[optional-args]"
   allowed_tools:
     - Read
     - Glob
     - Grep
   ---

   Instructions for Claude when the user invokes this command.
   ```

5. **प्लगइन को marketplace.json में जोड़ें**

   `.claude-plugin/marketplace.json` में `plugins` array में एक प्रविष्टि जोड़ें:

   ```json
   {
     "name": "example-plugin",
     "description": "What this plugin does",
     "version": "1.0.0",
     "author": { "name": "f5-sales-demo" },
     "source": "./plugins/example-plugin",
     "category": "productivity",
     "homepage": "https://github.com/f5-sales-demo/marketplace/tree/main/plugins/example-plugin",
     "license": "Apache-2.0",
     "keywords": ["relevant", "keywords"],
     "tags": ["searchable", "tags"],
     "repository": "https://github.com/f5-sales-demo/marketplace"
   }
   ```

6. **स्थानीय रूप से परीक्षण करें**

   Claude Code को स्थानीय निर्देशिका की ओर इंगित करके प्रकाशित किए बिना
   अपने प्लगइन का परीक्षण करें:

   ```bash
   claude --plugin-dir ./plugins/example-plugin
   ```

   सत्यापित करें कि स्किल्स सही संदर्भ में सक्रिय होती हैं और
   कमांड अपेक्षित आउटपुट उत्पन्न करते हैं।

</Steps>

## अपना प्लगइन सबमिट करना

<Aside type="tip" title="रिपॉज़िटरी वर्कफ़्लो">
  यह रिपॉज़िटरी एक सख्त गवर्नेंस वर्कफ़्लो लागू करती है।
  पूरी प्रक्रिया के लिए [CONTRIBUTING.md](https://github.com/f5-sales-demo/marketplace/blob/main/CONTRIBUTING.md)
  देखें।
</Aside>

<Steps>

1. नए प्लगइन का वर्णन करते हुए एक GitHub इश्यू बनाएं
2. `main` से एक फीचर ब्रांच बनाएं
3. अपनी प्लगइन निर्देशिका जोड़ें और `marketplace.json` अपडेट करें
4. `docs/plugins/example-plugin.mdx` पर एक दस्तावेज़ीकरण पृष्ठ जोड़ें
5. `Closes #N` के साथ इश्यू से लिंक करते हुए एक PR खोलें
6. किसी भी CI विफलताओं को ठीक करें और जाँच पास होने के बाद मर्ज करें

</Steps>

## सर्वोत्तम प्रथाएं

- **स्किल्स को केंद्रित रखें** — एक स्किल को एक काम अच्छी तरह करना चाहिए
- **संदर्भ फ़ाइलें उपयोग करें** — बड़े डेटासेट, स्कीमा, और
  लुकअप तालिकाएं SKILL.md में इनलाइन करने के बजाय `references/` में रखें
- **बुद्धिमानी से स्कोप करें** — जब संभव हो तो कार्यशील सेट को संकुचित करने के लिए
  uncommitted परिवर्तनों की जाँच करें
- **आउटपुट को गंभीरता के अनुसार समूहित करें** — ERROR, WARNING, और INFO
  स्तरों का उपयोग करें ताकि उपयोगकर्ता सुधारों को प्राथमिकता दे सकें
- **README लिखें** — उपयोग के उदाहरण और प्रत्येक स्किल और कमांड का
  विवरण शामिल करें
