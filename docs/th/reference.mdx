## สคีมา marketplace.json

ไฟล์แมนิเฟสต์ของ marketplace อยู่ที่
`.claude-plugin/marketplace.json` ในรูทของรีโพซิทอรี

### ฟิลด์ระดับบนสุด

| ฟิลด์ | จำเป็น | คำอธิบาย |
|-------|----------|-------------|
| `$schema` | ไม่ | URL ของ JSON Schema สำหรับการตรวจสอบความถูกต้อง |
| `name` | ใช่ | ตัวระบุเฉพาะของ marketplace |
| `version` | ไม่ | เวอร์ชันสคีมาของ marketplace (semver) |
| `metadata.description` | ใช่ | คำอธิบาย marketplace ที่อ่านเข้าใจได้ |
| `owner.name` | ใช่ | องค์กรหรือผู้ใช้ที่เป็นเจ้าของ marketplace |
| `owner.url` | ไม่ | URL ไปยังโปรไฟล์หรือเว็บไซต์ของเจ้าของ |
| `owner.email` | ไม่ | อีเมลสำหรับติดต่อ |
| `plugins` | ใช่ | อาร์เรย์ของรายการปลั๊กอิน |

### ฟิลด์รายการปลั๊กอิน

| ฟิลด์ | จำเป็น | คำอธิบาย |
|-------|----------|-------------|
| `name` | ใช่ | ตัวระบุปลั๊กอิน (ต้องตรงกับ `plugin.json`) |
| `description` | ใช่ | คำอธิบายสั้น ๆ ของปลั๊กอิน |
| `version` | ใช่ | สตริงเวอร์ชันแบบ semantic |
| `author.name` | ใช่ | ผู้พัฒนาปลั๊กอิน |
| `source` | ใช่ | เส้นทางหรือ URL ไปยังปลั๊กอิน (ดูประเภทของ Source) |
| `category` | ใช่ | หมวดหมู่ปลั๊กอิน (เช่น `productivity`) |
| `homepage` | ไม่ | URL ไปยังเอกสารประกอบหรือหน้าแรกของปลั๊กอิน |
| `license` | ไม่ | ตัวระบุสัญญาอนุญาต SPDX |
| `keywords` | ไม่ | อาร์เรย์ของคำค้นหา |
| `tags` | ไม่ | อาร์เรย์ของแท็กสำหรับการกรอง |
| `repository` | ไม่ | URL ของรีโพซิทอรี |

## ประเภทของ Source ของปลั๊กอิน

ฟิลด์ `source` ใน marketplace.json รองรับหลายรูปแบบ:

| ประเภท | ตัวอย่าง | คำอธิบาย |
|------|---------|-------------|
| เส้นทางสัมพัทธ์ | `./plugins/my-plugin` | ปลั๊กอินในรีโพซิทอรีเดียวกัน |
| รูปแบบย่อ GitHub | `owner/repo` | ปลั๊กอินที่รูทของรีโพซิทอรี |
| GitHub พร้อมเส้นทาง | `owner/repo/path/to/plugin` | ปลั๊กอินในไดเรกทอรีย่อย |
| URL ของ Git | `https://github.com/owner/repo.git` | รีโพซิทอรี Git |
| แพ็กเกจ npm | `npm:@scope/package` | แพ็กเกจ npm ที่เผยแพร่แล้ว |

## สคีมา plugin.json

แต่ละปลั๊กอินมีไฟล์แมนิเฟสต์อยู่ที่
`.claude-plugin/plugin.json` ภายในไดเรกทอรีของมัน

| ฟิลด์ | จำเป็น | คำอธิบาย |
|-------|----------|-------------|
| `name` | ใช่ | ตัวระบุปลั๊กอิน |
| `description` | ใช่ | สิ่งที่ปลั๊กอินทำ |
| `version` | ใช่ | เวอร์ชันแบบ semantic |
| `author.name` | ใช่ | ผู้พัฒนาปลั๊กอิน |
| `author.url` | ไม่ | URL ไปยังโปรไฟล์หรือเว็บไซต์ของผู้พัฒนา |
| `homepage` | ไม่ | URL ของเอกสารประกอบ |
| `keywords` | ไม่ | คำค้นหา |
| `license` | ไม่ | ตัวระบุสัญญาอนุญาต SPDX |
| `repository` | ไม่ | URL ของรีโพซิทอรี |

## โครงสร้างไดเรกทอรีของปลั๊กอิน

```
plugins/my-plugin/
  .claude-plugin/
    plugin.json          # Plugin manifest (required)
  skills/
    skill-name/
      SKILL.md           # Skill definition (required per skill)
      references/        # Reference data files (optional)
  commands/
    command-name.md      # Command definition (one per command)
  agents/
    agent-name.md        # Agent definition (one per agent)
  README.md              # Plugin documentation (recommended)
```

## ตัวแปรสภาพแวดล้อม

ปลั๊กอินสามารถใช้ตัวแปรเหล่านี้ในไฟล์สกิลและคำสั่ง:

| ตัวแปร | คำอธิบาย |
|----------|-------------|
| `$\{CLAUDE_PLUGIN_ROOT\}` | เส้นทางสมบูรณ์ไปยังไดเรกทอรีรูทของปลั๊กอิน |

## การตรวจสอบความถูกต้อง

ตรวจสอบแมนิเฟสต์ของ marketplace และปลั๊กอินในเครื่องของคุณ:

```bash
claude plugin validate .
```

คำสั่งนี้จะตรวจสอบ:

- ความสอดคล้องกับสคีมาของ `marketplace.json`
- ไดเรกทอรีปลั๊กอินที่อ้างอิงทั้งหมดมีอยู่จริง
- แต่ละปลั๊กอินมี `plugin.json` ที่ถูกต้อง
- สกิลมี frontmatter ของ `SKILL.md` ที่จัดรูปแบบถูกต้อง
- คำสั่งมีฟิลด์ frontmatter ที่จำเป็น

## การกำหนดค่าทีม

### การกำหนดค่า marketplace ล่วงหน้า

เพิ่มใน `.claude/settings.json` ในรีโพซิทอรีใดก็ได้:

```json
{
  "extraKnownMarketplaces": [
    "f5xc-salesdemos/marketplace"
  ]
}
```

### การเปิดใช้งานปลั๊กอินล่วงหน้า

```json
{
  "enabledPlugins": [
    "f5xc-docs-tools@f5xc-salesdemos-marketplace"
  ]
}
```

<Aside type="tip">
  การคอมมิตการตั้งค่าเหล่านี้ไปยังรีโพซิทอรีของคุณจะทำให้สมาชิกทุกคนในทีม
  ได้รับการกำหนดค่า marketplace และปลั๊กอินเดียวกันโดยอัตโนมัติ
</Aside>

## การแก้ไขปัญหา

### ไม่พบ Marketplace

ตรวจสอบว่า marketplace ถูกเพิ่มแล้ว:

```
/plugin marketplace list
```

หากไม่พบ ให้เพิ่มใหม่อีกครั้ง:

```
/plugin marketplace add f5xc-salesdemos/marketplace
```

### ปลั๊กอินไม่ทำงาน

- ยืนยันว่าปลั๊กอินถูกติดตั้งแล้ว:
  `/plugin list`
- ตรวจสอบว่าเวอร์ชัน Claude Code ของคุณรองรับปลั๊กอิน
  (v1.0.33+)
- ตรวจสอบว่าเงื่อนไขการทริกเกอร์ของสกิลตรงกับบริบทปัจจุบันของคุณ

### สกิลไม่ทริกเกอร์โดยอัตโนมัติ

สกิลจะทำงานตามฟิลด์ `description` ใน frontmatter ของ
SKILL.md หากสกิลไม่ทำงานตามที่คาดหวัง:

- ตรวจสอบว่าคำอธิบายระบุบริบทการทริกเกอร์ได้อย่างถูกต้อง
- ใช้คำสั่งโดยตรง (เช่น `/f5xc-docs-tools:review-mdx`)
  เป็นวิธีแก้ปัญหาชั่วคราว

### ข้อผิดพลาดในการบิลด์หลังจากตรวจสอบ

ปลั๊กอินจะรายงานปัญหาแต่ไม่ได้แก้ไขโดยอัตโนมัติ ให้นำการแก้ไขที่แนะนำ
ไปใช้ด้วยตนเอง จากนั้นเรียกใช้การตรวจสอบอีกครั้งเพื่อยืนยันว่าปัญหาได้รับการแก้ไขแล้ว
