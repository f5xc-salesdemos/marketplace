ปลั๊กอิน **f5xc-devcontainer** ให้การรับรู้คอนเทนเนอร์สำหรับ
สภาพแวดล้อมการพัฒนา f5-sales-demo โดยดูแลรักษา
แคตตาล็อกเครื่องมือที่ครอบคลุม เปิดใช้งานการตรวจสอบคอนเทนเนอร์แบบสดๆ
และจัดการการติดตั้งเครื่องมือพร้อมการติดตามความคงอยู่ผ่าน
GitHub issues

<Badge text="v1.1.6" variant="note" /> <Badge text="Productivity" variant="tip" />

## การติดตั้ง

```
/plugin install f5xc-devcontainer@f5-sales-demo-marketplace
```

## ทักษะ

### tool-catalog

ฐานความรู้ของเครื่องมือ CLI ทุกชิ้นที่ติดตั้งใน คอนเทนเนอร์สำหรับพัฒนา
จัดระเบียบตามหมวดหมู่ (คลาวด์, เครือข่าย, ความปลอดภัย, การพัฒนา
ฯลฯ) เปิดใช้งานเมื่อคุณถามว่าควรใช้เครื่องมือใด วิธีใช้เครื่องมือ
มีการติดตั้งบางอย่างหรือไม่ หรือต้องการความช่วยเหลือกับ
การดำเนินการเฉพาะ

### self-awareness

ทักษะการระบุตัวตนและการวินิจฉัยตนเองของคอนเทนเนอร์ ให้การ
ตรวจสอบคอนเทนเนอร์ที่กำลังทำงานแบบสดๆ ผ่าน GitHub API และ
ข้อมูลเมตาการบิลด์ในเครื่อง เปิดใช้งานสำหรับคำถามเกี่ยวกับตัวตน เวอร์ชัน
ข้อมูลการบิลด์ การตรวจสอบสุขภาพ และประวัติผู้มีส่วนร่วม

<Aside type="tip">
  ถาม "who are you?" หรือ "what version am I running?" เพื่อเรียกใช้
  self-awareness
</Aside>

## เอเจนต์

### tool-advisor

อ่านแคตตาล็อกเครื่องมือและส่งคืนคำแนะนำพร้อมวัตถุประสงค์
คำสั่งเริ่มต้นอย่างรวดเร็ว และข้อกำหนดการยืนยันตัวตนสำหรับคำถาม
เกี่ยวกับเครื่องมือ CLI ทุกชิ้น

### tool-auditor

ตรวจสอบ Dockerfile ของ คอนเทนเนอร์สำหรับพัฒนา เทียบกับไฟล์อ้างอิง
แคตตาล็อกเครื่องมือเพื่อตรวจจับความเบี่ยงเบน &mdash; เครื่องมือที่เพิ่มลงใน
image แต่ยังไม่ได้จัดหมวดหมู่ หรือรายการแคตตาล็อกสำหรับเครื่องมือที่
ถูกลบออกไปแล้ว

### container-introspector

ดำเนินการตรวจสอบคอนเทนเนอร์แบบสดๆ ผ่าน GitHub API และ
ข้อมูลเมตาในเครื่องสำหรับตัวตน ลำดับวงศ์ตระกูล การวินิจฉัยตนเอง
ประวัติผู้มีส่วนร่วม และสายการบิลด์ เฉพาะการดำเนินการอ่านอย่างเดียวเท่านั้น

### container-maintainer

ติดตั้ง ลบ ค้นหา และอัปเดตเครื่องมือ CLI ในคอนเทนเนอร์ที่กำลังทำงาน
โดยใช้รูปแบบตัวจัดการแพ็กเกจที่ถูกต้อง ยื่น GitHub issues ต่อ
ที่เก็บ คอนเทนเนอร์สำหรับพัฒนา เพื่อการติดตามความคงอยู่
