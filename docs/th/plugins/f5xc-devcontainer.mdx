ปลั๊กอิน **f5xc-devcontainer** ให้ความสามารถในการรับรู้คอนเทนเนอร์สำหรับ
สภาพแวดล้อมการพัฒนา f5xc-salesdemos โดยจะดูแลรักษา
แค็ตตาล็อกเครื่องมือที่ครอบคลุม เปิดใช้งานการตรวจสอบสดภายในคอนเทนเนอร์
และจัดการการติดตั้งเครื่องมือพร้อมการติดตามความคงอยู่ผ่าน
GitHub issues

<Badge text="v1.1.6" variant="note" /> <Badge text="Productivity" variant="tip" />

## การติดตั้ง

```
/plugin install f5xc-devcontainer@f5xc-salesdemos-marketplace
```

## ทักษะ

### tool-catalog

ฐานความรู้ของเครื่องมือ CLI ทุกตัวที่ติดตั้งใน devcontainer
จัดหมวดหมู่ตามประเภท (คลาวด์ เครือข่าย ความปลอดภัย การพัฒนา
ฯลฯ) จะเปิดใช้งานเมื่อคุณถามว่าควรใช้เครื่องมือใด วิธีใช้
เครื่องมือ มีบางอย่างติดตั้งอยู่หรือไม่ หรือต้องการความช่วยเหลือเกี่ยวกับ
การดำเนินการเฉพาะ

### self-awareness

ทักษะการระบุตัวตนของคอนเทนเนอร์และการวินิจฉัยตนเอง ให้การ
ตรวจสอบสดภายในคอนเทนเนอร์ที่กำลังทำงานผ่าน GitHub API และ
เมตาดาต้าการสร้างในเครื่อง จะเปิดใช้งานสำหรับคำถามเกี่ยวกับตัวตน เวอร์ชัน
ข้อมูลการสร้าง การตรวจสอบสุขภาพ และประวัติผู้มีส่วนร่วม

<Aside type="tip">
  ถาม "who are you?" หรือ "what version am I running?" เพื่อเรียกใช้
  self-awareness
</Aside>

## เอเจนต์

### tool-advisor

อ่านแค็ตตาล็อกเครื่องมือและส่งคืนคำแนะนำพร้อมจุดประสงค์
คำสั่งเริ่มต้นอย่างรวดเร็ว และข้อกำหนดการยืนยันตัวตนสำหรับคำถาม
เกี่ยวกับเครื่องมือ CLI ใดๆ

### tool-auditor

ตรวจสอบ Dockerfile ของ devcontainer เทียบกับไฟล์อ้างอิงแค็ตตาล็อกเครื่องมือ
เพื่อตรวจจับความคลาดเคลื่อน &mdash; เครื่องมือที่เพิ่มเข้าไปในอิมเมจ
แต่ไม่ได้ลงทะเบียนในแค็ตตาล็อก หรือรายการแค็ตตาล็อกสำหรับเครื่องมือที่ถูก
ลบออกไปแล้ว

### container-introspector

ดำเนินการตรวจสอบสดภายในคอนเทนเนอร์ผ่าน GitHub API และเมตาดาต้า
ในเครื่องสำหรับตัวตน สายสัมพันธ์ การวินิจฉัยตนเอง ประวัติ
ผู้มีส่วนร่วม และสายการสร้าง เป็นการดำเนินการแบบอ่านอย่างเดียวเท่านั้น

### container-maintainer

ติดตั้ง ลบ ค้นหา และอัปเดตเครื่องมือ CLI ในคอนเทนเนอร์ที่กำลังทำงาน
โดยใช้รูปแบบตัวจัดการแพ็คเกจที่ถูกต้อง สร้าง GitHub issues
เทียบกับ repository ของ devcontainer สำหรับการติดตามความคงอยู่
