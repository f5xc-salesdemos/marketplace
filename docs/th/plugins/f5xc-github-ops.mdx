ปลั๊กอิน **f5xc-github-ops** ให้ระบบอัตโนมัติสำหรับการดำเนินงาน
GitHub โดยเฉพาะสำหรับ repository ของ f5-sales-demo ปลั๊กอินนี้
จัดการวงจรชีวิตของ Git และ GitHub ทั้งหมดตั้งแต่การสร้าง Issue
จนถึงการตรวจสอบหลังการ merge โดยบังคับใช้การพัฒนาที่ขับเคลื่อนด้วย
Issue และ lint gate ก่อนการ commit

<Badge text="v2.2.2" variant="note" /> <Badge text="ผลิตภาพ" variant="tip" />

## การติดตั้ง

```
/plugin install f5xc-github-ops@f5-sales-demo-marketplace
```

## ทักษะ

### workflow-lifecycle

เวิร์กโฟลว์การกำกับดูแล repository ที่จัดการการสร้าง Issue, กฎการตั้งชื่อ
branch, การสร้าง PR, การ poll CI พร้อมส่งข้อผิดพลาดกลับไปที่ Issue,
การตรวจสอบหลังการ merge, การตรวจยืนยัน และเกณฑ์การเสร็จสิ้นงาน
จะทำงานเมื่อคุณพูดว่า "commit", "push", "create a PR", "merge"
หรือดำเนินการ Git ใด ๆ

### upstream-contribution

เวิร์กโฟลว์การมีส่วนร่วมกับ upstream สำหรับการพัฒนาแบบ fork
จัดการการตรวจจับ fork, การค้นคว้า upstream (issues, PRs, CONTRIBUTING.md,
รูปแบบ CI), การสร้าง Issue และ PR ข้ามรีโพ, การจัดการ Issue ติดตาม
ในเครื่อง, การตรวจสอบ CI ของ upstream และการแนะนำการแก้ไข

## เอเจนต์

### github-ops

เอเจนต์เฉพาะสำหรับการดำเนินงาน Git และ GitHub เชิงกลไกทั้งหมด
จัดการ lint gate ก่อนการ commit, การสร้าง Issue, การสร้าง branch,
การ staging, การ commit, การ push, การสร้าง PR, การ poll CI
พร้อมส่งข้อผิดพลาดกลับ, การลองใหม่เมื่อโครงสร้างพื้นฐานล้มเหลว,
การตรวจสอบหลังการ merge, การล้าง branch และการจัดการการตั้งค่า
repository ผ่าน GitHub API

ส่งรายงานสถานะแบบมีโครงสร้าง: `COMPLETE`,
`PRE_COMMIT_FAILED`, `CI_FAILED`, `BLOCKED` หรือ `FAILED`

<Aside type="caution">
  เอเจนต์นี้เป็นตัวจัดการ**เฉพาะ**สำหรับการดำเนินงาน Git
  อย่ารัน `git commit`, `git push` หรือ `gh pr create` โดยตรง
  ใน repository ที่ใช้ปลั๊กอินนี้
</Aside>
