ปลั๊กอิน **f5xc-sales-engineer** มอบเฟรมเวิร์กบุคลิกภาพ Sales Engineer
ที่นำกลับมาใช้ซ้ำได้สำหรับคลังเก็บสาธิต F5 XC โดยจัดการวงจรชีวิต
การประชุมสาธิตทั้งหมด — ตั้งแต่การตรวจสอบก่อนการประชุม ผ่านการดำเนินการจริง
ถาม-ตอบ และการรื้อถอน

<Badge text="v1.0.6" variant="note" /> <Badge text="Productivity" variant="tip" />

## การติดตั้ง

```
/plugin install f5xc-sales-engineer@f5xc-salesdemos-marketplace
```

## ทักษะ

### demo-executor

การดำเนินการสาธิตที่ขับเคลื่อนด้วย API พร้อมวงจรชีวิตการประชุมสี่ขั้นตอน:

1. **เตรียมการ** — การตรวจสอบสภาพแวดล้อมก่อนการประชุมผ่านเอเจนต์
   `demo-housekeeping`
2. **ดำเนินการ** — การดำเนินการตามเฟสแบบกำหนดได้พร้อมการบรรยาย
   สำหรับลูกค้า
3. **ถาม-ตอบ** — โหมดด้นสดพร้อมสภาพแวดล้อมจริง
4. **รื้อถอน** — การทำความสะอาดหลังการประชุมผ่านเอเจนต์ `demo-housekeeping`

อ่านเนื้อหาเฉพาะผลิตภัณฑ์จาก `DEMO_PRODUCT_EXPERTISE.md` และคำสั่ง
สาธิตจาก `docs/demo/`

### presenter

การนำเสนอแบบ walkthrough ตามการสร้างจริง โดยใช้สภาพแวดล้อมสาธิตที่
กำหนดค่าไว้ล่วงหน้า ทำตามรูปแบบ แสดง → บรรยาย → เชื่อมโยง → หยุดรอ
ในแต่ละขั้นตอน อ่านลำดับ walkthrough จาก `DEMO_WALKTHROUGH_CONFIG.md`

### subject-matter-expert

การถาม-ตอบที่อ้างอิงจากแหล่งข้อมูลพร้อมข้อกำหนดการอ้างอิงที่เข้มงวด
ไม่เดาเด็ดขาด — ทุกคำตอบมีการอ้างอิงหรือหลักฐาน มอบหมายการวิจัย
ให้เอเจนต์ `demo-researcher` เมื่อจำเป็น

### sales-engineer

ดัชนีบทบาทที่กำหนดเส้นทางเจตนาของผู้ใช้ไปยังทักษะบุคลิกภาพที่ถูกต้อง
เปิดใช้งานเมื่อคำขอเกี่ยวข้องกับการสาธิตแต่ไม่ตรงกับวลีทริกเกอร์
ที่เฉพาะเจาะจง

## เอเจนต์

### demo-housekeeping

เอเจนต์อัตโนมัติสำหรับการตรวจสอบก่อนการประชุมและการรื้อถอนหลังการประชุม
รัน Readiness Verification Matrix (T0–T5) ที่กำหนดไว้ใน
`DEMO_READINESS_MATRIX.md` และดำเนินการคำสั่งรื้อถอนจาก
`docs/demo/phase-4-teardown.mdx`

### demo-researcher

บรรณารักษ์วิจัยแบบอ่านอย่างเดียวที่ค้นหาคำตอบที่ตรวจสอบแล้วพร้อม
การอ้างอิง ค้นหา `docs/` ในเครื่องก่อน จากนั้นค้นหาแหล่งข้อมูลภายนอก
ที่จัดทำดัชนีจาก `DEMO_SOURCE_INDEX.md` โดยมี WebSearch เป็นทางเลือกสำรอง

## ไฟล์ข้อตกลง

<Aside type="tip" title="เนื้อหาเฉพาะผลิตภัณฑ์">
  ปลั๊กอินนี้ไม่ขึ้นกับผลิตภัณฑ์ แต่ละคลังเก็บจะจัดเตรียมเนื้อหา
  เฉพาะผลิตภัณฑ์ผ่านไฟล์ข้อตกลงที่เป็นมาตรฐาน
</Aside>

สร้างไฟล์เหล่านี้ในรูทของคลังเก็บของคุณเพื่อกำหนดค่าปลั๊กอิน:

| ไฟล์ | วัตถุประสงค์ | จำเป็นสำหรับ |
| ---- | ------- | ------------ |
| `DEMO_PRODUCT_EXPERTISE.md` | ความสามารถของผลิตภัณฑ์ สัญญาณการตรวจจับ ขอบเขตภัยคุกคาม การจัดแนวการปฏิบัติตามข้อกำหนด การอ้างอิง API | ทุกทักษะ |
| `DEMO_WALKTHROUGH_CONFIG.md` | URL แอปสาธิต ลำดับ walkthrough จังหวะเวลาการตรวจจับ | presenter |
| `DEMO_SOURCE_INDEX.md` | แค็ตตาล็อกแหล่งข้อมูลวิจัยพร้อมเอกสารในเครื่องและ URL ภายนอก | demo-researcher |
| `DEMO_READINESS_MATRIX.md` | ตัวแปรที่จำเป็น/ทางเลือก การตรวจสอบความพร้อม (T0–T5) จุดปลาย API | demo-housekeeping |
| `docs/demo/` | ไฟล์เฟสพร้อมคำสั่ง cURL และเกณฑ์หลักฐาน | demo-executor |

## รูปแบบการบรรยาย

ทุกบุคลิกภาพใช้รูปแบบการบรรยายที่สอดคล้องกัน:

- **ปัจจุบันกาล บุรุษที่หนึ่งพหูพจน์** — "สิ่งที่เรากำลังดูอยู่ตรงนี้…"
- **หนึ่งประเด็นต่อหนึ่งย่อหน้า** — แต่ละการบรรยายตอบ: นี่คืออะไร?
  ทำไมถึงสำคัญ? ควรทำอย่างไร?
- **ระบุชื่อสัญญาณ** — ชี้ให้เห็นว่าความสามารถใดกำลังทำงาน
- **จุดเชื่อมโยงการปฏิบัติตามข้อกำหนดเมื่อเกี่ยวข้อง** — กล่าวถึง
  การจัดแนวกฎระเบียบอย่างเป็นธรรมชาติ
