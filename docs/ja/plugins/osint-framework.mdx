**osint-framework** プラグインは、OSINTツールカタログと
調査スキルを提供します — [osintframework.com](https://osintframework.com) からマッピングされた
34カテゴリにわたる1,064の無料インテリジェンス収集ツールが含まれています。
カテゴリベースのスキル、実行可能な調査パイプライン、
CLIツール実行、OPSEC対応ワークフロー、および調査間の
相関分析のための永続エンティティグラフが含まれています。

<Badge text="v1.0.1" variant="note" /> <Badge text="Security" variant="tip" />

## インストール

```
/plugin install osint-framework@f5xc-salesdemos-marketplace
```

## コマンド

### /osint-investigate

任意のターゲットに対してフルOSINT調査を実行します。ターゲットの種類
（メールアドレス、ドメイン、IP、ユーザー名、企業、人名）を自動検出し、
適切なカテゴリスキルにルーティングします。

```
/osint-investigate j.smith@example.com
/osint-investigate example.com
/osint-investigate 198.51.100.1
/osint-investigate example-user
```

### /osint-search

機能クエリに一致するツールをツールカタログから検索します。

```
/osint-search subdomain enumeration
/osint-search email breach check
/osint-search satellite imagery
```

### /osint-catalog

カテゴリ別にフルツールカタログを閲覧します。引数なしの場合、
ツール数付きの全34カテゴリを表示します。

```
/osint-catalog
/osint-catalog domain
/osint-catalog threat-intel
```

## スキル

### osint-index

トップレベルのインテントルーター。「〜について情報を探す」、
「調査する」、「検索する」、「身元調査」などの発言や、
オープンソースインテリジェンスタスクの説明時に自動的に
アクティブになります。ターゲットの種類に基づいて適切な
カテゴリスキルにルーティングします。

| ターゲット | ルーティング先 |
|--------|----------|
| ユーザー名またはハンドル | `username-recon` |
| メールアドレス | `email-recon` |
| ドメイン名 | `domain-recon` |
| IPアドレス | `ip-address-recon` |
| 人名 + 企業 | `people-search` + `business-records` |
| 企業 / 組織 | `business-records` + `domain-recon` |
| マルウェアハッシュ | `malicious-file-analysis` + `threat-intelligence` |

### カテゴリスキル（全34種）

各カテゴリには、CLIツール、Webリソース、調査ワークフロー、
クロスカテゴリピボット、OPSECノートを備えた専用スキルがあります。

| カテゴリ | ツール数 | 主要CLIツール |
|----------|-------|--------------|
| `username-recon` | 18 | sherlock, maigret, sylva |
| `email-recon` | 28 | holehe, h8mail, theHarvester |
| `domain-recon` | 131 | subfinder, amass, whois, dig |
| `cloud-recon` | 23 | cloud_enum, S3Scanner, ScoutSuite |
| `ip-address-recon` | 55 | nmap, masscan, checkip |
| `images-videos` | 89 | exiftool, exiv2, yt-dlp |
| `social-networks` | 63 | maigret, yt-dlp, Osintgram |
| `threat-intelligence` | 38 | iocextract, ioc_parser, pymisp |
| `malicious-file-analysis` | 33 | oletools, pdfid, Ghidra |
| `search-engines` | 73 | gitleaks, pagodo, gobuster |
| `geolocation` | 47 | Hyperlapse |
| `opsec` | 47 | 47のWebベースプライバシーツール |
| `people-search` | 18 | GitHub API, GitLab API |
| `business-records` | 28 | SEC EDGAR, OpenCorporates |
| *(その他20カテゴリ)* | 500+ | — |

<Aside type="tip">
  `/osint-catalog` で全34カテゴリを閲覧するか、
  `/osint-search <機能>` で特定のタスクに対応するツールを検索できます。
</Aside>

### osint-catalog

ツールカタログブラウザ。フルツールレジストリ、カテゴリインデックス、
調査パイプライン、レート制限リファレンス、および相関エンジンの
ドキュメントへのアクセスを提供します。

## エージェント

### osint-investigator

自律型マルチツール調査オーケストレーター。フル調査ワークフローを
実行します — ツール可用性チェック、ターゲット種類の検出、
順次ツール実行（パッシブ優先）、結果の集約、信頼度レベルと
OPSEC分類を含む構造化レポートの生成。

エンティティグラフと統合されており、各ツールの結果は
`/tmp/osint-graph/` にエンティティとリレーションシップを作成し、
調査間の相関分析に活用されます。

<Aside type="caution">
  アクティブ偵察ツール（nmap、nuclei、sherlock）はターゲット
  インフラストラクチャに直接接触します。OPSECの警告フックが
  アクティブツールの実行前にアラートを発します。必ず事前に
  認可を取得してください。
</Aside>

### osint-researcher

読み取り専用のカタログ検索エージェント。参照ファイルを検索して、
指定されたタスクに最適なツールを推奨します。ツールを実行することは
なく、推奨と説明のみを行います。カテゴリスキルがツール発見のために
内部的に使用します。

### osint-executor

単一ツールCLI実行エージェント。特定のパラメータで1つのツールを
実行し、出力を解析して構造化された結果を返します。20以上のツールに
対する正確なフラグ、出力形式、jq解析パターンを含むツールごとの
実行レシピが含まれています。

## レート制限

すべてのAPI呼び出しは、HTTP 429/503に対する指数バックオフが
組み込まれた `osint_curl` ヘルパーを使用します。主要な制限:

| API | 無料制限 | 環境変数 |
|-----|-----------|---------|
| ipinfo.io | 1,000/日 | — |
| crt.sh | 60/分 | — |
| NVD | 5/30秒（キーなし） | `NVD_API_KEY` |
| OpenCorporates | キーが必要 | `OPENCORPORATES_API_KEY` |
| Shodan | 1/秒 | `SHODAN_API_KEY` |
| GitHub API | 60/時（認証なし） | `GH_TOKEN` |

キーが必要なAPIは、環境変数が設定されていない場合、
明確なメッセージとともに自動的にスキップされます。

## 相関エンジン

プラグインには、調査間で発見を蓄積する永続的なJSONエンティティグラフが
`/tmp/osint-graph/` に含まれています。

```bash
source plugins/osint-framework/scripts/osint-graph.sh
osint_graph_init

# エンティティの追加
P=$(osint_entity_add "person" "J. Smith" --tool github-api)
C=$(osint_entity_add "company" "F5" --tool github-api)

# リンクの作成
osint_rel_add "$P" "$C" "works_at" --tool github-api

# クエリ: 人物から2ホップ以内のすべてのエンティティ
osint_graph_query "$P" 2

# レポートの生成
osint_graph_report
```

**信頼度スコアリング** はマルチソースフュージョンを使用します:
各確認ソースに対して `confidence = 1 - product(1 - ci)`。
0.80の2つのソース → 結合信頼度0.96。
