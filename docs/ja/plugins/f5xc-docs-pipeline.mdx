**f5xc-docs-pipeline** プラグインは、f5xc-salesdemos リポジトリ向けのドキュメントパイプラインアーキテクチャの知識を提供します。コンテンツ作成をガイドし、マルチリポジトリビルド全体にわたる設定の所有権を明確にし、ローカルの Docker ベースのプレビューを提供します。

<Badge text="v1.0.2" variant="note" /> <Badge text="生産性" variant="tip" />

## インストール

```
/plugin install f5xc-docs-pipeline@f5xc-salesdemos-marketplace
```

## スキル

### content-author

ファイル構造、MDX 構文ルール、フロントマター要件、画像参照、およびローカル Docker プレビューを網羅するコンテンツ作成ガイドです。`docs/` コンテンツの作成や編集、MDX ファイルの操作、またはローカルプレビューのセットアップ時にアクティブになります。

### pipeline-navigator

4つの上流リポジトリ（docs-theme、docs-builder、docs-control、docs-icons）全体にわたって、どこで変更を行うべきかを説明するドキュメントパイプラインアーキテクチャです。設定の所有権（`astro.config.mjs`、`content.config.ts`）、リリースディスパッチチェーン、および管理対象ファイルの追跡を明確にします。

<Aside type="tip">
  特定の設定ファイルやコンポーネントをどのリポジトリが所有しているかを理解する必要がある場合は、pipeline-navigator を使用してください。
</Aside>

## コマンド

### /preview-docs

```
/f5xc-docs-pipeline:preview-docs
```

ドキュメントをプレビューするためのローカル Docker 開発サーバーを起動します。共有 Docker ビルドイメージを実行し、ローカルの `docs/` ディレクトリをマウントしてライブプレビューを行います。
