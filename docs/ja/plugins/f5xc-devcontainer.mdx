**f5xc-devcontainer** プラグインは、f5-sales-demo 開発環境に
コンテナ認識機能を提供します。包括的なツールカタログを維持し、
ライブコンテナイントロスペクションを有効にし、GitHub issues を通じた
永続化追跡を伴うツールインストールを処理します。

<Badge text="v1.1.6" variant="note" /> <Badge text="Productivity" variant="tip" />

## インストール

```
/plugin install f5xc-devcontainer@f5-sales-demo-marketplace
```

## スキル

### tool-catalog

開発コンテナにインストールされているすべての CLI ツールの
ナレッジベースで、カテゴリー（クラウド、ネットワーク、セキュリティ、
開発など）ごとに整理されています。使用すべきツール、ツールの使い方、
インストールされているかどうか、または特定の操作についてサポートが
必要な場合にアクティブになります。

### self-awareness

コンテナのアイデンティティと自己診断スキル。GitHub API および
ローカルビルドメタデータを通じて、実行中のコンテナのライブ
イントロスペクションを提供します。アイデンティティ、バージョン、
ビルド情報、ヘルスチェック、およびコントリビューター履歴に関する
質問に対してアクティブになります。

<Aside type="tip">
  「who are you?」または「what version am I running?」と尋ねることで
  self-awareness をトリガーできます。
</Aside>

## エージェント

### tool-advisor

ツールカタログを参照し、CLI ツールに関するあらゆる質問に対して、
目的、クイックスタートコマンド、および認証要件を含む推奨事項を
返します。

### tool-auditor

ツールカタログのリファレンスファイルに対して開発コンテナの
Dockerfile を監査し、ドリフトを検出します &mdash; イメージに
追加されたがカタログ化されていないツール、またはすでに削除された
ツールのカタログエントリを特定します。

### container-introspector

GitHub API およびローカルメタデータを通じてライブコンテナ
イントロスペクションを実行し、アイデンティティ、系譜、自己診断、
コントリビューター履歴、およびビルド系統を提供します。
読み取り専用の操作のみ対応しています。

### container-maintainer

正しいパッケージマネージャーパターンを使用して、実行中のコンテナ内の
CLI ツールのインストール、削除、検索、および更新を行います。
永続化追跡のために、開発コンテナリポジトリに対して GitHub issues を
作成します。
