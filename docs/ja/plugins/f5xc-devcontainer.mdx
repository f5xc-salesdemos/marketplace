**f5xc-devcontainer** プラグインは、f5xc-salesdemos 開発環境にコンテナ認識機能を提供します。包括的なツールカタログを管理し、ライブコンテナイントロスペクションを可能にし、GitHub issues を介した永続性追跡によるツールインストールを処理します。

<Badge text="v1.1.6" variant="note" /> <Badge text="生産性" variant="tip" />

## インストール

```
/plugin install f5xc-devcontainer@f5xc-salesdemos-marketplace
```

## スキル

### tool-catalog

devcontainer にインストールされているすべての CLI ツールのナレッジベースで、カテゴリ別（クラウド、ネットワーキング、セキュリティ、開発など）に整理されています。どのツールを使用すべきか、ツールの使い方、何かがインストールされているかどうか、または特定の操作に関するヘルプが必要な場合に起動します。

### self-awareness

コンテナのアイデンティティと自己診断スキル。GitHub API とローカルビルドメタデータを介して、実行中のコンテナのライブイントロスペクションを提供します。アイデンティティ、バージョン、ビルド情報、ヘルスチェック、コントリビューター履歴に関する質問で起動します。

<Aside type="tip">
  「あなたは誰ですか？」や「実行中のバージョンは何ですか？」と尋ねると、self-awareness が起動します。
</Aside>

## エージェント

### tool-advisor

ツールカタログを読み取り、CLI ツールに関する質問に対して、目的、クイックスタートコマンド、認証要件を含む推奨事項を返します。

### tool-auditor

devcontainer の Dockerfile をツールカタログの参照ファイルと照合して監査し、ドリフトを検出します &mdash; イメージに追加されたがカタログに登録されていないツール、または削除されたツールのカタログエントリを検出します。

### container-introspector

GitHub API とローカルメタデータを介して、アイデンティティ、系譜、自己診断、コントリビューター履歴、ビルド系統のためのライブコンテナイントロスペクションを実行します。読み取り専用の操作のみです。

### container-maintainer

正しいパッケージマネージャーパターンを使用して、実行中のコンテナ内の CLI ツールのインストール、削除、検索、更新を行います。永続性追跡のために devcontainer リポジトリに対して GitHub issues を作成します。
