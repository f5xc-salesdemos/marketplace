# Mutual Action Plan Builder

Create a buyer-approved Mutual Action Plan that makes the Decision
Process explicit and ties every milestone to MEDDPICC elements.

## What is a MAP?

A Mutual Action Plan is a **customer-facing** document that both
seller and buyer agree to. It converts the Decision Process and
Paper Process into a concrete, actionable roadmap with dates, owners,
and exit criteria.

## MAP Design Protocol

### Step 1 — Gather inputs

Ask the user for:

- Account/deal name
- Target close date (working backward)
- Current stage and known Decision Process steps
- Key stakeholders (customer and seller side)
- Known procurement/legal/security requirements

### Step 2 — Build the plan

Structure the MAP around these sections:

1. **Business Context** — Pain, Metrics, and why now (from MEDDPICC)
2. **Evaluation Milestones** — Technical proof, workshops, demos
3. **Business Case** — Value model review, executive alignment
4. **Procurement** — Security, legal, vendor onboarding, signature
5. **Implementation Readiness** — Success plan, adoption milestones

Refer to [map-template.md](references/map-template.md) for the
full template.

### Step 3 — Assign ownership

Every task must have:

- **Owner:** Named person (customer or seller side)
- **Date:** Specific date, not "TBD"
- **Exit criteria:** How we know this step is complete
- **Dependencies:** What must happen first

### Step 4 — MEDDPICC alignment

Map each section to MEDDPICC elements:

| MAP Section | MEDDPICC Elements |
| ----------- | ----------------- |
| Business Context | Metrics + Pain |
| Evaluation Milestones | Decision Criteria + Decision Process |
| Business Case | Metrics + Economic Buyer |
| Procurement | Paper Process + Decision Process |
| Implementation Readiness | Metrics (value realization) |
| Sponsorship | Champion + Economic Buyer |

## Output Format

Produce the MAP in the format defined in
[map-template.md](references/map-template.md). The plan should be
ready to share with the customer after seller-side review.

## Coaching Notes

- A MAP without customer buy-in is just a seller's wish list
- Dates should work backward from the target close date
- Include buffer for procurement — it always takes longer
- The MAP is a living document — update it after every meeting
- If the customer won't agree to a MAP, that's a qualification signal
